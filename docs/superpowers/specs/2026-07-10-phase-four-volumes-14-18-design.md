# 第四阶段第 14-18 卷扩写设计

## 背景

项目已经完成第 01-13 卷：

```text
第01-05卷：基础工作流
-> 14天入门训练营
-> Ross 案例复盘
-> 第06-10卷：画面生成与导演判断
-> 第11-13卷：声音、音乐、剪辑与成片
```

第 14-18 卷是最后一组占位卷，主题从“学会制作”进入“商业项目、系列内容、自动化和变现”。读者仍然是完全零基础，因此第四阶段不能写成创业鸡血，也不能把自动化写成盲目批量生产。它应该回答：

```text
我已经能做一条 AI 视频，接下来怎样接商业项目、做短剧、提高效率、连接工具，并把能力变成可销售服务？
```

## 用户约束

- 用户希望先完成完整版后再开始学习。
- 用户对 AI 视频、自动化和商业流程均可视为零基础。
- 本阶段继续使用中文 Markdown 教程体例。
- 不写平台收益承诺、广告投放效果承诺、灰产打法、规避审核、搬运盗用或刷量方法。
- 不写死 n8n、MCP、广告平台、短剧平台、商单平台的价格、套餐、权限、界面按钮和收益规则。
- 自动化内容必须强调权限、密钥、失败重试、人工终审和安全边界。
- 商业内容必须强调客户沟通、授权、合同范围、复盘和长期维护。

## 备选方案

### 方案 A：一次完成第 14-18 卷

这是推荐方案。第 14-18 卷共同构成最后的“商业化和系统化”阶段，彼此引用频繁：

- 第 14 卷讲单个商业广告项目。
- 第 15 卷讲连续内容和短剧。
- 第 16 卷讲如何自动化重复流程。
- 第 17 卷讲 Agent 如何安全连接工具。
- 第 18 卷讲如何报价、交付、产品化和复购。

优点：

- 用户可以在完整版完成后从头学习。
- 最后一组占位会一次清空。
- 学习路线变成完整闭环。

风险：

- 内容量较大，需要严格分卷提交和核验。

### 方案 B：只扩写第 14-15 卷

只先完成案例卷，把广告和短剧讲清。

优点：

- 单次范围小。
- 更快得到可读案例。

缺点：

- 自动化、MCP 和变现仍然缺口较大。
- 用户仍不能开始“完整版学习”。

### 方案 C：先扩写第 16-17 卷

优先讲自动化和 MCP。

优点：

- 技术链更完整。

缺点：

- 零基础读者还没有商业场景，自动化容易变成无目的折腾。
- 不符合“先完成完整版”的当前目标。

## 推荐设计

采用方案 A。第四阶段以“项目案例 -> 系列内容 -> 自动化 -> 工具连接 -> 商业变现”的顺序完成全书。

最终学习路线变成：

```text
第01-05卷：基础工作流
-> 14天入门训练营
-> Ross 案例复盘
-> 第06-10卷：画面生成与导演判断
-> 第11-13卷：声音、音乐、剪辑与成片
-> 第14-18卷：商业案例、自动化与变现
```

## 第 14 卷设计：商业广告案例

第 14 卷以一个虚构但真实可执行的商业广告项目为主线，讲清从客户需求到交付归档的流程。

需要覆盖：

- 客户访谈。
- Brief。
- 报价边界。
- 脚本和分镜。
- 品牌风格控制。
- 产品与人物镜头生成。
- 配音、音乐、字幕和剪辑。
- 客户修改管理。
- 交付文件包。
- 项目复盘。

核心模板：

- 客户需求访谈表。
- 广告 Brief。
- 报价范围说明。
- 商业广告分镜表。
- 客户修改记录。
- 交付确认表。

## 第 15 卷设计：AI 短剧案例

第 15 卷讲 AI 短剧，不追求“做一部长剧”，而是让新手做一个低成本试播集。

需要覆盖：

- AI 短剧和普通短视频区别。
- 人设表。
- 关系表。
- 世界观和场景表。
- 分集大纲。
- 单集剧本。
- 对话和配音。
- 镜头连续性。
- 试播集制作。
- 数据复盘和是否继续。

核心模板：

- 人设表。
- 人物关系表。
- 世界观表。
- 场景资产表。
- 单集大纲表。
- 试播集复盘表。

## 第 16 卷设计：n8n 自动化

第 16 卷讲自动化，但必须避免“什么都自动化”的误区。自动化只适合重复、规则明确、低风险、可回滚的环节。

需要覆盖：

- n8n 基本概念。
- 节点、触发器、凭证、表达式、执行记录。
- 自动化适合和不适合的环节。
- 选题收集。
- 脚本草稿。
- 文件归档。
- 生成任务清单。
- 下载和记录。
- 失败重试。
- 人工终审。
- 成本控制。

核心模板：

- AI 视频自动化流程图。
- 节点清单。
- 凭证清单。
- 失败重试表。
- 批量任务记录表。
- 人工终审清单。

## 第 17 卷设计：MCP 工作流

第 17 卷讲 MCP，让读者理解 Agent 为什么需要工具、资源和安全边界。它不应变成协议细节大全，而要服务 AI 视频项目。

需要覆盖：

- MCP 基本概念。
- Tool、Resource、Prompt 的区别。
- Agent 为什么需要连接文件、浏览器、平台 API 和素材库。
- 文件系统 MCP。
- 浏览器 MCP。
- 视频平台或生成平台接口。
- LibTV Skill 或类似工作流。
- 权限最小化。
- 密钥和账号安全。
- 日志、审计和回滚。

核心模板：

- MCP 能力清单。
- Agent 权限表。
- 工具调用审计表。
- 安全边界检查表。
- Codex + MCP 视频项目示例流程。

## 第 18 卷设计：商业变现

第 18 卷把全书收束到商业能力，不承诺收入，而是讲如何把 AI 视频能力变成可销售、可交付、可复购的服务。

需要覆盖：

- AI 视频服务类型。
- 报价方式。
- 成本核算。
- 项目周期。
- 客户沟通。
- 版权、肖像、声音、音乐和素材授权。
- 个人 IP 内容。
- 模板、课程和顾问产品。
- 维护和复购。
- 不建议做的变现方式。

核心模板：

- 服务菜单。
- 报价表。
- 成本核算表。
- 客户沟通流程。
- 授权确认表。
- 复购维护计划。

## 资料来源策略

第四阶段涉及商业、自动化和协议事实，必须分层处理：

- n8n 功能事实优先引用 n8n 官方文档。
- MCP 概念事实优先引用 Model Context Protocol 官方文档和 OpenAI 官方 MCP/Codex 文档。
- 广告与商业合规只写原则性提醒，不提供法律意见。
- AI 版权、广告声明、代言披露等内容优先记录官方监管或版权机构资料，并明确“需按所在地法律和平台规则确认”。
- 短剧平台规则、流量收益和商业合作规则不写死。
- 报价、收益、成本只给核算方法，不给保底承诺。
