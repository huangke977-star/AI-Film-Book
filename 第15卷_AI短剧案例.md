# 第 15 卷：AI 短剧案例

## 学习目标

学完本卷后，你应该能做到：

- 理解 AI 短剧和普通短视频、广告片、单条剧情视频的区别。
- 会用“试播集”思路降低制作风险，而不是一开始规划几十集。
- 能建立人设表、人物关系表、世界观表和场景资产表。
- 会把故事拆成分集大纲、单集剧本和镜头表。
- 知道人物一致性、场景连续性、对白、配音、音乐和剪辑在短剧里的作用。
- 能做一集低成本 AI 短剧试播，并用数据和观众反馈决定是否继续。
- 知道短剧商业化、平台规则、版权和演员声音等问题需要谨慎处理。

本卷不教你做一部长剧。它教你从一集试播开始。

## 本卷导读：短剧先做试播集

很多人听到 AI 短剧，第一反应是：

```text
我要做一个 100 集连续剧。
```

这通常会失败。因为短剧不是把镜头生成得好看就行，它还需要：

- 持续人物。
- 稳定关系。
- 可延展冲突。
- 每集钩子。
- 场景复用。
- 对白节奏。
- 配音和字幕。
- 数据反馈。

AI 可以降低试错成本，但不能替你解决所有叙事问题。

新手最稳的路径是：

```text
先做 1 集试播
-> 控制在 30 到 60 秒
-> 只用 2 到 3 个主要人物
-> 只用 1 到 2 个主要场景
-> 测试观众是否愿意看下一集
```

如果试播集都无法让人看懂，就不要急着扩成系列。

## 1. AI 短剧和普通短视频的区别

| 维度 | 普通短视频 | AI 短剧 |
| --- | --- | --- |
| 核心 | 一个观点或一个卖点 | 一个连续故事 |
| 人物 | 可以没有固定人物 | 需要稳定人设 |
| 场景 | 可以每条不同 | 需要可复用场景 |
| 结构 | 开头钩子 + 信息输出 | 冲突 + 选择 + 悬念 |
| 制作难点 | 信息清楚 | 连续性和情绪 |
| 复盘 | 看完播率和互动 | 看是否期待下一集 |

AI 短剧更像“连续内容产品”，不是单条视频。

### 1.1 AI 短剧适合的题材

新手适合：

- 都市轻剧情。
- 职场小冲突。
- 科幻概念短篇。
- 反转小故事。
- 品牌人物小剧场。
- 教程类剧情包装。

新手不适合一开始做：

- 大规模战争。
- 多人群像。
- 复杂古装。
- 长时间动作戏。
- 多地点追逐。
- 需要大量真实演员表演的剧情。

不是不能做，而是第一次成本太高。

## 2. 短剧最小闭环

最小闭环不是完整剧集季，而是一集能看懂的试播。

```text
一句话设定
-> 主角
-> 冲突
-> 选择
-> 反转或悬念
-> 是否想看下一集
```

### 2.1 试播集目标

试播集只验证三件事：

1. 人物是否有吸引力。
2. 冲突是否容易看懂。
3. 观众是否愿意看下一集。

不需要验证所有世界观。

### 2.2 试播集限制

```text
时长：30 到 60 秒
人物：2 到 3 个
场景：1 到 2 个
镜头：6 到 10 个
冲突：1 个
悬念：1 个
```

限制不是降低质量，而是让你能做完。

## 3. 人设表

短剧先有人，再有剧情。

### 3.1 人设模板

```markdown
# Character Card

## 角色编号

CHAR_LIN_001

## 姓名

林夏

## 年龄

27

## 外观

短发，浅色风衣，干净利落，城市白领感。

## 性格

理性、克制、观察力强，不轻易相信别人。

## 欲望

想查清公司内部一个 AI 项目失控的真相。

## 弱点

太相信证据，容易忽略人的情绪。

## 说话方式

短句，直接，不喜欢废话。

## 视觉关键词

冷色办公楼、玻璃反光、夜晚电脑屏幕。

## 禁止变化

不要换发型，不要换年龄感，不要变成夸张特工风。
```

### 3.2 角色数量

第一集最好只做：

- 主角。
- 对手或阻碍者。
- 信息提供者。

不要一上来做 8 个人。AI 人物一致性会变难，观众也记不住。

## 4. 人物关系表

短剧的冲突来自关系。

```markdown
| 人物 A | 人物 B | 表面关系 | 真实关系 | 冲突 |
| --- | --- | --- | --- | --- |
| 林夏 | 周屿 | 同事 | 周屿知道项目秘密 | 林夏不信任他 |
| 林夏 | 程总 | 下属和上司 | 程总在隐瞒风险 | 林夏想查清真相 |
| 周屿 | 程总 | 技术负责人和老板 | 周屿被迫沉默 | 周屿想保护某人 |
```

关系表的好处是让每句对白都有目的。

## 5. 世界观和场景表

AI 短剧要控制世界规模。

### 5.1 世界观一句话

```text
一家短视频 AI 公司开发出能预测爆款内容的系统，但系统开始提前生成还未发生的偷拍视频。
```

这句话提供：

- 类型：都市科幻悬疑。
- 场景：AI 公司。
- 冲突：技术和伦理。
- 延展：系统为什么会预测未来。

### 5.2 场景表

```markdown
| 场景编号 | 场景 | 用途 | 视觉关键词 | 是否复用 |
| --- | --- | --- | --- | --- |
| SCENE_OFFICE_NIGHT | 夜晚办公室 | 主角发现异常 | 冷色屏幕、玻璃墙、空工位 | 是 |
| SCENE_MEETING_ROOM | 会议室 | 上司施压 | 长桌、投影、低光 | 是 |
| SCENE_ELEVATOR | 电梯 | 短促对话 | 金属反光、狭窄空间 | 可选 |
```

场景越少，连续性越稳。

## 6. 分集大纲

不要一开始写完整剧本，先写分集大纲。

### 6.1 三集试播大纲

```markdown
## 第 1 集：系统提前生成了她的辞职视频

林夏深夜加班，发现系统生成了一条自己第二天辞职的视频。她以为是同事恶作剧，但视频里的细节还没有发生。

结尾悬念：视频最后出现一句字幕：不要相信周屿。

## 第 2 集：周屿说系统不是在预测

林夏质问周屿。周屿告诉她，系统不是预测，而是在影响人的选择。林夏不信。

结尾悬念：系统生成了程总受伤的画面。

## 第 3 集：她按视频做了相反选择

林夏故意改变行动路线，却发现自己仍然走向视频里的结果。

结尾悬念：系统界面出现“第 4 集生成中”。
```

每集只解决一个问题，并留下一个新问题。

## 7. 单集剧本

以第 1 集为例。

```markdown
# Episode 01 Script

## 主题

系统生成了主角未来的视频。

## 时长

45 秒。

## 场景

夜晚办公室。

## 人物

林夏、周屿。

## 剧情

林夏加班检查系统输出，发现一条以自己为主角的视频。视频里她站在会议室里递交辞职信。她惊讶，因为这件事还没有发生。周屿突然出现，试图让她关掉系统。林夏拒绝。视频最后跳出一句字幕：不要相信周屿。
```

### 7.1 对白版本

```text
林夏：这是谁生成的？
周屿：别看了，关掉。
林夏：视频里的人是我。可这件事还没发生。
周屿：所以你更应该关掉。
林夏：你知道它是什么。
周屿：我只知道，继续看下去，你会后悔。
```

对白要短。短剧不是小说朗读。

## 8. 镜头连续性

AI 短剧最大难点是连续性。

### 8.1 连续性检查

| 项目 | 检查 |
| --- | --- |
| 人物 | 发型、服装、年龄、气质是否一致 |
| 场景 | 办公室布局、灯光、色调是否一致 |
| 道具 | 电脑、屏幕、辞职信是否前后一致 |
| 情绪 | 人物从惊讶到警惕是否连贯 |
| 动作 | 上一个镜头做的动作，下一个镜头是否接得上 |

### 8.2 镜头表

```markdown
| 镜头 | 时长 | 内容 | 连续性重点 |
| --- | --- | --- | --- |
| S01 | 4s | 夜晚办公室远景 | 建立空间 |
| S02 | 5s | 林夏看屏幕 | 服装和发型固定 |
| S03 | 6s | 屏幕视频里的辞职画面 | 不生成可读复杂文字 |
| S04 | 5s | 周屿出现在门口 | 周屿外观固定 |
| S05 | 8s | 两人对话 | 用近景和反打 |
| S06 | 5s | 屏幕出现警告字幕 | 后期添加真实字幕 |
| S07 | 5s | 林夏回头看周屿 | 情绪转为怀疑 |
```

复杂动作越少，连续性越稳。

## 9. 对话和配音

短剧配音和广告旁白不同。角色说话要符合人设。

### 9.1 角色声音

| 角色 | 声音 |
| --- | --- |
| 林夏 | 冷静、克制、语速偏快 |
| 周屿 | 压低声音、紧张、想阻止她 |
| 程总 | 稳重、有压迫感 |

### 9.2 配音注意

- 不要每个角色都像同一个旁白。
- 角色台词要短。
- 情绪不要过度表演。
- 如果使用声音克隆，要有明确授权。
- 如果用 TTS，保存每个角色的声音卡。

### 9.3 字幕

短剧字幕要帮助观众理解情节，但不要把画面塞满。

```text
这是谁生成的？
别看了，关掉。
这件事还没发生。
```

关键悬念字幕可以单独突出：

```text
不要相信周屿。
```

这类文字最好后期添加，不依赖视频模型生成。

## 10. 低成本试播集

低成本不是低质量，而是控制变量。

### 10.1 试播集制作路线

```text
写一句话设定
-> 写 3 个主要人物
-> 写 3 集大纲
-> 只制作第 1 集
-> 控制 7 个镜头以内
-> 使用 1 个主场景
-> 完成配音、字幕和音乐
-> 发布或内部测试
-> 复盘是否继续
```

### 10.2 文件目录

```text
short_drama_pilot/
├── series_bible.md
├── characters/
│   ├── CHAR_LIN_001.md
│   └── CHAR_ZHOU_001.md
├── scenes/
│   └── SCENE_OFFICE_NIGHT.md
├── episodes/
│   ├── E01_outline.md
│   └── E01_script.md
├── storyboard/
│   └── E01_storyboard.md
├── prompts/
├── outputs/
├── audio/
├── edit/
└── review/
```

## 11. 数据复盘

短剧不是你觉得好看就继续做。要看反馈。

### 11.1 复盘指标

| 指标 | 看什么 |
| --- | --- |
| 完播率 | 观众有没有看完 |
| 前 3 秒留存 | 开头是否抓人 |
| 评论 | 观众是否讨论人物和剧情 |
| 收藏 | 是否有人想回看 |
| 关注 | 是否带来系列期待 |
| 私信或互动 | 是否有人问下一集 |

不要只看播放量。播放量可能受推荐影响，评论内容更能说明剧情是否成立。

### 11.2 观众反馈分类

```markdown
| 反馈 | 类型 | 处理 |
| --- | --- | --- |
| 想看下一集 | 正向 | 保留设定 |
| 没看懂系统是什么 | 信息不足 | 第 2 集补解释 |
| 人物有点不像同一个人 | 连续性问题 | 强化参考图 |
| 对白太像念稿 | 表演问题 | 改短句和配音 |
```

## 12. 是否继续制作

试播集后有三种选择：

| 情况 | 处理 |
| --- | --- |
| 人物被记住，悬念有效 | 做第 2 集 |
| 设定有兴趣，但第 1 集不清楚 | 重剪或重写第 1 集 |
| 观众完全没反应 | 暂停，换设定或换表达方式 |

继续制作前，要补齐：

- Series Bible。
- Character Bible。
- 场景资产。
- 音乐主题。
- 字幕风格。
- 前 3 集大纲。

不要一边做一边随意改设定，否则后面会乱。

## 13. 完整案例：未来视频系统

### 13.1 一句话设定

```text
一家 AI 公司开发出能生成未来片段的系统，女主发现系统第一个预测对象是自己。
```

### 13.2 类型

```text
都市科幻悬疑，竖屏短剧，每集 45 秒。
```

### 13.3 主角

```markdown
林夏，27 岁，AI 内容审核负责人。她相信证据，不相信直觉。她的目标是查清系统为什么生成未来片段。
```

### 13.4 第 1 集镜头任务

```markdown
| 镜头 | 生成提示重点 |
| --- | --- |
| S01 | 夜晚办公室，冷色屏幕光，空旷工位 |
| S02 | 林夏坐在电脑前，神情疲惫，屏幕反光 |
| S03 | 屏幕里播放一段模糊会议室视频，文字后期加 |
| S04 | 周屿站在门口，表情紧张 |
| S05 | 林夏回头，眼神警惕 |
| S06 | 屏幕特写，黑底字幕后期添加 |
```

### 13.5 试播集结尾

```text
屏幕黑下来。
字幕出现：不要相信周屿。
林夏慢慢回头。
周屿站在门口，没有解释。
切黑。
```

这是一个可继续的悬念。

## 14. Series Bible：短剧项目总说明

只要你准备做超过一集，就需要 Series Bible。它不是给观众看的，而是给你自己、Codex、配音、剪辑和后续协作者看的。

```markdown
# Series Bible

## 剧名

未来视频系统

## 类型

都市科幻悬疑，竖屏短剧。

## 一句话设定

AI 公司开发出能生成未来片段的系统，主角发现系统正在改变现实。

## 主题

当算法比人更早知道人的选择，人还剩多少自由？

## 目标观众

喜欢科幻悬疑、职场秘密、反转剧情的短视频用户。

## 单集长度

45 到 60 秒。

## 视觉风格

夜晚办公室、冷色屏幕光、玻璃反光、低饱和、真实科技感。

## 主要人物

- 林夏：内容审核负责人，理性，想查清真相。
- 周屿：技术负责人，知道系统秘密，想阻止林夏。
- 程总：公司负责人，隐藏项目风险。

## 固定场景

- 夜晚办公室。
- 会议室。
- 电梯。

## 每集结构

开头 3 秒出现异常。
中段人物做选择。
结尾出现新的未来片段。

## 禁止

不要突然变成超能力动作片。
不要扩大到太多组织和角色。
不要让系统规则每集都变。
```

Series Bible 的价值是防止你在第 3 集忘记第 1 集设定。

## 16. 制作排期

短剧容易失控，所以要排期。

### 16.1 一集试播排期

| 阶段 | 时间 | 输出 |
| --- | --- | --- |
| 设定 | 半天 | 一句话设定、人设、场景表 |
| 剧本 | 半天 | E01 剧本、镜头表 |
| 资产 | 1 天 | 角色参考图、场景参考图 |
| 生成 | 1 到 2 天 | 6 到 8 个候选镜头 |
| 声音 | 半天 | 对白配音、音乐 |
| 剪辑 | 1 天 | 试播集成片 |
| 复盘 | 半天 | 数据和反馈记录 |

这只是教学排期。真实项目会因为工具、客户、素材和修改增加时间。

### 16.2 每集制作清单

```markdown
| 项目 | E01 | E02 | E03 |
| --- | --- | --- | --- |
| 大纲 | 完成 | 完成 | 草稿 |
| 剧本 | 完成 | 草稿 | 未写 |
| 镜头表 | 完成 | 未写 | 未写 |
| 角色参考 | 完成 | 复用 | 复用 |
| 场景参考 | 完成 | 补会议室 | 复用 |
| 配音 | 完成 | 未做 | 未做 |
| 剪辑 | 完成 | 未做 | 未做 |
```

短剧不要只靠脑子记进度。

## 17. 发布和复盘表

试播集发布后，先不要立刻做第 2 集。至少等一轮反馈。

### 17.1 发布记录

```markdown
| 平台 | 发布时间 | 标题 | 封面 | 版本 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 抖音 | 2026-07-10 20:00 | AI 系统生成了我的未来 | COVER_E01_v01.png | E01_v01 | 竖屏 |
| 小红书 | 2026-07-10 21:00 | 如果 AI 提前生成了你明天的视频 | COVER_E01_v02.png | E01_v01 | 标题略改 |
```

### 17.2 复盘问题

```markdown
| 问题 | 观察 |
| --- | --- |
| 观众是否理解设定 |  |
| 主角是否被记住 |  |
| 悬念是否有效 |  |
| 哪一秒流失最多 |  |
| 评论在问什么 |  |
| 是否值得做第 2 集 |  |
```

复盘的目标不是证明自己做得好，而是决定下一步。

## 18. 商业化前的边界

如果短剧要用于商业合作，要先明确：

- IP 归属。
- 角色形象归属。
- 生成素材来源。
- 配音和音乐授权。
- 客户是否可复用角色。
- 是否允许衍生广告。
- 是否需要给平台或客户提供源文件。

短剧角色一旦有观众记忆，就不再只是一次性素材。角色、世界观和字幕风格都可能变成资产。

### 18.1 合作说明模板

```markdown
# Short Drama Collaboration Scope

## 项目

AI 短剧试播集 E01。

## 交付

- 1 条 45 秒竖屏短剧。
- 1 张封面。
- 1 份角色和场景说明。
- 1 份字幕文件。

## 不包含

- 后续集数制作。
- 角色长期授权。
- 线下拍摄。
- 广告投放操作。

## 需要另行确认

- IP 归属。
- 角色形象复用。
- 音乐和配音商用范围。
- 客户是否能自行二次剪辑。
```

这能避免试播集做完后，双方对“后续能怎么用”理解不同。

## 19. 短剧选题评估表

开做之前，先给题材打分。

```markdown
| 维度 | 问题 | 分数 |
| --- | --- | --- |
| 一句话是否清楚 | 10 秒内能否说清设定 |  |
| 主角是否明确 | 观众能否记住主角 |  |
| 冲突是否持续 | 是否能延展 3 集以上 |  |
| 场景是否可控 | 是否能用 1 到 3 个主要场景完成 |  |
| 人物数量是否少 | 试播集是否不超过 3 个主要人物 |  |
| 视觉是否可生成 | 是否避免复杂群戏和大动作 |  |
| 结尾是否有钩子 | 每集能否留下新问题 |  |
| 成本是否可承受 | 是否能在现有工具和时间内完成 |  |
```

评分不是为了精确计算，而是防止自己被一个听起来很大的设定带跑。新手最好的短剧题材，通常不是世界观最大，而是最容易让观众在 5 秒内理解冲突。

### 19.1 适合 AI 试播的题材特征

- 场景少。
- 人物少。
- 冲突清楚。
- 对白短。
- 镜头动作克制。
- 视觉风格统一。
- 每集有一个明确问题。

### 19.2 暂时不适合的题材特征

- 每集都换城市。
- 多个家族和组织同时出现。
- 大量打斗和追车。
- 十几个角色同时出场。
- 需要复杂真实表演。
- 需要非常准确的历史服化道。

这些不是永远不能做，而是不适合新手试播。先做小，才能做完。

如果一个题材第一集就需要解释大量背景、展示多个阵营、切换很多地点，那它更适合以后再做。新手阶段，最重要的是让观众立刻看懂：谁遇到了什么麻烦，他为什么必须现在做选择。

试播集的成功标准也很简单：观众能记住主角，能说出冲突，愿意知道下一步会发生什么。只要这三点成立，画面还可以继续打磨；如果这三点不成立，画面再精致也救不了系列。

先让故事成立，再让画面升级。

这是 AI 短剧最稳的学习顺序。

先做一集，复盘一集，再决定下一集；这比一次写满全季更适合新手，也更适合 AI 工作流。

稳住。

## 20. 常见错误

### 错误 1：开局人物太多

修正：试播集只保留 2 到 3 个主要人物。

### 错误 2：设定太大

修正：世界观只露出和本集冲突有关的一小部分。

### 错误 3：每集没有钩子

修正：每集开头有问题，结尾有新问题。

### 错误 4：人物外观漂移

修正：使用 Character Bible、参考图和稳定服装。

### 错误 5：对白太解释

修正：让人物带着目的说话，不要把世界观全塞进台词。

## 21. 故障排查

### 观众看不懂

处理：

- 检查第一集是否一次讲太多设定。
- 增加一个清楚的视觉目标。
- 用字幕补充关键关系。
- 删除不必要支线。

### 人物不像同一个人

处理：

- 固定发型、服装、年龄和脸部特征。
- 先生成角色参考图。
- 镜头动作不要太复杂。
- 避免长镜头和大幅转身。

### 场景不连续

处理：

- 减少场景数量。
- 建立场景表。
- 用参考图固定空间。
- 不要每个镜头都重新描述不同办公室。

### 对白不自然

处理：

- 改成短句。
- 每句台词都要有目的。
- 让角色说话方式不同。
- 配音时控制情绪，不要过度表演。

### 不知道是否继续

处理：

- 看评论是否提到人物、设定或下一集。
- 看完播率和前 3 秒留存。
- 做一次小范围重剪测试。
- 不要因为自己喜欢就无限投入。

## 23. 本卷总结

AI 短剧的稳定路线是：

```text
先做试播
-> 控制人物
-> 控制场景
-> 写清关系
-> 每集一个冲突
-> 结尾留悬念
-> 复盘观众是否想继续
```

不要从 100 集开始。先让 1 集成立。

短剧不是素材堆叠，而是人物在冲突中做选择。AI 可以生成画面，但人物欲望、关系、选择和悬念，仍然需要你设计。

## 24. 课后练习

### 练习 1：一句话设定

写 3 个 AI 短剧设定，每个不超过 40 字。

### 练习 2：人设表

为其中一个设定写 2 个角色卡。

### 练习 3：三集大纲

写 3 集大纲，每集必须有结尾悬念。

### 练习 4：试播集镜头表

为第 1 集写 6 到 8 个镜头，并标注连续性重点。
