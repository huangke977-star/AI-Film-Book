# 第 10 卷：AI 绘图

## 学习目标

学完本卷后，你应该能做到：

- 理解 AI 图像不是孤立作品，而是 AI 视频项目里的视觉资产。
- 区分概念图、角色参考图、场景参考图、道具参考图、封面图、分镜参考图和图生视频起点。
- 知道什么时候先生成图片，再用图片生成视频。
- 能为参考图建立命名、归档、版本和用途说明。
- 能把第 09 卷的人物一致性要求转成角色参考图工作流。
- 能根据项目目标选择图像工具，而不是追逐永久排名。
- 能把图像提示词转换成视频生成任务卡。

本卷不追求把所有绘图工具讲一遍，而是讲“AI 图像如何服务视频项目”。

## 本卷导读：AI 图像是视频项目的视觉地基

很多人学习 AI 绘图时，只关注一张图好不好看。但在 AI 视频里，一张图的价值不只是好看，而是能不能帮助后续生成更稳定。

它可能是：

- 角色参考图。
- 场景参考图。
- 产品参考图。
- 分镜构图参考。
- 封面图。
- 图生视频输入。
- 客户确认稿。

也就是说，AI 图像是项目资产，不只是朋友圈作品。

稳定的视频项目应该有：

```text
references/
assets/
outputs/
```

而不是把所有图片散落在下载目录里。

## 1. AI 绘图和 AI 视频的关系

AI 图像和 AI 视频的关系可以这样理解：

| 图像资产 | 视频中的作用 |
| --- | --- |
| 概念图 | 帮你确定整体视觉方向 |
| 角色参考图 | 固定人物外观 |
| 场景参考图 | 固定空间风格 |
| 道具/产品参考图 | 固定商业对象 |
| 分镜参考图 | 帮模型理解构图 |
| 封面图 | 提升发布点击率 |
| 图生视频起点 | 让视频生成更可控 |

如果没有图像资产，你每个镜头都要靠文字重新描述世界。这样很容易漂移。

有图像资产后，工作流变成：

```text
brief.md
-> style_guide.md
-> concept images
-> reference images
-> storyboard
-> image-to-video tasks
-> iteration_log.md
```

## 2. 概念图工作流

概念图回答的是：“这个项目大概长什么样？”

它不一定直接进入成片，但会影响风格手册和后续参考图。

### 2.1 什么时候做概念图

适合在这些时候做：

- 项目风格不明确。
- 客户需要先看方向。
- 团队对“电影感”“科技感”“真实感”理解不一致。
- 你准备做多镜头系列，需要先统一视觉世界。

### 2.2 概念图 Prompt

```text
为一条 9:16 AI 视频短片生成概念图。
主题：AI 视频从手搓提示词进入项目化工作流。
画面：夜晚工作室，一位创作者坐在电脑前，屏幕上是抽象项目文件结构，桌面有分镜笔记和键盘。
风格：真实纪录片 + 轻微科技感，低饱和，屏幕冷光，暖色台灯，浅景深。
不要：可读中文、品牌 logo、夸张赛博朋克、卡通风格。
```

### 2.3 概念图评审

```markdown
| 文件 | 是否符合主题 | 风格是否可延展 | 能否作为参考 | 问题 | 结论 |
| --- | --- | --- | --- | --- | --- |
| CONCEPT_workflow_v01.png | 是 | 是 | 是 | 屏幕太亮 | 保留 |
| CONCEPT_workflow_v02.png | 部分 | 否 | 否 | 太赛博朋克 | rejected |
```

概念图不是一次定终身。它是帮助你做判断。

## 3. 角色参考图

角色参考图直接影响第 09 卷的人物一致性。

### 3.1 角色参考图类型

| 类型 | 用途 | 文件名示例 |
| --- | --- | --- |
| 正面头像 | 固定脸 | REF_character_host_v01_front.png |
| 半身图 | 固定服装 | REF_character_host_v01_half_body.png |
| 全身图 | 固定体型 | REF_character_host_v01_full_body.png |
| 表情图 | 固定情绪范围 | REF_character_host_v01_expression_calm.png |
| 侧面图 | 支持转头镜头 | REF_character_host_v01_side.png |

### 3.2 角色参考图 Prompt

```text
生成一个虚拟口播主持人的角色参考图。
年轻亚洲男性，25 到 30 岁，短黑发，椭圆脸，自然肤色，灰绿色连帽衫。
气质冷静、专注、可信，像认真做项目的创作者。
半身正面，平视，干净背景，真实纪录片质感，自然光，不要夸张笑容，不要手势，不要品牌 logo。
```

### 3.3 角色参考图检查

| 检查项 | 问题 |
| --- | --- |
| 脸 | 是否清楚 |
| 年龄 | 是否符合角色圣经 |
| 发型 | 是否稳定 |
| 服装 | 是否可跨镜头保持 |
| 气质 | 是否符合导演阐述 |
| 权利 | 是否不侵犯真人肖像 |

如果角色参考图本身不稳定，后续视频更难稳定。

## 4. 场景参考图

场景参考图固定空间世界。

### 4.1 常见场景参考

| 场景 | 用途 |
| --- | --- |
| 夜晚工作室 | AI 创作者、Codex 工作流 |
| 城市夜景 | 科技感开场 |
| 桌面俯拍 | 项目文件、分镜、提示词 |
| 抽象生成界面 | 视频生成过程 |
| 产品展示台 | 商业广告 |

### 4.2 场景参考图 Prompt

```text
生成一张 9:16 夜晚创作者工作室场景参考图。
空间：小型工作室，电脑桌，显示器，键盘，笔记本，台灯，窗外城市灯光。
光线：屏幕冷光 + 暖色台灯，低饱和，真实纪录片质感。
构图：桌面和屏幕是视觉中心，底部留字幕空间。
不要：可读文字、品牌 logo、过度赛博朋克、凌乱桌面。
```

### 4.3 场景图如何进入视频

场景图可以用于：

- 给视频模型做图生视频输入。
- 给 Codex 生成更具体的分镜。
- 给客户确认视觉方向。
- 给剪辑师和设计师做封面或字幕风格参考。

## 5. 道具和产品参考图

商业视频经常需要固定产品和道具。

### 5.1 为什么要单独做

如果你在每个视频镜头里都用文字描述产品，模型可能会改变产品形状、颜色和比例。

道具参考图可以固定：

- 产品外形。
- 颜色。
- 材质。
- 角度。
- 使用方式。

### 5.2 文件命名

```text
REF_prop_laptop_v01_angle.png
REF_product_bottle_v01_front.png
REF_product_box_v01_hero.png
```

### 5.3 商业提醒

真实品牌产品最好使用客户提供的官方素材或经过授权的图片。AI 生成的“像某个品牌”的产品图可能带来误导和侵权风险。

## 6. 封面图

封面图不是视频生成素材，但它影响点击。

### 6.1 封面图要解决的问题

- 观众是否知道视频主题。
- 主体是否清楚。
- 字幕位置是否有空间。
- 平台缩略图里是否能看懂。
- 是否和视频内容一致。

### 6.2 封面图 Prompt

```text
为一条抖音知识短视频设计封面背景图。
主题：AI 视频工作流。
画面：创作者工作室、电脑屏幕、抽象项目文件结构。
构图：人物或电脑在上半部分，下方留大标题空间。
风格：真实、干净、科技感、低饱和。
不要生成可读中文，标题后期添加。
```

封面里的中文标题建议后期加，不要依赖图像模型生成复杂中文。

## 7. 分镜参考图

分镜参考图用于帮助模型理解构图。

### 7.1 什么时候需要

- 文本很难描述构图。
- 客户需要先看画面方向。
- 某个镜头重复生成失败。
- 需要固定镜头空间关系。

### 7.2 分镜参考图命名

```text
SB_S01_v01_composition.png
SB_S02_v01_desk_topdown.png
SB_S03_v01_screen_closeup.png
```

### 7.3 分镜参考图和最终画面的关系

分镜参考图不一定要漂亮。它的任务是说明：

- 主体在哪里。
- 镜头多远。
- 字幕空间在哪里。
- 主要光线方向。
- 背景大概结构。

它更像工程图，不是海报。

## 8. 图生视频起点

当你需要更强可控性时，可以先图像，再视频。

### 8.1 适合图生视频的场景

| 场景 | 原因 |
| --- | --- |
| 固定人物 | 角色外观更稳定 |
| 固定产品 | 产品形状不易漂移 |
| 固定场景 | 空间风格一致 |
| 封面延展 | 从封面动起来 |
| 关键镜头 | 先确定构图，再做运动 |

### 8.2 图像到视频任务卡

```markdown
# S01 图生视频任务卡

## 输入图

references/REF_character_host_v01_front.png

## 镜头目的

建立虚拟主持人可信度。

## 运动

轻微呼吸，眼神自然变化，不做手势。

## 运镜

固定机位。

## 时长

3 到 5 秒。

## 禁止

不要换衣服，不要变年龄，不要转身，不要生成字幕。
```

### 8.3 图生视频的限制

参考图不是万能。它可能：

- 限制动作。
- 在大幅运动中崩坏。
- 让背景难以变化。
- 被模型错误理解。

所以图生视频适合“稳定关键镜头”，不适合一开始就做复杂剧情。

## 9. 常用图像工具的选择原则

工具会变，选择原则更稳定。

| 需求 | 选择维度 |
| --- | --- |
| 角色参考 | 脸部稳定、写实能力、参考图编辑 |
| 场景概念 | 风格控制、构图能力、画幅 |
| 封面设计 | 图文排版、后期可编辑性 |
| 产品图 | 结构准确、材质表现、授权清楚 |
| 图生视频前图 | 与视频模型兼容、主体清楚 |

不要问：

```text
哪个图像工具永远最好？
```

应该问：

```text
这个项目需要哪种图像资产？
这个工具是否擅长这种资产？
输出能不能进入后续视频流程？
```

OpenAI 官方图像生成文档把图像生成、编辑和参考输入放在 API 工作流中说明，具体模型、费用、质量、尺寸和组织要求要以当前官方页面为准。本书只把它作为“视频资产准备”的一个工具方向，不把任何工具写成永久唯一答案。

## 10. 从图像到视频的提示词转换

图像提示词和视频提示词不一样。

### 10.1 图像提示词关注

- 主体外观。
- 构图。
- 光线。
- 风格。
- 画幅。
- 背景。

### 10.2 视频提示词还要加

- 动作。
- 时长。
- 运镜。
- 运动幅度。
- 连续性。
- 失败风险。

### 10.3 转换示例

图像 Prompt：

```text
年轻创作者坐在夜晚工作室电脑前，屏幕冷光，暖色台灯，低饱和真实纪录片风格。
```

视频任务卡：

```markdown
## 输入图

references/REF_scene_creator_workspace_v01.png

## 动作

人物轻微呼吸，眼神看向屏幕。

## 运镜

very slow push in。

## 时长

4 秒。

## 禁止

不要改变人物服装，不要生成可读文字，不要快速镜头运动。
```

## 11. 参考图文件命名和归档

推荐目录：

```text
references/
├── characters/
├── scenes/
├── props/
├── covers/
└── storyboard/
```

命名规则：

```text
类型_对象_版本_角度或用途.ext
```

示例：

```text
REF_character_host_v01_front.png
REF_scene_workspace_v01_wide.png
REF_prop_laptop_v01_angle.png
COVER_douyin_v01_title.png
SB_S01_v01_composition.png
```

每张参考图都应该有用途说明：

```markdown
| 文件 | 用途 | 关联镜头 | 是否可商用 | 备注 |
| --- | --- | --- | --- | --- |
| REF_character_host_v01_front.png | 主持人正面参考 | S01、S05 | 待确认 | 固定脸和服装 |
```

## 12. Ross 风格视频中的图像资产推断

Ross 案例中可以观察到：

- 电影感人物和场景。
- 男性虚拟口播人物。
- 城市夜景和火箭发射场等背景。
- 手机播放画面。
- 黄色大字幕。

我们不能断言它的真实素材文件，但可以推断类似项目可能需要：

| 资产 | 用途 |
| --- | --- |
| 主持人参考图 | 稳定虚拟口播人物 |
| 城市夜景参考图 | 开场和背景 |
| 工作流界面参考 | 展示 Codex + LibTV |
| 封面背景图 | 发布封面 |
| 字幕样式参考 | 保持短视频识别度 |

如果你复刻工作流，不要先问“用了哪条提示词”，而要先问“哪些图像资产支撑了视频稳定性”。

## 13. 完整案例：从概念图到图生视频

下面用一条 30 秒视频演示完整流程。

主题：

```text
AI 视频不是一句提示词，而是一套项目化工作流。
```

### 13.1 资产清单

先列资产，不要先打开绘图工具。

```markdown
| 资产 | 用途 | 文件名 |
| --- | --- | --- |
| 概念图 | 确定整体风格 | CONCEPT_ai_workflow_v01.png |
| 主持人正面图 | 固定人物脸和服装 | REF_character_host_v01_front.png |
| 工作室场景图 | 固定主场景 | REF_scene_workspace_v01_wide.png |
| 桌面道具图 | 固定文件和键盘 | REF_prop_desk_v01_topdown.png |
| S01 分镜参考 | 开场构图 | SB_S01_v01_composition.png |
| 抖音封面背景 | 发布封面 | COVER_douyin_v01_bg.png |
```

### 13.2 第一步：生成概念图

目的：

```text
确定“真实纪录片 + 轻微科技感”的视觉方向。
```

不要把概念图直接当最终图。先用它判断：

- 色彩是否适合。
- 光线是否适合。
- 场景是否能延展到多个镜头。
- 是否给字幕留空间。

### 13.3 第二步：生成角色参考图

读取第 09 卷的 Character Bible：

```text
CHAR_HOST_001，年轻亚洲男性，短黑发，灰绿色连帽衫，冷静可信，动作克制。
```

生成角色参考图后，不要马上做视频。先问：

```text
这个人物能不能连续出现 5 个镜头？
服装是否容易保持？
脸是否足够清楚？
气质是否符合导演阐述？
```

如果答案不清楚，先改角色图。

### 13.4 第三步：生成场景参考图

场景图不需要出现人物。它负责固定空间：

```text
夜晚工作室
电脑桌
屏幕冷光
暖色台灯
城市窗景
低饱和真实质感
```

场景参考图可以帮助后续 S01、S02、S03 保持同一个视觉世界。

### 13.5 第四步：生成分镜参考图

分镜参考图用于说明构图：

| 镜头 | 分镜参考重点 |
| --- | --- |
| S01 | 人物和工作室的空间关系 |
| S02 | 桌面文件俯拍 |
| S03 | 屏幕特写和字幕空间 |
| S04 | 多版本素材对比 |
| S05 | 手机播放成片 |

分镜参考图可以粗糙，但构图必须清楚。

### 13.6 第五步：转成图生视频任务

示例：

```markdown
# S01 图生视频任务

## 输入图

references/characters/REF_character_host_v01_front.png

## 镜头目的

建立虚拟主持人的可信度。

## 动作

轻微呼吸，眼神自然看向镜头。

## 运镜

固定机位。

## 时长

4 秒。

## 禁止变化

不要换衣服，不要变年龄，不要转身，不要出现可读字幕。
```

### 13.7 第六步：归档和复盘

```markdown
| 文件 | 进入视频了吗 | 原因 |
| --- | --- | --- |
| REF_character_host_v01_front.png | 是 | 人物稳定 |
| REF_scene_workspace_v01_wide.png | 是 | 场景符合导演阐述 |
| CONCEPT_ai_workflow_v02.png | 否 | 风格过于赛博朋克 |
```

这样，图片不是堆在文件夹里，而是变成项目决策记录。

## 14. 图像资产评审表

每张图都可以用这张表评审：

```markdown
| 文件 | 用途 | 是否服务项目 | 是否和风格一致 | 是否能进入视频 | 风险 | 结论 |
| --- | --- | --- | --- | --- | --- | --- |
| REF_character_host_v01_front.png | 角色参考 | 是 | 是 | 是 | 需确认授权 | selected |
| REF_scene_workspace_v01.png | 场景参考 | 是 | 是 | 是 | 字幕空间不足 | selected_partial |
| COVER_v01.png | 封面 | 部分 | 是 | 否 | 主体太小 | 重新生成 |
```

### 14.1 评审优先级

1. 是否服务项目用途。
2. 是否和导演阐述一致。
3. 是否能进入后续视频流程。
4. 是否容易命名和复用。
5. 是否足够好看。

如果一张图漂亮但没有用途，它只是素材库，不是当前项目资产。

## 15. 常见项目目录

推荐把图像资产放进项目目录：

```text
project/
├── brief.md
├── director_statement.md
├── style_guide.md
├── character_bible.md
├── storyboard.md
├── references/
│   ├── concepts/
│   ├── characters/
│   ├── scenes/
│   ├── props/
│   ├── covers/
│   └── storyboard/
├── prompts/
├── outputs/
│   ├── raw/
│   ├── selected/
│   └── rejected/
└── iteration_log.md
```

### 15.1 references 和 outputs 的区别

| 目录 | 放什么 | 说明 |
| --- | --- | --- |
| references | 生成前的参考资产 | 用于指导生成 |
| outputs/raw | 生成后的原始结果 | 不一定可用 |
| outputs/selected | 可进入剪辑的结果 | 经过筛选 |
| outputs/rejected | 失败样本 | 用于复盘 |

不要把参考图放进 raw，也不要把生成失败的视频放进 references。

## 18. OpenAI 图像生成的使用边界

OpenAI 官方 image generation guide 说明了图像生成、编辑和以图像作为输入的工作流方向；GPT Image 2 模型页记录了模型相关能力和约束。实际使用时，必须以当前官方页面为准。

在本书里，你只需要先记住：

```text
OpenAI 图像能力可以作为视频项目资产生产的一种选择。
```

不要把它理解成唯一工具，也不要在教程里写死当前费用、速率、尺寸、质量和账号要求。这些属于会变化的信息。

更稳妥的学习方式是：

```text
先定义图像用途
-> 再选择工具
-> 再生成资产
-> 再进入视频工作流
```

### 19.2 客户确认清单

如果是商业项目，参考图要让客户确认：

| 确认项 | 问题 |
| --- | --- |
| 角色外观 | 是否符合品牌气质 |
| 场景风格 | 是否符合项目方向 |
| 产品外形 | 是否和真实产品一致 |
| 封面方向 | 是否允许作为发布封面 |
| 授权 | 是否可商用 |

不要等视频都生成完，才发现客户不接受角色参考图。图像资产越早确认，后面的返工越少。

## 20. 常见错误

### 错误 1：只追求漂亮图

修正：写清图片用途。它是角色参考、场景参考、封面，还是图生视频输入？

### 错误 2：参考图不命名

修正：放进 `references/`，按类型、对象、版本命名。

### 错误 3：让图像模型生成复杂中文

修正：中文标题和字幕后期添加。

### 错误 4：参考图和角色圣经冲突

修正：先改角色圣经，再改参考图，不要让两者互相打架。

### 错误 5：忽略商用授权

修正：真实客户、真人、品牌产品都要记录来源和授权。

## 21. 故障排查

### 角色图不稳定

处理：

- 减少风格词。
- 固定年龄、发型、服装。
- 先生成正面半身图。
- 不混用差异很大的参考图。

### 场景图太乱

处理：

- 删除无关物件。
- 明确视觉中心。
- 控制光线。
- 给字幕留空间。

### 封面图不适合发布

处理：

- 简化背景。
- 给大标题留空间。
- 主体不要太小。
- 文字后期加。

### 图生视频后变形

处理：

- 缩小动作。
- 固定机位。
- 缩短时长。
- 换更清晰的参考图。

## 22. 本卷总结

AI 绘图在视频项目中不是“额外装饰”，而是视觉地基。

稳定流程：

```text
先写用途
-> 生成概念图
-> 选择角色/场景/道具参考图
-> 命名归档
-> 写入分镜和任务卡
-> 用于图生视频或封面
-> 复盘是否服务项目
```

当你把图像当资产管理，视频生成会稳定很多。

新手最该记住的是：先问用途，再问美感。概念图服务方向，角色图服务一致性，场景图服务空间，封面图服务点击，分镜参考图服务构图，图生视频参考图服务稳定生成。用途不同，提示词、命名、评审标准都不同。

把这些图片管理好，就是在给后面的生成、剪辑和商业交付省时间。

资产清楚，工作流才清楚。

这就是本卷最重要的原则。

先归档，再生成。

再复盘。

稳住。
