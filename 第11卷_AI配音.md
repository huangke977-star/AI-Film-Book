# 第 11 卷：AI 配音

## 学习目标

学完本卷后，你应该能做到：

- 理解配音不是“给视频加声音”，而是给视频建立信息节奏。
- 区分旁白、口播、角色配音、解说和声音克隆。
- 把书面脚本改写成适合朗读的口播稿。
- 用 TTS 或 AI 配音工具制作多版本声音素材。
- 判断一段配音是否适合当前视频，而不是只听它像不像真人。
- 建立音频文件命名、版本管理、评审和入库流程。
- 知道声音克隆、真人声音、商业发布和授权风险的基本边界。
- 把配音和剪辑、字幕、音乐、音效配合起来。

本卷不是某个配音工具的按钮说明书。工具界面会变，声音创作的判断不会变。

## 本卷导读：声音是视频信息的时间轴

前 10 卷主要解决“画面怎么生成”。但一条视频真正被观众看懂，经常不是因为画面最炫，而是因为声音把信息交代清楚了。

声音决定三件事：

1. 观众什么时候听到关键信息。
2. 观众用什么情绪理解画面。
3. 观众愿不愿意继续看下去。

很多新手做 AI 视频时，画面已经有电影感，但一加配音就像廉价广告。原因通常不是声音工具差，而是没有把配音当成创作流程：

```text
脚本
-> 口播稿
-> 配音 Brief
-> TTS 或录音
-> 多版本试听
-> 剪辑时间线试放
-> 字幕校对
-> 音乐避让
-> 最终混音
```

如果你只在最后随便找一个声音读稿，视频会变得散。声音进入得越早，后期越稳。

## 1. AI 配音在视频流程中的位置

AI 视频项目可以分成几条线：

| 线 | 主要文件 | 作用 |
| --- | --- | --- |
| 叙事线 | `brief.md`、`script.md` | 决定说什么 |
| 画面线 | `storyboard.md`、`prompts/`、`outputs/` | 决定看什么 |
| 声音线 | `voice_brief.md`、`voiceover_script.md`、`audio/` | 决定怎么听 |
| 剪辑线 | `edit/`、`timeline_notes.md` | 决定成片节奏 |

配音不是单独的后期动作，它和脚本、分镜、字幕、音乐都相关。

### 1.1 配音会反过来影响脚本

书面文字看起来顺，但读出来可能很别扭。

比如：

```text
随着生成式人工智能技术在视频生产流程中的持续渗透，传统依靠人工逐条撰写提示词的方式正在被项目化工作流替代。
```

这句话作为文章还行，作为短视频口播就太重。可以改成：

```text
以前做 AI 视频，很多人是一条一条手写提示词。
现在更好的做法，是把它变成一个项目工作流。
```

口播稿要让人听懂，不是让人读着显得高级。

### 1.2 配音会影响镜头长度

一个镜头是否够长，不只看画面，也看这句话说完需要几秒。

如果某句口播需要 6 秒，而镜头只有 3 秒，你只有三个选择：

- 缩短台词。
- 延长镜头。
- 把一句话拆到两个镜头。

所以配音应该在粗剪前就进入流程，而不是成片快导出时才补。

### 1.3 配音会影响字幕

字幕不是把配音逐字贴上去那么简单。短视频字幕要考虑：

- 每行字数。
- 断句位置。
- 关键词突出。
- 画面遮挡。
- 观众阅读速度。

如果配音句子太长，字幕就会拥挤；如果配音停顿不清楚，字幕断句也会乱。

## 2. 旁白、口播、角色配音和声音克隆

先把常见类型分清楚。

| 类型 | 用途 | 典型场景 | 核心要求 |
| --- | --- | --- | --- |
| 旁白 | 解释画面和信息 | 纪录片、广告、知识短视频 | 清楚、稳定、有节奏 |
| 口播 | 像真人对观众说话 | 抖音知识视频、课程、产品介绍 | 亲近、自然、可信 |
| 角色配音 | 为角色说话 | 短剧、动画、虚拟人物 | 情绪和人设一致 |
| 解说 | 快速讲清事件 | 影视解说、案例拆解 | 信息密度高、节奏快 |
| 声音克隆 | 复用某个声音特征 | 品牌声音、授权主播、虚拟人 | 授权、稳定、合规 |

不要把它们混在一起。

如果你做的是 Ross 风格的 AI 视频工作流讲解，更接近“口播 + 旁白”。它需要可信、清楚、稍有情绪，但不需要夸张表演。

如果你做短剧，则需要“角色配音”。每个人物要有年龄、性格、情绪和说话方式。

如果你做品牌广告，则可能需要“品牌旁白”。声音要符合品牌气质，不能太随意。

## 3. 先写声音角色，而不是先选音色

新手常犯的错误是打开工具后开始试听音色：男声、女声、磁性、温柔、新闻腔、年轻感。试听一小时后，仍然不知道选哪个。

正确顺序是先写声音角色。

### 3.1 声音角色卡

```markdown
# Voice Character Card

## 声音编号

VOICE_HOST_001

## 视频类型

AI 视频工作流知识短视频。

## 观众

想学习 AI 视频但完全零基础的创作者。

## 声音气质

冷静、可信、轻微兴奋，不卖弄，不端着。

## 年龄感

25 到 35 岁。

## 语速

中等略快，但关键词处要停顿。

## 情绪范围

开头有吸引力，中段清楚解释，结尾有行动感。

## 禁止

不要播音腔，不要夸张广告腔，不要过度低沉，不要像客服机器人。
```

这张卡比“找一个好听的男声”更有用。

### 3.2 声音和画面角色要一致

如果视频里有一个年轻创作者形象，声音却像严肃新闻播报，会割裂。

你要检查：

- 声音年龄感是否和人物外观一致。
- 语气是否符合视频主题。
- 口音是否符合受众习惯。
- 情绪是否和画面节奏一致。
- 声音是否会抢走画面注意力。

## 4. 口播稿改写

AI 配音最重要的输入不是工具，而是稿件。

### 4.1 书面稿和口播稿的区别

| 维度 | 书面稿 | 口播稿 |
| --- | --- | --- |
| 句子长度 | 可以较长 | 尽量短 |
| 信息密度 | 可以复杂 | 一句一个重点 |
| 连接方式 | 依赖标点和段落 | 依赖停顿和语气 |
| 语气 | 正式 | 更像说话 |
| 观众处理方式 | 可回看 | 听过去就过去 |

口播稿要遵守一个原则：

```text
耳朵能一次听懂。
```

### 4.2 改写规则

把书面稿改成口播稿时，可以按这些规则：

1. 长句拆短。
2. 抽象词换成具体说法。
3. 一个句子只讲一个动作或判断。
4. 关键句前后加停顿。
5. 删除听起来像论文的连接词。
6. 把“本文将”改成“我们先看”。
7. 把“因此可以得出”改成“所以你会发现”。

### 4.3 改写示例

原稿：

```text
AI 视频生产的核心变化，是创作者从单条提示词生产，转向由上下文、分镜、参考资产和迭代记录共同组成的项目化流程。
```

口播稿：

```text
AI 视频最大的变化，不是多了一个新工具。
而是创作者不再只靠一条提示词。
你要有上下文，有分镜，有参考图，还要记录每一次失败。
这才叫项目化工作流。
```

这个版本更适合配音，因为它有自然停顿。

### 4.4 口播稿标注方式

你可以在稿件里标注停顿和情绪：

```text
以前做 AI 视频，很多人是一条一条手写提示词。[短停顿]
但现在，这种方式已经不够用了。[强调]
真正稳定的方法，是把提示词放进项目文件夹。[短停顿]
让 Codex 读取上下文，再生成分镜、任务卡和复盘记录。
```

不同工具对停顿标记支持不同。即使工具不识别这些标记，它也能帮助你自己评审节奏。

## 5. TTS 工作流

TTS 是 Text To Speech，也就是文字转语音。OpenAI 官方文档把文字转语音放在音频生成工作流里说明；其他平台也有类似能力。本书不绑定某一个工具，而是讲通用流程。

### 5.1 最小流程

```text
voice_character.md
-> voiceover_script.md
-> TTS 生成
-> audio/raw/
-> 试听评审
-> audio/selected/
-> 剪辑时间线
-> 混音和导出
```

### 5.2 生成前检查

生成配音前先检查：

| 项目 | 问题 |
| --- | --- |
| 用途 | 这是旁白、口播、角色配音还是广告 VO |
| 观众 | 给谁听 |
| 语速 | 快、中、慢 |
| 情绪 | 冷静、兴奋、温柔、紧张、权威 |
| 停顿 | 哪些句子需要留空 |
| 版本 | 需要几个候选 |
| 文件名 | 生成后放在哪里 |
| 授权 | 是否涉及真人声音或商业发布 |

### 5.3 生成多版本

不要只生成一个版本就开始剪辑。至少生成 2 到 3 个版本：

| 版本 | 方向 |
| --- | --- |
| v01 | 中性、清楚、稳 |
| v02 | 更有短视频节奏 |
| v03 | 更克制、更像纪录片 |

配音要在时间线上试，不要只在播放器里听。单独听很好听的声音，放进视频可能太慢、太吵、太抢戏。

## 6. 中文配音工具选择原则

中文 AI 配音工具很多，包括综合模型平台、剪辑软件内置配音、独立语音平台、开源或本地模型方向。你不需要一开始全部学。

选择中文配音工具时，看这些维度：

| 维度 | 说明 |
| --- | --- |
| 普通话自然度 | 是否像真实说话 |
| 断句能力 | 是否能处理短视频口播 |
| 情绪控制 | 是否能表达轻微兴奋、疑问、强调 |
| 字音准确 | 多音字、英文、数字是否可靠 |
| 导出格式 | 是否能导出常见音频文件 |
| 商用授权 | 是否允许商业发布 |
| 稳定性 | 同一声音多次生成是否差异过大 |
| 编辑能力 | 是否支持局部重生成或改句 |

对于零基础新手，优先选择：

```text
稳定、易导出、中文自然、授权清楚
```

而不是参数最多的工具。

## 7. 英文配音工具选择原则

英文配音更容易遇到“听起来很专业，但不适合中文观众”的问题。

如果视频主要面向中文平台，英文配音只适合：

- 国际化品牌片。
- 英文教程。
- 科技感氛围片。
- 需要海外发布的版本。

评估英文配音时，重点看：

- 发音是否清楚。
- 情绪是否符合视频。
- 是否有过度广告腔。
- 是否和字幕匹配。
- 是否有商用授权。

ElevenLabs 等工具在英文语音领域常被创作者使用，但具体声音克隆、商用发布和账号权益必须以官方页面及你的账号状态为准。不要把网上某个教程里的权限当成永久规则。

## 8. 声音克隆的边界

声音克隆是最容易出问题的部分。

声音克隆可以用于：

- 克隆你自己的声音。
- 克隆已取得明确授权的主播声音。
- 创建经过授权的品牌声音。
- 为虚拟角色建立稳定声音。

不应该用于：

- 未经同意克隆真人。
- 冒充名人、客户、同事或家人。
- 制作误导性声明。
- 绕过平台身份验证。
- 用他人声音生成商业广告。

### 8.1 你必须先问的问题

在任何声音克隆项目开始前，先回答：

| 问题 | 说明 |
| --- | --- |
| 这个声音是谁的 | 本人、授权主播、客户员工、虚拟角色 |
| 是否有书面同意 | 口头同意不够稳 |
| 使用范围是什么 | 内部样片、公开发布、商业广告 |
| 使用期限多长 | 一次性、一个项目、一年、长期 |
| 是否允许二次加工 | 改稿、改情绪、跨平台发布 |
| 是否允许客户复用 | 交付后客户能否继续使用 |
| 是否需要撤回机制 | 对方撤回授权怎么办 |

### 8.2 授权记录模板

```markdown
# Voice Consent Record

## 声音来源

姓名或角色：
联系方式：
是否本人：

## 授权用途

项目名称：
视频类型：
发布平台：
是否商业用途：

## 授权范围

允许生成内容：
允许修改稿件：
允许使用期限：
允许发布平台：

## 禁止事项

不得用于政治、医疗、金融、成人、欺诈或误导性内容。
不得冒充本人进行真实身份表达。
不得转授权给第三方。

## 确认

授权人：
日期：
项目负责人：
```

这不是法律意见，但它能提醒你：声音不是普通素材。

## 9. 配音 Brief 模板

每次配音前，先写 Brief。

```markdown
# Voiceover Brief

## 项目名称

Ross 风格 AI 视频工作流演示

## 视频目标

让零基础观众理解：AI 视频已经从手搓提示词进入项目化工作流。

## 目标观众

想学 AI 视频但不知道从哪里开始的人。

## 声音类型

知识短视频口播 + 旁白。

## 声音气质

冷静、可信、干净，有一点兴奋，但不要夸张。

## 语速

中等略快，关键句前后留短停顿。

## 情绪变化

开头：抓注意力。
中段：解释清楚。
结尾：鼓励行动。

## 禁止

不要广告叫卖，不要新闻播报腔，不要机械客服感。

## 输出

3 个候选版本，WAV 或高质量音频文件，放入 `audio/raw/`。
```

Brief 写得越清楚，后面评审越轻松。

## 10. TTS 任务卡

和视频镜头一样，配音也应该有任务卡。

```markdown
# TTS Task Card

## 文件编号

VO_MAIN_001

## 输入稿件

scripts/voiceover_v01.md

## 声音角色

VOICE_HOST_001

## 生成目的

作为 45 秒知识短视频主旁白。

## 风格

自然口播，清楚可信，中等略快。

## 停顿要求

每个核心观点后停顿 0.3 到 0.6 秒。

## 输出版本

- VO_MAIN_001_v01_neutral.wav
- VO_MAIN_001_v02_fast.wav
- VO_MAIN_001_v03_documentary.wav

## 评审重点

是否听得懂、是否自然、是否能和字幕对齐、是否适合剪辑节奏。
```

这样，配音不再是“随便导出一个音频”，而是可复盘的生产任务。

## 11. 声音版本评审

试听时不要只问“像不像真人”。要问它是否服务视频。

### 11.1 评审表

```markdown
| 文件 | 清晰度 | 自然度 | 节奏 | 情绪 | 和画面匹配 | 问题 | 结论 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| VO_MAIN_001_v01_neutral.wav | 高 | 中 | 稳 | 偏平 | 可用 | 开头不够抓人 | 备选 |
| VO_MAIN_001_v02_fast.wav | 中 | 高 | 快 | 有活力 | 适合短视频 | 第三句太急 | 局部重做 |
| VO_MAIN_001_v03_documentary.wav | 高 | 高 | 慢 | 克制 | 适合品牌片 | 抖音节奏偏慢 | 不选 |
```

### 11.2 评审优先级

1. 观众能否听懂。
2. 是否符合视频气质。
3. 是否能和镜头长度匹配。
4. 是否方便字幕断句。
5. 是否足够自然。
6. 是否足够好听。

好听排在后面。因为视频不是配音作品集，声音必须服务信息。

## 12. 音频清理和响度

AI 生成的音频也要做基本检查。

### 12.1 常见问题

| 问题 | 表现 |
| --- | --- |
| 开头有空白 | 时间线放进去后起音慢 |
| 结尾有多余呼吸 | 影响下一句或转场 |
| 音量忽高忽低 | 听感不稳定 |
| 齿音重 | s、sh、x 听起来刺耳 |
| 爆音 | 某些字突然炸开 |
| 背景噪声 | 声音底噪明显 |
| 断句奇怪 | 语义被拆错 |
| 数字读错 | 年份、金额、型号错误 |

### 12.2 基本处理

你可以在剪辑软件或音频软件里做：

- 裁掉开头和结尾多余空白。
- 调整整体音量。
- 用降噪处理轻微底噪。
- 对刺耳齿音做轻微处理。
- 让主配音音量保持稳定。
- 和音乐一起试听，避免人声被盖住。

不建议新手一开始追求复杂混音。先做到：

```text
听得清楚
不刺耳
不忽大忽小
和音乐不打架
```

## 13. 文件命名与入库

声音素材也要进入项目目录。

推荐目录：

```text
project/
├── scripts/
│   ├── script_v01.md
│   └── voiceover_v01.md
├── audio/
│   ├── raw/
│   ├── selected/
│   ├── rejected/
│   └── masters/
└── edit/
```

命名规则：

```text
VO_用途_镜头或段落_版本_特点.ext
```

示例：

```text
VO_MAIN_ALL_v01_neutral.wav
VO_MAIN_ALL_v02_fast.wav
VO_S01_HOOK_v01_energetic.wav
VO_S05_END_v02_soft.wav
```

### 13.1 入库说明

```markdown
| 文件 | 用途 | 来源 | 是否授权 | 是否进入剪辑 | 备注 |
| --- | --- | --- | --- | --- | --- |
| VO_MAIN_ALL_v02_fast.wav | 主旁白 | TTS 工具 | 按账号确认 | 是 | 第三句需降速 |
| VO_MAIN_ALL_v01_neutral.wav | 备选 | TTS 工具 | 按账号确认 | 否 | 过平 |
```

不要把音频一直留在下载目录。否则等你做第二条视频时，很快找不到哪个版本是最终版。

## 14. 和剪辑软件配合

配音进入剪辑软件后，你会发现一些问题：

- 某句太长，镜头装不下。
- 停顿太短，字幕来不及读。
- 情绪太平，开头抓不住人。
- 音乐太大，人声被盖住。
- AI 画面节奏慢，配音节奏快。

### 14.1 粗剪阶段

先把配音放到时间线上，按声音分段切画面。

常见做法：

```text
先定配音主线
-> 按句子放镜头
-> 发现画面不够就补 B-roll
-> 发现句子太长就改稿或重配
```

知识短视频通常适合先定声音，再放画面。因为信息线最重要。

### 14.2 精剪阶段

精剪时检查：

- 每句配音是否有对应画面。
- 关键句是否有视觉强调。
- 字幕是否和声音同步。
- 音乐是否在说话时降低。
- 转场音效是否抢人声。

### 14.3 不要把配音锁死

如果你发现画面和声音对不上，不要只调剪辑。有时最好的办法是回到口播稿，删掉一句废话。

后期不是把所有素材硬塞到一起，而是持续删减。

## 15. Ross 风格案例中的配音推断

Ross 那条“全程用 Codex 和 LibTV 完成”的视频，页面信息和关键画面已经在项目里核验记录。我们不能断言它真实使用了哪一个配音工具，但可以从类似视频形态推断它需要一条清楚的声音主线。

这类视频通常有：

- 开头一句强钩子。
- 中段解释工具链。
- 穿插虚拟形象或画面展示。
- 用字幕强化关键词。
- 结尾引导观众理解趋势。

配音策略可以这样设计：

```text
00:00-00:03  钩子：手搓提示词的时代过去了
00:03-00:12  解释：现在要用项目化工作流
00:12-00:30  展示：Codex 负责上下文和任务拆解
00:30-00:50  展示：LibTV 或同类工具负责生成素材
00:50-01:05  总结：模型要组合使用
01:05-01:12  结尾：创作者要升级工作方式
```

对应配音要求：

| 段落 | 声音要求 |
| --- | --- |
| 开头 | 有冲击，但不要喊 |
| 工具解释 | 清楚、可信 |
| 成本/效率 | 稍微加重 |
| 模型选择 | 语速放稳 |
| 结尾 | 有判断感 |

如果你想复刻类似风格，先不要问“用哪个音色”。先写这张声音结构表。


这类说明不一定能替代合同，但能减少沟通误会。

## 17. 常见错误

### 错误 1：脚本没改成口播稿

表现：听起来像读文章，句子太长。

修正：长句拆短，一句一个重点。

### 错误 2：只选好听的音色

表现：声音单独听不错，放进视频不合适。

修正：按视频目标和观众评审声音。

### 错误 3：声音和字幕不同步

表现：字幕提前或滞后，观众读着累。

修正：先按配音断句，再做字幕。

### 错误 4：音乐盖住人声

表现：画面有情绪，但信息听不清。

修正：降低音乐，尤其在人声频段留空间。

### 错误 5：未经授权使用声音克隆

表现：项目看似高级，但存在重大风险。

修正：没有明确授权就不要克隆真人声音。

### 错误 6：不保存失败版本

表现：后面不知道为什么最终选了这个声音。

修正：把 raw、selected、rejected 分目录保存。

## 18. 故障排查

### 配音太像机器人

处理：

- 改口播稿，减少书面词。
- 增加短句和自然停顿。
- 换更适合口播的声音角色。
- 生成多个速度版本。
- 在剪辑里加轻微呼吸空间，但不要过度装饰。

### 配音太慢

处理：

- 删除无用连接词。
- 把一句长解释拆成两句短判断。
- 重新生成略快版本。
- 剪辑里缩短无声段。

### 配音太快

处理：

- 给关键句加入停顿提示。
- 减少每段信息量。
- 把字幕断成更短的行。
- 对复杂概念加画面解释，不全靠说。

### 字幕跟不上声音

处理：

- 不要逐字铺满屏幕。
- 每行保留一个语义单元。
- 关键词可以单独做强调。
- 必要时重配一句更短的版本。

### 声音和人物不匹配

处理：

- 回到 Voice Character Card。
- 检查年龄感、气质、语速。
- 如果虚拟人物偏年轻，不要用过度成熟的播音腔。
- 如果品牌片偏稳重，不要用太活泼的短视频腔。

### 声音克隆结果不稳定

处理：

- 检查样本质量。
- 不要混入多个场景、多个设备、多个情绪差异太大的样本。
- 缩小使用范围，先做短句测试。
- 重新确认授权和平台要求。

## 20. 从脚本到配音的完整案例

主题：

```text
AI 视频不是一句提示词，而是一套项目化工作流。
```

### 20.1 原始脚本

```text
传统的 AI 视频制作方式，往往依赖创作者手动编写大量提示词，并不断试错。随着 Codex、LibTV 以及多种生成模型的结合，创作者可以把脚本、分镜、风格和提示词都放进项目文件夹，由 Agent 读取上下文并持续迭代。
```

### 20.2 口播稿

```text
以前做 AI 视频，很多人靠手写提示词。
一条不行，就再改一条。
但这样很难稳定。

现在更好的方式，是把视频当成一个项目。
脚本、分镜、风格、参考图、失败记录，都放进文件夹。
然后让 Codex 读取上下文，帮你拆镜头、写任务卡、复盘失败。

这时候，LibTV 或同类工具负责生成素材。
Codex 负责管理整个创作流程。
你做的就不再是单条提示词，而是一套可以复用的工作流。
```

### 20.3 配音任务

```markdown
声音：年轻男性，可信，轻微兴奋。
语速：中等略快。
风格：知识短视频口播。
版本：生成 3 个。
重点：第一句要抓人，中段解释清楚，结尾有判断感。
```

### 20.4 评审结论

```markdown
| 版本 | 结论 |
| --- | --- |
| v01 | 太平，适合课程，不适合短视频 |
| v02 | 节奏好，但第二段太快，需要局部重做 |
| v03 | 声音质感好，但年龄感偏成熟 |
```

最终选择 v02，重做第二段。

这就是可复盘配音流程。

## 21. 本卷总结

AI 配音不是按一下“生成语音”。

稳定流程是：

```text
先写声音角色
-> 改口播稿
-> 写配音 Brief
-> 生成多版本
-> 在时间线上试听
-> 做字幕和音乐避让
-> 记录授权和版本
-> 入库交付
```

新手最该记住的是：声音服务信息。听得清楚，比声音炫酷更重要。节奏合适，比声音像真人更重要。授权明确，比临时效果更重要。

当你能把声音当成项目资产管理，视频会立刻变得更专业。

第 12 卷会继续讲音乐。音乐不是背景填空，而是情绪和节奏的骨架。它要和人声配合，而不是抢走人声。

## 22. 课后练习

### 练习 1：声音角色卡

为一条 30 秒 AI 视频写一张 Voice Character Card。

必须包含：

- 视频类型。
- 目标观众。
- 声音气质。
- 语速。
- 情绪范围。
- 禁止事项。

### 练习 2：口播稿改写

把下面这句话改成适合短视频朗读的口播稿：

```text
AI 视频生产已经逐渐从单点工具调用，转向由脚本、分镜、参考资产、生成记录和后期流程共同组成的系统化创作方式。
```

要求：

- 至少拆成 3 句。
- 每句不超过 20 个字左右。
- 听起来像人说话。

### 练习 3：TTS 任务卡

为你自己的 15 秒试验片写一张 TTS Task Card。

必须包含：

- 输入稿件。
- 声音角色。
- 生成目的。
- 风格。
- 输出版本。
- 评审重点。

### 练习 4：授权检查

假设你想克隆一个朋友的声音给广告片配音，写一张授权检查表。

必须回答：

- 是否本人同意。
- 是否商业用途。
- 是否允许公开发布。
- 是否允许客户复用。
- 是否有使用期限。
