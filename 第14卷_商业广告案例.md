# 第 14 卷：商业广告案例

## 学习目标

学完本卷后，你应该能做到：

- 理解商业广告项目和个人练习作品的区别。
- 会做客户需求访谈，不再一上来就问“想要什么风格”。
- 能把客户需求整理成 Brief、脚本、分镜、生成任务和交付清单。
- 知道报价不是随口报一个数字，而是由范围、修改次数、交付文件和授权边界决定。
- 能用 AI 视频工具制作产品展示、品牌氛围、口播说明和发布素材。
- 会管理客户修改，避免项目变成无限返工。
- 能准备商业交付文件包和复盘记录。
- 知道广告内容、AI 素材、声音、音乐、字体、商标和产品表现都需要授权和合规意识。

本卷用一个完整虚构案例讲流程。案例品牌、产品和客户均为教学设定，不代表真实商业合作。

## 本卷导读：商业广告不是炫技样片

很多人学 AI 视频后，第一反应是：“我能不能接广告单？”

答案不是看你会不会生成一段漂亮画面，而是看你能不能稳定交付。

商业广告项目最关心的不是：

```text
画面是不是最炫
模型是不是最新
Prompt 是不是很复杂
```

而是：

```text
客户要卖什么
观众是谁
卖点是否清楚
品牌是否准确
交付是否准时
修改是否可控
授权是否清楚
最终文件是否能发布
```

AI 让广告制作变快，但商业责任没有消失。你仍然要确认产品信息、客户资质、素材来源、平台规则和交付边界。

本卷案例：

```text
客户：本地精品咖啡品牌「晨岸咖啡」
产品：夏季新品冷萃咖啡
目标：制作 30 秒竖屏短视频，用于抖音、小红书和视频号发布
风格：清爽、真实、都市早晨、轻微高级感
预算：小型商家试水预算
交付：1 条 30 秒成片 + 1 张封面 + 1 份发布文案 + 可复盘项目文件
```

## 1. 商业广告项目的完整流程

商业广告项目不要从生成画面开始，而要从需求确认开始。

推荐流程：

```text
客户需求访谈
-> 项目 Brief
-> 报价和范围确认
-> 脚本
-> 分镜
-> 风格参考
-> AI 素材生成
-> 配音和音乐
-> 剪辑和字幕
-> 客户初审
-> 修改
-> 终审
-> 交付和归档
-> 复盘
```

每一步都有一个产出文件：

| 阶段 | 文件 |
| --- | --- |
| 需求 | `client_interview.md` |
| Brief | `ad_brief.md` |
| 报价 | `quote_scope.md` |
| 脚本 | `script_v01.md` |
| 分镜 | `storyboard_ad_v01.md` |
| 风格 | `brand_style_guide.md` |
| 生成 | `generation_tasks.md` |
| 修改 | `revision_log.md` |
| 交付 | `delivery_checklist.md` |
| 复盘 | `project_review.md` |

商业项目的专业感，很多时候来自这些文件，而不是来自某个神奇提示词。

## 2. 客户需求访谈

客户常常不会用专业语言表达需求。他可能说：

```text
想要高级一点、年轻一点、有质感一点，最好能火。
```

这句话不能直接变成 Prompt。你要继续问。

### 2.1 访谈问题

```markdown

## 产品信息

核心卖点：
价格区间：
目标人群：
购买渠道：

## 品牌信息

品牌调性：
品牌色：
禁用颜色：

已有字体：

## 视频目标

让观众知道什么：
让观众感受到什么：
希望观众做什么：

## 交付要求

视频时长：
画幅：
是否需要配音：
是否需要字幕：
是否需要封面：
是否需要多平台版本：

## 限制和风险

是否有医疗、金融、功效或夸张表述：

```

### 2.2 晨岸咖啡访谈摘要

```markdown
产品：夏季冷萃咖啡
卖点：低酸、清爽、适合早晨通勤和午后提神
目标人群：22 到 35 岁城市上班族
发布平台：抖音、小红书、视频号
品牌调性：干净、真实、轻微高级，不做夸张促销
必须出现：产品杯、门店、夏季新品、到店或外卖购买
不要出现：夸张功效、医疗暗示、虚假销量
```

这个摘要比“做一个高级咖啡广告”更可执行。

## 3. 广告 Brief

Brief 是把客户语言翻译成制作语言。

```markdown
# AI 商业广告 Brief

## 项目名称

晨岸咖啡夏季冷萃短视频广告

## 视频目标

让本地年轻上班族知道晨岸咖啡推出夏季冷萃新品，并产生到店或外卖下单兴趣。

## 核心信息

夏天早晨，一杯清爽低酸的冷萃咖啡，可以让通勤更轻一点。

## 目标观众

22 到 35 岁城市上班族，习惯短视频平台获取消费信息。

## 视频形式

30 秒竖屏广告，真实生活方式 + 产品展示 + 字幕卖点。

## 风格

清晨自然光、城市街道、干净咖啡店、真实手部动作、低饱和、清爽。

## 禁止

不要夸张功效，不要虚假销量，不要医疗或健康承诺，不要使用未授权品牌元素。

## 交付

- 30 秒 9:16 MP4。
- 1 张封面图。
- 1 份发布文案。
- 字幕文案。
- 项目复盘和素材来源记录。
```

Brief 不需要很长，但必须能回答：

```text
谁看
看完知道什么
看完做什么
用什么风格表达
不能说什么
交付什么
```

## 4. 报价边界和交付范围

报价不是本书重点，也不应该给固定价格。不同城市、行业、客户、周期和能力差异都很大。你需要学的是报价结构。

### 4.1 报价由什么组成

| 项目 | 说明 |
| --- | --- |
| 策划 | 需求访谈、Brief、脚本、分镜 |
| AI 生成 | 图片、视频、重试次数、模型成本 |
| 配音音乐 | TTS、音乐、音效、授权确认 |
| 剪辑后期 | 字幕、混音、调色、导出 |
| 修改 | 包含几轮修改，每轮修改范围 |
| 交付 | 成片、封面、文案、源文件或项目文件 |
| 授权 | 素材、声音、音乐、字体、二次使用范围 |

### 4.2 范围说明模板

```markdown
# 报价范围说明

## 包含

- 1 次需求访谈。
- 1 版广告 Brief。
- 1 版 30 秒脚本。
- 1 版 6 镜头分镜。
- AI 视频素材生成与筛选。
- 1 条 30 秒竖屏成片。
- 1 张封面图。
- 1 份发布文案。
- 2 轮小修改。

## 不包含

- 线下拍摄。
- 明星或真人授权。
- 客户品牌 Logo 重设计。
- 广告投放账户操作。
- 超出约定平台的多版本适配。
- 未约定的长期素材复用授权。

## 修改规则

小修改：字幕、节奏、局部镜头替换、音乐调整。
大修改：重新改脚本、改整体风格、改产品卖点、重做大部分镜头。
```

范围越清楚，后面越不容易争。

## 5. 脚本和卖点结构

商业广告脚本要服务卖点。

### 5.1 30 秒广告结构

| 时间 | 内容 | 目标 |
| --- | --- | --- |
| 0-3 秒 | 场景钩子 | 让观众停下 |
| 3-8 秒 | 痛点或生活情境 | 让观众代入 |
| 8-18 秒 | 产品出现和卖点 | 说明为什么买 |
| 18-25 秒 | 使用场景 | 让观众想象自己使用 |
| 25-30 秒 | 行动提示 | 到店、下单、关注 |

### 5.2 晨岸咖啡脚本

```markdown
# script_v01

## 0-3 秒

字幕：夏天早晨，别让第一口咖啡太沉重。
画面：清晨街道，阳光照进咖啡店门口。

## 3-8 秒

字幕：低酸、清爽、适合通勤路上慢慢喝。
画面：店员把冷萃倒入透明杯，冰块轻轻晃动。

## 8-18 秒

字幕：晨岸夏季冷萃新品。
画面：产品杯特写，杯身水珠，桌面自然光。

## 18-25 秒

字幕：早晨到店，午后外卖，都刚刚好。
画面：年轻上班族拿着咖啡走出店门。

## 25-30 秒

字幕：这个夏天，从一杯清爽冷萃开始。
画面：产品杯和门店 Logo，结尾封面感构图。
```

广告脚本不需要塞满信息。一个广告只抓一到三个核心卖点。

## 6. 商业广告分镜

分镜要把脚本变成生成任务。

```markdown
| 镜头 | 时长 | 目的 | 画面 | AI 生成方式 | 注意 |
| --- | --- | --- | --- | --- | --- |
| S01 | 3s | 建立夏日清晨 | 咖啡店门口，自然光 | 文生视频或图生视频 | 不要真实品牌误用 |
| S02 | 5s | 展示制作 | 冷萃倒入透明杯，冰块 | 图生视频 | 手部动作简单 |
| S03 | 5s | 产品特写 | 杯身水珠，桌面光影 | 图生视频 | 产品外观需客户确认 |
| S04 | 7s | 使用场景 | 上班族拿杯走出门店 | 文生视频 | 人物不必露脸 |
| S05 | 5s | 消费联想 | 桌面、电脑、咖啡 | 文生视频 | 真实生活方式 |
| S06 | 5s | 收束行动 | 产品杯 + 门店氛围 | 图生视频 | 留 Logo 和字幕空间 |
```

商业广告分镜要有“目的”列。没有目的的镜头，即使好看也可能被删。

## 7. 产品镜头和品牌风格

商业项目最容易出问题的是产品不准。

### 7.1 产品镜头原则

- 真实产品以客户官方素材为准。
- AI 生成产品图必须让客户确认。
- 不要让 AI 随机改变包装、颜色、Logo、容量、价格。
- 重要产品镜头优先用客户提供图片做参考。
- 如果无法稳定生成准确产品，就使用真实拍摄素材或设计图。

### 7.2 品牌风格表

```markdown
# Brand Style Guide

## 品牌气质

清爽、真实、城市早晨、轻微高级。

## 色彩

主色：浅咖啡色、奶白、低饱和绿色。
避免：高饱和霓虹色、重金属色、过度赛博朋克。

## 光线

自然晨光、柔和侧光、干净阴影。

## 字幕

简洁中文，无复杂花字。

## 音乐

轻快、清爽、无歌词背景音乐。

## 禁止

不要夸张促销感，不要虚假销量，不要医疗健康暗示。
```

风格表是给 Codex、生成工具、剪辑和客户共同看的。

## 8. AI 生成任务拆解

不要把广告写成一条总 Prompt。

每个镜头单独生成：

```markdown
# S02 生成任务卡

## 镜头目的

展示冷萃咖啡的清爽感。

## 画面

近景，透明杯中冰块，深色冷萃咖啡缓慢倒入，杯壁有细小水珠。

## 风格

清晨自然光，低饱和，真实商业广告质感，干净桌面。

## 运镜

固定机位，轻微慢推。

## 时长

4 到 5 秒。

## 禁止

不要可读错误文字，不要其他品牌 Logo，不要夸张奶油，不要多余手指。

## 参考

客户产品图：references/product/cold_brew_v01.png
```

### 8.1 生成评审表

```markdown
| 镜头 | 文件 | 是否服务卖点 | 产品是否准确 | 风格是否一致 | 问题 | 结论 |
| --- | --- | --- | --- | --- | --- | --- |
| S02 | S02_v01.mp4 | 是 | 部分 | 是 | 杯身形状不同 | 备选 |
| S02 | S02_v02.mp4 | 是 | 是 | 是 | 结尾有轻微抖动 | 裁切使用 |
| S02 | S02_v03.mp4 | 否 | 否 | 否 | 太像酒吧广告 | 不选 |
```

商业项目里，“产品是否准确”比“画面是否惊艳”更重要。

## 9. 配音、音乐、字幕和剪辑

广告声音要配合品牌。

### 9.1 配音策略

晨岸咖啡适合：

```text
温和、清楚、轻松、年轻但不过度兴奋
```

不适合：

```text
叫卖腔、新闻腔、过度低沉、夸张网红腔
```

### 9.2 音乐策略

音乐 Brief：

```markdown
清爽、轻快、无歌词、早晨城市感、轻鼓点、不要夜店感。
```

### 9.3 字幕策略

字幕只突出卖点：

- 夏季新品。
- 低酸清爽。
- 通勤和午后都适合。
- 到店或外卖。

不要把所有配音逐字铺满屏幕。

## 10. 客户修改管理

客户修改不可怕，可怕的是没有边界。

### 10.1 修改记录

```markdown
# Revision Log

| 日期 | 版本 | 客户意见 | 类型 | 处理方式 | 是否影响费用 |
| --- | --- | --- | --- | --- | --- |
| 07-10 | v01 | 字幕“低酸”改成“清爽低酸” | 小修改 | 修改字幕 | 否 |
| 07-10 | v01 | 第 3 个镜头产品杯不准确 | 质量问题 | 重生成 S03 | 否 |
| 07-11 | v02 | 想改成夜晚酒吧风 | 大修改 | 需重新确认 Brief | 是 |
```

### 10.2 修改分类

| 类型 | 例子 |
| --- | --- |
| 小修改 | 字幕、剪辑节奏、个别镜头替换 |
| 质量修正 | 产品错误、错别字、Logo 错误 |
| 大方向修改 | 改风格、改脚本、改受众、改卖点 |

质量修正应该负责处理；大方向修改要重新确认范围。

## 12. 合规和授权提醒

广告项目尤其要注意真实和授权。

### 12.1 常见风险

| 风险 | 说明 |
| --- | --- |
| 产品效果夸大 | 没有依据不要写功效承诺 |
| 虚假销量 | 不要写客户无法证明的销量或排名 |
| 未授权音乐 | AI 音乐也要记录来源和条款 |
| 未授权声音 | 声音克隆必须有明确授权 |
| 商标误用 | AI 画面不要出现其他品牌 |
| 版权素材 | 参考图、字体、音效都要有来源 |
| AI 生成误导 | 不要让观众误解为真实拍摄事实 |

### 12.2 广告文案自检

```markdown
| 文案 | 是否有依据 | 风险 |
| --- | --- | --- |
| 夏季新品 | 客户可确认 | 低 |
| 低酸清爽 | 需客户确认产品描述 | 中 |
| 全城第一 | 需要客观证明 | 高 |
| 喝了立刻提神一整天 | 可能夸大 | 高 |
```

本书不是法律意见。真实商业广告需要按当地法律、平台规则和客户行业要求确认。

## 13. 不同广告类型怎么拆

商业广告不是只有一种。你接到的项目不同，流程重点也不同。

| 类型 | 重点 | AI 视频注意点 |
| --- | --- | --- |
| 门店探店广告 | 空间、氛围、到店路径 | 真实门店最好用客户素材，AI 只补氛围镜头 |
| 产品展示广告 | 产品外观、卖点、使用场景 | 产品形状和包装必须准确 |
| 品牌形象片 | 情绪、价值观、品牌气质 | 风格统一比信息密度更重要 |
| 活动促销广告 | 时间、优惠、参与方式 | 日期、价格、活动规则必须客户确认 |
| 招商加盟广告 | 信任、实力、流程 | 不要写无法证明的收益承诺 |
| 课程知识广告 | 痛点、方法、转化 | 不要夸大结果，不要承诺学习效果 |

新手最适合先做：

```text
产品展示广告
门店氛围广告
个人 IP 知识广告
```

这些项目的结构相对清楚，风险比复杂招商、医疗、金融、教育承诺类广告低。遇到高监管行业时，要格外谨慎，必要时让客户提供审核过的文案。

### 13.1 产品展示广告

产品展示广告的核心是：

```text
看见产品
理解卖点
想象使用场景
知道下一步动作
```

脚本不要复杂。比如咖啡、香薰、文创、餐饮新品，都可以按这个结构：

```text
场景钩子
-> 产品出现
-> 1 到 2 个卖点
-> 使用场景
-> 行动提示
```

### 13.2 门店氛围广告

门店广告最怕 AI 生成一个“很像但不是这家店”的空间。客户如果已经有门店照片，优先使用真实照片或视频。AI 可以补：

- 清晨城市氛围。
- 顾客生活方式镜头。
- 产品特写氛围。
- 字幕和封面背景。

不要把 AI 生成门店当成真实门店展示，除非已经明确标注或客户确认用途。

### 13.3 活动广告

活动广告最容易出错的是时间和规则。

必须确认：

- 活动开始和结束日期。
- 优惠条件。
- 是否限门店。
- 是否限数量。
- 是否需要会员。
- 是否有平台限制。

AI 可以帮你做画面，但不能替客户编活动规则。

## 14. 客户沟通话术

商业项目里，沟通能力比工具熟练度更重要。

### 14.1 需求不清时

客户说：

```text
你帮我做得高级一点。
```

可以回答：

```text
可以。为了避免我理解错，我们先把“高级”拆成几个可执行方向：色调更克制、镜头更慢、字幕更简洁、音乐更轻，还是产品特写更精致？我可以先给你 2 个风格方向确认。
```

### 14.2 客户临时加需求时

客户说：

```text
顺便再帮我做一个横屏版本和三张海报吧。
```

可以回答：

```text
可以做。我先帮你拆一下，这属于新增交付物，不是原来 30 秒竖屏视频里的小修改。我可以给你补一版新增范围和时间，你确认后我再排进去。
```

### 14.3 客户反复改风格时

客户说：

```text
还是感觉不对，再换一种风格。
```

可以回答：

```text
我们先回到 Brief 里确认：目标观众、核心卖点和品牌气质有没有变化？如果这些没有变，我建议只在色彩、音乐和字幕上调整；如果整体方向变了，就需要按新版 Brief 重做。
```

### 14.4 客户要求夸张文案时

客户说：

```text
写得猛一点，就说全城最好。
```

可以回答：

```text
如果有客观证明，我们可以把证明依据放进文案。如果没有，我建议换成更稳的表达，比如“适合夏天通勤的一杯清爽冷萃”。这样更安全，也更可信。
```

这些话术的目的不是拒绝客户，而是把模糊需求变成可执行范围。

## 15. 商业广告验收标准

最终成片至少要过这张表：

```markdown
| 验收项 | 是否通过 | 备注 |
| --- | --- | --- |
| 视频目标清楚 |  | 观众是否知道广告在卖什么 |
| 核心卖点出现 |  | 卖点是否准确 |
| 产品外观正确 |  | 包装、颜色、形状是否客户确认 |
| 品牌风格一致 |  | 色彩、字体、音乐是否统一 |
| 字幕无错别字 |  | 日期、价格、品牌名重点检查 |
| 配音音乐授权记录 |  | AI 生成也要记录来源 |
| 画幅和分辨率正确 |  | 是否适合发布平台 |
| 修改意见已处理 |  | revision_log 是否更新 |
| 客户确认交付 |  | 是否有确认记录 |
```

商业项目的终点不是你导出视频，而是客户确认交付范围内的文件已经收到并可用。

### 15.1 新手首单建议

如果你第一次做商业广告，不建议一上来接复杂项目。更稳的首单范围是：

```text
15 到 30 秒
一个产品
一个核心卖点
一个发布平台
一条成片
一张封面
一份发布文案
一到两轮小修改
```

不要同时承诺多平台、多版本、多语言、长期投放、复杂人物剧情和大量源文件交付。你可以把能力做深，但范围要先做小。第一单的目标不是证明你什么都能做，而是证明你能按约定交付。

首单结束后一定要复盘三件事：客户最在意什么、哪一步返工最多、哪些交付物下次要提前说明。把这些写下来，你的第二单会比第一单稳很多。

同时也要记住：不会做的内容要提前说清楚。商业合作最怕临时逞强，最后用加班和返工补漏洞。

边界清楚，客户反而更容易信任你。

这也是商业项目的底层专业感。

也是复购的起点。

```markdown

```

复盘不是责备谁，而是让下一单更稳。

## 17. 常见错误

### 错误 1：没做需求访谈就报价

修正：先问清目标、平台、时长、交付、素材和修改范围。

### 错误 2：广告只追求画面高级

修正：每个镜头都要服务卖点。

### 错误 3：产品不准还强行交付

修正：商业产品镜头必须客户确认。

### 错误 4：修改没有记录

修正：所有客户意见写进 `revision_log.md`。

### 错误 5：授权不清

修正：配音、音乐、字体、图片、Logo、产品素材都要记录来源。

## 18. 故障排查

### 客户说“感觉不高级”

处理：

- 让客户指出具体镜头。
- 对照 Brief 看“高级”指色彩、光线、镜头还是字体。
- 给 2 到 3 个参考方向让客户选择。
- 不要只靠自己猜。

### 客户临时改卖点

处理：

- 判断是否影响脚本和分镜。
- 如果只改字幕，可以按小修改处理。
- 如果影响整体结构，要重新确认范围和周期。

### 产品镜头总是不准

处理：

- 使用客户真实产品图。
- 减少 AI 改动。
- 改成真实照片 + 动态字幕或轻微动效。
- 必要时建议客户补拍产品素材。

### 客户要很多平台版本

处理：

- 先确认每个平台画幅、时长和封面要求。
- 多平台版本应写进报价范围。
- 不要默认所有平台适配都免费。

## 20. 本卷总结

商业广告的核心不是“我能生成漂亮画面”，而是：

```text
我能理解客户目标
-> 写清范围
-> 做出服务卖点的脚本和分镜
-> 稳定生成素材
-> 控制修改
-> 合规交付
-> 复盘下一次更好
```

AI 能提高效率，但商业项目仍然需要沟通、判断、记录和责任感。

当你能把广告项目拆成文件、任务、版本和交付清单，你就不再是“会玩工具的人”，而是在建立可交付能力。

## 21. 课后练习

### 练习 1：客户访谈

任选一个本地商家，假设它要做 30 秒广告，写一份客户需求访谈表。

### 练习 2：广告 Brief

根据访谈表写一份广告 Brief，必须包含：

- 视频目标。
- 目标观众。
- 核心信息。
- 风格。
- 禁止内容。
- 交付范围。

### 练习 3：分镜表

写 6 个镜头，每个镜头必须有目的。

### 练习 4：修改记录

模拟客户提出 5 条意见，把它们分成小修改、质量修正和大方向修改。
