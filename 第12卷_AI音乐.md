# 第 12 卷：AI 音乐

## 学习目标

学完本卷后，你应该能做到：

- 理解音乐在 AI 视频中承担的情绪、节奏和结构作用。
- 区分背景音乐、主题曲、片头音效、转场音效、氛围声和 Foley。
- 为短视频、广告片、短剧写音乐 Brief。
- 用情绪、速度、段落、乐器和使用场景写 AI 音乐提示词。
- 判断一首 AI 音乐能不能进入剪辑，而不是只判断它好不好听。
- 让音乐避让人声，避免盖住配音和字幕节奏。
- 建立音乐素材命名、版本评审、授权记录和交付检查。
- 理解 AI 音乐商用、版权、上传素材和客户交付的风险边界。

本卷不教你“按哪个按钮生成爆款神曲”。它教你把音乐变成视频项目资产。

## 本卷导读：音乐是情绪和节奏的骨架

如果说第 11 卷的配音是信息时间轴，那么音乐就是情绪时间轴。

同一个画面，配不同音乐，观众会得到完全不同的感受：

| 画面 | 音乐 | 观众感受 |
| --- | --- | --- |
| 创作者看着电脑屏幕 | 极简电子氛围 | 科技、专注 |
| 创作者看着电脑屏幕 | 紧张鼓点 | 危机、悬念 |
| 创作者看着电脑屏幕 | 温暖钢琴 | 个人成长、治愈 |
| 创作者看着电脑屏幕 | 快速流行节奏 | 短视频效率感 |

所以音乐不是“背景填空”。音乐会告诉观众：

```text
这段应该紧张
这段应该相信
这段应该兴奋
这段应该停下来想一想
```

新手做 AI 视频时，常见问题是画面和音乐各自都不错，但放在一起很奇怪。原因通常是没有先写音乐 Brief。

## 1. AI 音乐在视频项目中的位置

AI 音乐应该在粗剪前就参与，而不是最后临时找一首歌。

推荐流程：

```text
brief.md
-> script.md
-> storyboard.md
-> voiceover_script.md
-> music_brief.md
-> AI 音乐生成或选曲
-> music_review.md
-> 剪辑时间线试放
-> 人声避让和音量调整
-> 授权记录
-> 最终导出
```

音乐会影响：

- 剪辑速度。
- 镜头切点。
- 字幕出现节奏。
- 观众情绪走向。
- 结尾是否有记忆点。

你不能只问“这首音乐好不好听”。你要问：

```text
这首音乐是否服务这个视频？
```

## 2. 背景音乐、主题曲、音效和氛围声

先把声音素材类型分清楚。

| 类型 | 作用 | 示例 |
| --- | --- | --- |
| 背景音乐 | 托住情绪和节奏 | 知识视频底乐、广告片氛围 |
| 主题曲 | 让视频有记忆点 | 品牌片主旋律、短剧片尾歌 |
| 片头音效 | 抓注意力 | 开场 whoosh、点击声 |
| 转场音效 | 强化切换 | swish、hit、riser |
| 氛围声 | 增加空间真实感 | 城市夜景、办公室底噪 |
| Foley | 模拟具体动作声音 | 键盘、脚步、纸张 |

AI 音乐工具主要解决“背景音乐”和“主题曲”。音效和氛围声也可以由 AI 或素材库补充，但它们的使用逻辑不同。

### 2.1 背景音乐

背景音乐的任务是托住视频，不是抢走视频。

适合：

- 知识短视频。
- 产品展示。
- 工作流演示。
- 纪录片式旁白。

要求：

- 不要旋律过于抢耳。
- 不要人声太突出。
- 不要频繁大起大落。
- 要方便剪辑循环或裁切。

### 2.2 主题曲

主题曲更适合：

- 系列短剧。
- 品牌栏目。
- 片尾记忆点。
- 需要传播的音乐化内容。

主题曲通常要考虑歌词、旋律、版权、平台分发和观众记忆。它比背景音乐复杂。

### 2.3 音效和氛围声

音效不一定要多。短视频里过度音效会显得廉价。

常用音效：

- 开场 hit。
- 转场 whoosh。
- 关键词弹出声。
- 按钮点击声。
- 成片展示时的轻微冲击声。

使用原则：

```text
每个音效都要有动作依据。
```

画面没有切换，就不要乱加转场音。

## 3. 音乐 Brief

生成音乐之前先写 Brief。

```markdown
# Music Brief

## 项目名称

Ross 风格 AI 视频工作流演示

## 视频类型

45 到 60 秒知识短视频。

## 目标观众

想学习 AI 视频但不懂工具链的新手。

## 核心情绪

科技感、效率感、轻微兴奋、可信。

## 不要的情绪

不要恐怖、不要史诗战争、不要过度煽情、不要夜店感。

## 音乐类型

极简电子 + 轻微鼓点 + 现代科技感。

## 速度

中速，适合剪 1 到 3 秒一个信息点。

## 结构

开头 3 秒抓注意力，中段稳定推进，结尾轻微上扬。

## 人声

不要主唱，不要明显歌词，避免盖住旁白。

## 输出

2 到 4 个候选版本，30 秒或 60 秒。
```

Brief 的作用是先把判断标准写清楚。没有 Brief，生成 20 首歌也只是碰运气。

## 4. AI 音乐 Prompt

AI 音乐提示词通常要描述：

- 情绪。
- 类型。
- 节奏。
- 乐器。
- 使用场景。
- 是否有人声。
- 段落结构。
- 禁止项。

### 4.1 背景音乐 Prompt

```text
Create a 60-second instrumental background track for a vertical short video about AI video workflow.
Mood: focused, modern, slightly exciting, trustworthy.
Style: minimal electronic, light percussion, soft synth pulses, clean technology feeling.
Tempo: medium, suitable for narration and quick editing.
Structure: short attention-grabbing intro, steady middle section, gentle lift at the end.
No vocals, no lyrics, no heavy bass, no dramatic trailer sound.
```

即使你用中文工具，也可以按这个结构写中文：

```text
为一条 60 秒竖屏知识短视频生成背景音乐。
主题是 AI 视频工作流。
情绪：专注、现代、轻微兴奋、可信。
风格：极简电子、轻鼓点、柔和合成器、干净科技感。
速度：中速，适合旁白和快速剪辑。
结构：开头 3 秒抓注意力，中段稳定推进，结尾轻微上扬。
不要人声，不要歌词，不要重低音，不要史诗预告片感。
```

### 4.2 主题曲 Prompt

主题曲可以更强调旋律和记忆点：

```text
Create a short theme song for a recurring AI creator series.
Mood: optimistic, curious, productive.
Style: bright electronic pop, clean synth melody, light drums.
Length: 30 seconds.
Structure: 4-second intro hook, 16-second main motif, 10-second ending.
Vocals: optional humming only, no full lyrics.
Use case: intro and outro of a knowledge video series.
```

但新手先不要急着做主题曲。大多数教程和广告短片，先把背景音乐做好就够了。

## 5. Suno 或同类工具的工作流理解

Suno 这类 AI 音乐工具常用于生成歌曲、背景音乐和不同风格的音乐素材。它们的具体功能、权利规则和账号权益会变化，所以本书只讲工作流。

推荐使用方式：

```text
Music Brief
-> Prompt
-> 生成 2 到 4 首候选
-> 下载或记录链接
-> 放入剪辑时间线
-> 和配音一起试听
-> 评审选用
-> 记录授权和来源
```

### 5.1 不要一开始就追求完整歌曲

短视频项目通常需要的是：

- 15 秒开场音乐。
- 30 秒背景音乐。
- 60 秒循环底乐。
- 结尾 3 秒上扬。

不是一首完整流行歌。

### 5.2 生成后要做剪辑测试

AI 音乐单独听时可能很好，但一放到视频里会有问题：

- 鼓点太密，干扰字幕。
- 旋律太抓耳，抢配音。
- 开头太慢，3 秒内没有情绪。
- 结尾太突然，不适合导出。
- 歌词和旁白冲突。

所以必须放进时间线试听。

## 6. Udio 或同类工具的工作流理解

Udio 或同类 AI 音乐工具也可以用于生成音乐、延展音乐、根据上传音频生成变化等。只要涉及上传素材，就要特别注意素材权利。

你需要先问：

| 问题 | 原因 |
| --- | --- |
| 上传的音频是谁的 | 防止未经授权使用他人音乐或声音 |
| 是否允许改编 | 有些素材只允许试听或内部使用 |
| 输出是否可商用 | 按官方条款和账号状态确认 |
| 是否涉及歌词 | 歌词也可能有版权和授权问题 |
| 是否用于客户项目 | 客户交付需要更清楚的记录 |

如果你不确定某段音乐是否能上传或改编，就不要把它作为输入。最稳妥的方式是从零生成，或者使用你自己拥有权利的素材。

## 7. 节奏、BPM 和剪辑点

音乐节奏会影响剪辑点。

### 7.1 什么是 BPM

BPM 是 Beats Per Minute，每分钟节拍数。你不需要成为音乐人，但要知道大概感受：

| BPM 感觉 | 适合场景 |
| --- | --- |
| 慢 | 沉思、品牌、情绪片 |
| 中速 | 知识、产品、教程 |
| 快 | 高能混剪、游戏、促销 |

新手可以不用写具体 BPM，而写：

```text
中速，适合旁白和 1 到 3 秒一个切点。
```

### 7.2 剪辑点

音乐里常见的剪辑点：

- 鼓点。
- 低频冲击。
- 旋律变化。
- 段落转折。
- 停顿。

把重要画面放在音乐变化点上，观众会觉得更顺。

### 7.3 不要为了卡点牺牲信息

知识短视频里，配音和信息优先级高于音乐卡点。

如果卡点会让字幕来不及读，就不要硬卡。你可以让音乐服务信息，而不是让信息服从音乐。

## 8. 30 秒短视频音乐结构

以 30 秒短视频为例：

| 时间 | 音乐作用 | 剪辑作用 |
| --- | --- | --- |
| 0-3 秒 | 抓注意力 | 钩子画面和标题 |
| 3-10 秒 | 建立节奏 | 解释问题 |
| 10-22 秒 | 稳定推进 | 展示流程和案例 |
| 22-27 秒 | 稍微抬升 | 给结论 |
| 27-30 秒 | 收尾 | Logo、关注、结尾画面 |

音乐 Prompt 可以写：

```text
30-second instrumental background music.
0-3s: quick clean intro hook.
3-22s: steady minimal electronic rhythm for narration.
22-27s: subtle lift.
27-30s: clean ending, no abrupt stop.
```

这比只写“科技感音乐”更可控。

## 9. 音乐与人声避让

只要视频有旁白或口播，音乐就要让位。

### 9.1 人声优先

判断原则：

```text
观众听不清配音时，音乐再好也要降。
```

常见做法：

- 选择无歌词音乐。
- 降低音乐音量。
- 人声出现时让音乐自动或手动变小。
- 避免中频太满的音乐。
- 不在关键句下方放强冲击音效。

### 9.2 歌词的风险

有歌词的 AI 歌曲会和旁白冲突。

适合用歌词的场景：

- MV。
- 主题曲。
- 片尾。
- 无旁白的情绪片。

不适合：

- 知识解说。
- 产品卖点说明。
- 需要大量字幕的短视频。

### 9.3 混音检查

```markdown
| 检查项 | 是否通过 | 备注 |
| --- | --- | --- |
| 人声是否清楚 |  |  |
| 音乐是否盖住关键词 |  |  |
| 开头音乐是否太大 |  |  |
| 转场音效是否抢戏 |  |  |
| 手机外放是否听得清 |  |  |
| 戴耳机是否刺耳 |  |  |
```

要用手机外放听一次。很多短视频最终是在手机上被听到的，不是在专业音箱上。

## 10. 音效和转场声音

AI 视频常见画面比较“软”，音效可以帮它增加动作感。但音效用太多会变吵。

### 10.1 常用音效位置

| 位置 | 音效 |
| --- | --- |
| 标题出现 | clean hit |
| 镜头快速切换 | whoosh |
| 画面弹出对比 | click / pop |
| 结果展示 | light impact |
| 结尾 Logo | soft rise / resolve |

### 10.2 音效使用原则

- 有画面动作才加音效。
- 一个镜头不要叠太多音效。
- 音效不能盖住人声。
- 风格要和视频统一。
- 商业项目也要记录音效来源和授权。

如果你不确定要不要加音效，先不加。安静比廉价更好。

## 11. 音乐版本评审

AI 音乐生成后，做评审表。

```markdown
| 文件 | 情绪 | 节奏 | 是否抢人声 | 是否适合剪辑 | 授权状态 | 问题 | 结论 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| BGM_ai_workflow_v01.mp3 | 科技 | 稳 | 否 | 是 | 待确认 | 结尾太突然 | 可裁切 |
| BGM_ai_workflow_v02.mp3 | 兴奋 | 快 | 部分 | 部分 | 待确认 | 鼓点太密 | 备选 |
| BGM_ai_workflow_v03.mp3 | 温暖 | 慢 | 否 | 否 | 待确认 | 不够科技 | 不选 |
```

评审时不要只听前 10 秒。要检查：

- 开头。
- 中段循环。
- 情绪变化。
- 结尾。
- 是否有意外人声。
- 是否有奇怪歌词。
- 是否方便裁切。

## 12. 授权和商用风险

AI 音乐的权利问题比很多新手想象得复杂。不同平台、账号、地区和使用场景可能不同。

你需要记录：

| 项目 | 说明 |
| --- | --- |
| 工具来源 | Suno、Udio、其他平台或素材库 |
| 生成日期 | 方便回查条款 |
| 账号状态 | 免费、付费、企业、客户账号等 |
| 使用范围 | 内部样片、公开短视频、商业广告 |
| 是否含歌词 | 歌词可能带来额外风险 |
| 是否上传参考音频 | 上传音频必须有权使用 |
| 是否可二次剪辑 | 商业交付常需要裁切和混音 |
| 是否可分发 | 音乐平台、短视频平台、广告投放不同 |

### 12.1 音乐授权记录表

```markdown
# Music License Record

## 文件

BGM_ai_workflow_v01.mp3

## 来源

AI 音乐平台生成。

## 生成日期

2026-07-10

## 使用项目

AI 视频工作流知识短视频。

## 使用范围

抖音、视频号、小红书发布。

## 是否商业项目

否 / 是，按合同填写。

## 是否含上传素材

否。

## 官方条款核验

已查看平台当前 Terms / Rights / Ownership 页面。

## 风险备注

具体版权和商用范围需以当前官方页面、账号状态和项目合同为准。
```

### 12.2 不要做的事

- 不要上传你没有权利的歌曲。
- 不要让 AI 生成“像某某歌手”的声音或风格用于商业发布。
- 不要把平台生成结果的权利理解成全球永久无条件版权。
- 不要把免费账号的使用范围当成付费账号规则。
- 不要把个人学习样片的规则套到客户广告片。

## 13. 文件命名与入库

音乐目录建议：

```text
project/
├── audio/
│   ├── voice/
│   ├── music/
│   │   ├── raw/
│   │   ├── selected/
│   │   ├── rejected/
│   │   └── masters/
│   └── sfx/
└── edit/
```

音乐命名规则：

```text
BGM_项目或用途_版本_特点.ext
THEME_项目_版本_特点.ext
SFX_动作_版本.ext
AMB_场景_版本.ext
```

示例：

```text
BGM_ai_workflow_v01_minimal_tech.mp3
BGM_ai_workflow_v02_fast_pulse.wav
SFX_title_hit_v01.wav
AMB_city_night_v01.wav
```

入库表：

```markdown
| 文件 | 类型 | 用途 | 来源 | 是否授权确认 | 是否进入剪辑 | 备注 |
| --- | --- | --- | --- | --- | --- | --- |
| BGM_ai_workflow_v01_minimal_tech.mp3 | BGM | 主背景音乐 | AI 生成 | 待确认 | 是 | 结尾需淡出 |
| SFX_title_hit_v01.wav | SFX | 标题出现 | 素材库 | 待确认 | 是 | 音量降低 |
```

## 14. Ross 风格案例中的音乐推断

Ross 案例页面已经在项目中记录了可验证信息。我们不能断言它真实使用了哪首音乐或哪个音乐工具，但类似“AI 工作流知识短视频”通常需要音乐完成三个任务：

1. 开头抓注意力。
2. 中段托住科技感和效率感。
3. 结尾给出趋势判断。

音乐结构可以这样设计：

| 时间 | 音乐策略 |
| --- | --- |
| 00:00-00:03 | clean hit + 短促电子钩子 |
| 00:03-00:30 | 极简电子底乐，鼓点稳定 |
| 00:30-00:50 | 略微增强，配合工具展示 |
| 00:50-01:06 | 降低复杂度，让结论清楚 |
| 01:06-01:12 | 轻微上扬，干净结束 |

Prompt 可以写：

```text
Instrumental background music for a 72-second vertical short video about AI video workflow.
Modern, focused, clean technology feeling.
Minimal electronic rhythm, light percussion, subtle synth pulse.
No vocals, no lyrics, no heavy bass.
Opening 3 seconds should grab attention.
Middle section should support narration.
Ending should lift slightly and resolve cleanly.
```

注意，这只是复刻工作流的音乐设计，不是对原视频真实音乐来源的断言。

## 15. 常见错误

### 错误 1：先找音乐，后想用途

表现：歌很好听，但视频不匹配。

修正：先写 Music Brief。

### 错误 2：歌词盖住旁白

表现：观众同时听歌词和解说，信息混乱。

修正：知识视频优先用无歌词音乐。

### 错误 3：音乐太满

表现：整条视频一直高能，观众疲劳。

修正：让音乐有起伏，给重点句留空间。

### 错误 4：不记录授权

表现：客户问音乐能否商用时答不上来。

修正：每个音乐文件都写来源和授权记录。

### 错误 5：把 AI 音乐当最终母带

表现：音量、结尾、循环点不适合成片。

修正：进入剪辑软件后做裁切、淡入淡出和音量调整。

### 错误 6：音效乱飞

表现：每个字幕弹出都有声音，视频很吵。

修正：只给关键动作加音效。

## 16. 故障排查

### 音乐太抢人声

处理：

- 换无歌词版本。
- 降低音乐音量。
- 人声出现时手动压低音乐。
- 换更轻的编曲。
- 避免强旋律和强中频。

### 音乐开头不抓人

处理：

- 单独生成开场 3 秒音效。
- 在 Prompt 中写清 intro hook。
- 剪辑时从音乐更有动作的位置开始。
- 加轻微标题 hit，但不要盖住第一句配音。

### 音乐循环不自然

处理：

- 找节拍或段落边界裁切。
- 加淡入淡出。
- 生成更长版本。
- 换更稳定的背景音乐。

### 音乐结尾太突然

处理：

- 加淡出。
- 生成带 clean ending 的版本。
- 结尾用单独 resolve 音效。
- 把结尾镜头延长 0.5 秒。

### 音乐情绪和画面不匹配

处理：

- 回到 Music Brief。
- 写清核心情绪和不要的情绪。
- 对照导演阐述和配音语气。
- 不要因为音乐好听就硬用。

### 不确定能不能商用

处理：

- 查看平台当前 Terms、Rights、Ownership、FAQ。
- 记录核验日期。
- 不确定时不要用于客户商业项目。
- 需要商业交付时让客户确认音乐来源和授权范围。

## 18. 完整案例：为 AI 工作流短视频做音乐

主题：

```text
AI 视频不是一句提示词，而是一套项目化工作流。
```

### 18.1 音乐目标

```markdown
核心情绪：现代、效率、可信、轻微兴奋。
不需要：史诗、悲壮、夜店、恐怖、过度燃。
使用方式：配合主旁白，作为背景音乐。
```

### 18.2 Prompt

```text
为一条 45 秒竖屏知识短视频生成无歌词背景音乐。
主题是 AI 视频工作流。
情绪：现代、专注、效率感、轻微兴奋。
风格：极简电子、轻鼓点、柔和合成器脉冲。
速度：中速，适合旁白。
结构：开头 3 秒有干净钩子，中段稳定推进，结尾轻微上扬并自然收束。
不要人声，不要歌词，不要重低音，不要史诗预告片感。
```

### 18.3 评审

```markdown
| 版本 | 判断 |
| --- | --- |
| v01 | 最适合主片，节奏稳，结尾可淡出 |
| v02 | 开头更抓人，但鼓点太密，压人声 |
| v03 | 太温暖，不像科技工作流 |
```

### 18.4 剪辑处理

```text
选择 v01
-> 开头叠加轻微 title hit
-> 主旁白出现时降低音乐
-> 关键画面切换处轻微提高鼓点
-> 结尾 1.5 秒淡出
```

这就是音乐进入成片的方式。

## 19. 本卷总结

AI 音乐不是“生成一首好听的歌”。

稳定流程是：

```text
先写 Music Brief
-> 再写音乐 Prompt
-> 生成多个候选
-> 放进时间线试听
-> 检查人声避让
-> 裁切和淡入淡出
-> 记录来源和授权
-> 和配音、字幕、画面一起终审
```

新手最该记住：

- 背景音乐要托住视频，不要抢视频。
- 有旁白时，优先无歌词音乐。
- 音乐要服务剪辑点，但不能牺牲信息清晰。
- 商业项目必须记录音乐来源和授权。

当你能把音乐当成项目资产管理，剪辑会更稳，成片会更完整。

第 13 卷会把画面、配音、音乐、字幕、音效全部放进剪辑软件，完成真正的成片。

## 20. 课后练习

### 练习 1：Music Brief

为一条 30 秒 AI 视频写一份 Music Brief。

必须包含：

- 视频类型。
- 目标观众。
- 核心情绪。
- 不要的情绪。
- 音乐类型。
- 是否需要人声。
- 输出长度。

### 练习 2：三种音乐 Prompt

为同一个视频写三条不同音乐 Prompt：

1. 科技感。
2. 温暖成长感。
3. 紧张悬念感。

比较它们会如何改变观众感受。

### 练习 3：音乐评审表

找 3 首候选音乐，填一张评审表。

必须包含：

- 情绪。
- 节奏。
- 是否抢人声。
- 是否适合剪辑。
- 授权状态。
- 结论。

### 练习 4：授权记录

为一首 AI 生成音乐写授权记录。

必须记录：

- 工具来源。
- 生成日期。
- 是否商业用途。
- 是否上传参考音频。
- 当前官方条款核验位置。
