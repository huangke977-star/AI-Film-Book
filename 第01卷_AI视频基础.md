# 第 01 卷：AI 视频基础

## 学习目标

本卷是整套教程的地基。读完之后，你不需要马上会操作所有工具，但应该能建立一张清晰地图：

- AI 视频不是“输入一句话就出片”，而是创意、叙事、镜头、生成、剪辑和复盘的组合工程。
- 一条提示词只能解决局部问题，不能替代脚本、分镜、风格和角色一致性。
- 文生视频、图生视频、视频生视频、数字人、AI 口播视频属于不同生产方式。
- 电影感不是一个魔法词，而是镜头目的、光线、构图、运动、节奏和后期共同产生的结果。
- 做第一个作品时，不应该追求一分钟大片，而应该先完成一个 15 秒试验片的最小闭环。

本卷的目标不是让你记住术语，而是让你知道每个术语在真实制作中有什么用。

## 本卷导读：先建立地图

很多人第一次接触 AI 视频时，会立刻问：

```text
哪个模型最好？
提示词怎么写？
怎么生成电影感？
为什么人物总是变脸？
为什么别人做出来像电影，我做出来像廉价广告？
```

这些问题都重要，但顺序不能反。真正应该先问的是：

```text
我要做的视频由哪些环节组成？
每个环节的输入和输出是什么？
失败时我怎么判断问题出在哪里？
```

如果没有这张地图，你会陷入工具焦虑：今天研究 Seedance，明天研究 Kling，后天研究 Veo，再过两天又看到 LibTV、Pika、PixVerse、Hailuo。每个工具都好像很强，但你不知道该在什么时候用它，也不知道失败后该改提示词、换模型、换素材，还是进入剪辑软件修。

本卷先不追求炫技，只建立基础框架。后面的 Codex、Context Engineering、LibTV、镜头语言、导演思维、人物一致性，都会围绕这张地图展开。

## 1. AI 视频不是一个按钮

新手最容易把 AI 视频理解成：

```text
输入一句提示词 -> 等待生成 -> 下载视频
```

这确实是某些平台的表面操作，但不是完整制作。真正的视频制作更像：

```text
选题 -> 目标观众 -> 核心观点 -> 脚本 -> 分镜 -> 角色设定 -> 场景设定
-> 风格手册 -> 镜头提示词 -> 生成素材 -> 筛选 -> 迭代 -> 剪辑
-> 字幕 -> 配音/音乐/音效 -> 调色 -> 发布 -> 复盘
```

AI 可以加速其中很多环节，但它不会自动替你做导演判断。模型不知道你的观众是谁，不知道这个镜头为什么存在，不知道你下一镜头要怎么接，也不知道一个商业客户最在意产品 Logo、人物表情还是价格卖点。

你可以把 AI 视频理解为“让模型参与素材生产的视频制作”。它不是替代全部创作，而是把原来需要拍摄、找素材、做动效的一部分工作，转成了“用文字、图像、参考视频和上下文约束来生成素材”。

因此，学习 AI 视频时要先接受一个事实：

```text
模型负责生成可能性，创作者负责选择、约束、组织和复盘。
```

如果你把所有责任都推给模型，就会得到随机结果。如果你把项目拆清楚，模型就会变成一个很强的素材生产伙伴。

## 2. AI 视频制作的四层结构

为了不被工具淹没，我们把 AI 视频制作拆成四层：创意层、叙事层、镜头层、生产层。

### 2.1 创意层：这个视频为什么存在

创意层解决“为什么做”。它不是一句灵感，而是一组选择：

- 给谁看？
- 让观众看完记住什么？
- 视频要解决什么痛点？
- 情绪是震撼、轻松、专业、治愈，还是紧张？
- 发布在抖音、视频号、小红书、B 站，还是商业提案里？

示例：

```text
主题：用 Codex 和 LibTV 完成一条 AI 视频工作流演示。
观众：AI 视频新手、短视频创作者、想提升效率的人。
核心观点：手搓提示词正在被项目化 Agent 工作流替代。
情绪：惊喜、专业、未来感。
平台：抖音。
时长：60 到 75 秒。
```

这个信息看起来普通，但它决定后面的一切。如果目标观众是“完全新手”，你就不能大量使用术语；如果目标观众是“商业客户”，你就必须展示结果、成本和交付可信度。

### 2.2 叙事层：信息按什么顺序出现

叙事层解决“怎么讲”。即使是一条 15 秒短视频，也需要信息顺序。

常见结构：

| 结构 | 适合场景 | 示例 |
| --- | --- | --- |
| 问题开场 | 教程、知识类、工具类 | 为什么你的 AI 视频总是没有电影感 |
| 结果开场 | 作品展示、案例拆解 | 先给观众看成片，再讲怎么做 |
| 反差开场 | 成本、效率、工具对比 | 这条片只花了一顿外卖的钱 |
| 过程开场 | 工作流展示 | 从 brief 到分镜到生成到剪辑 |
| 情绪开场 | 电影感、品牌片、短剧 | 先用强画面抓住注意力 |

叙事层决定观众是否愿意继续看。很多 AI 视频单帧很好看，但成片不好看，原因就在于只有画面，没有叙事。

### 2.3 镜头层：观众到底看见什么

镜头层解决“怎么拍”。AI 视频提示词最常见的问题，是把“画面描述”误当成“镜头设计”。

弱提示词：

```text
一个未来城市，很震撼，电影感。
```

更好的镜头描述：

```text
夜晚的未来城市，雨后街面反射蓝色霓虹。低机位广角镜头从湿润地面缓慢向前推，前景有行人模糊经过，远处高楼玻璃幕墙反射灯光。画面冷暖对比，真实电影摄影，低饱和，浅景深，轻微胶片颗粒。
```

差别在哪里？

- 有时间：夜晚。
- 有空间：雨后街面、未来城市、高楼。
- 有主体：行人、街面、高楼。
- 有镜头：低机位、广角、缓慢向前推。
- 有光线：蓝色霓虹、反射灯光。
- 有质感：低饱和、浅景深、胶片颗粒。

镜头层不是堆形容词，而是把观众看到的画面、相机的位置、运动方式和视觉重点说清楚。

### 2.4 生产层：如何把想法变成文件

生产层解决“怎么落地”。它包括：

- 项目目录怎么建。
- 脚本、分镜、风格手册放在哪里。
- 每个镜头提示词怎么保存。
- 生成素材怎么命名。
- 失败样本怎么记录。
- 哪些文件提交到 Git。
- 哪些文件不该提交。
- 最终成片怎么导出和归档。

这就是 Codex、Git、LibTV、剪映、Premiere Pro、DaVinci Resolve 等工具进入的位置。

如果创意层、叙事层、镜头层都很清楚，但生产层混乱，你还是会遇到这些问题：

- 找不到上一版提示词。
- 不知道哪个素材能用。
- 不记得失败原因。
- 不知道成片用了哪个版本。
- 下次做类似项目又从零开始。

所以本书反复强调：AI 视频不是聊天记录，而是项目文件。

## 3. AI 视频和传统视频制作的关系

传统视频制作通常需要：

- 编剧写脚本。
- 导演控制表达。
- 摄影师拍画面。
- 灯光师处理光线。
- 美术设计场景和道具。
- 演员完成表演。
- 剪辑师完成节奏。
- 调色、声音、包装完成质感。

AI 视频不会让这些环节消失，而是改变它们的执行方式。

在 AI 工作流里：

| 传统环节 | AI 工作流中的对应方式 |
| --- | --- |
| 编剧 | ChatGPT/Codex 辅助脚本和口播稿 |
| 导演 | 人类决定意图，Codex 整理导演阐述 |
| 摄影 | 提示词中描述景别、机位、焦段、运镜 |
| 灯光 | 提示词中描述光线方向、强弱和色温 |
| 美术 | 风格手册和参考图控制场景、服装、道具 |
| 演员 | 数字人、图生视频角色、口播驱动模型 |
| 剪辑 | 剪映、PR、DaVinci 或其他工具完成组接 |
| 场记 | iteration log 记录失败样本和版本 |

这意味着，AI 视频创作者不一定要成为传统剧组里的每个岗位，但必须理解每个岗位在解决什么问题。你不懂镜头，就只能写“电影感”；你不懂剪辑，就只能堆素材；你不懂美术，就无法控制风格；你不懂场记，就无法复盘失败。

## 4. 文生视频、图生视频、视频生视频、数字人和口播视频

不同 AI 视频方式的输入和输出不一样，适合的场景也不一样。

| 类型 | 输入 | 输出 | 适合场景 | 常见风险 |
| --- | --- | --- | --- | --- |
| 文生视频 | 文字提示词 | 视频片段 | 概念镜头、风景、抽象画面 | 可控性较弱，人物容易漂移 |
| 图生视频 | 参考图 + 动作提示 | 视频片段 | 产品展示、角色动起来、场景微动 | 动作过大时容易变形 |
| 视频生视频 | 原始视频 + 风格/修改要求 | 新视频 | 风格化、换背景、增强质感 | 可能破坏原动作或细节 |
| 数字人 | 人物形象 + 音频/文本 | 口播视频 | 课程、知识口播、品牌解说 | 嘴型、表情、手势不自然 |
| AI 口播视频 | 脚本 + 虚拟形象 + 背景 | 解说型视频 | 工具讲解、案例拆解、带货说明 | 长时间单一画面容易无聊 |

Ross 案例更接近“AI 虚拟口播 + 多场景背景 + 自动字幕 + 工作流观点表达”。它不是单纯的文生视频，也不是单纯的数字人，而是多种能力组合后的短视频成片。

对新手来说，最稳的路线通常是：

```text
先写 brief
-> 拆分镜
-> 为每个镜头生成图像或视频素材
-> 用字幕和剪辑统一表达
-> 逐步尝试数字人或虚拟口播
```

不要一开始就挑战长剧情、复杂动作、多人对话和跨场景人物一致性。那会把所有难点同时堆在一起。

## 5. 为什么很多 AI 视频没有电影感

“电影感”是 AI 视频教程里最常见也最容易被误用的词。很多人会在提示词最后加一句：

```text
cinematic, movie style, 8k, ultra realistic
```

这不一定没用，但远远不够。电影感不是单个词，而是多个因素叠加。

### 5.1 缺少镜头目的

电影里的镜头不是为了好看而存在，而是为了表达。

一个推镜可能表达：

- 接近真相。
- 情绪压迫。
- 角色被逼近。
- 重点信息被揭示。

一个远景可能表达：

- 人物渺小。
- 环境宏大。
- 孤独感。
- 空间关系。

如果提示词只写“电影感”，模型不知道你要哪一种表达。

### 5.2 缺少光线设计

光线决定质感。常见电影感光线包括：

- 侧逆光：勾出人物轮廓。
- 窗边自然光：柔和真实。
- 屏幕光：适合创作者、程序员、AI 工作流主题。
- 霓虹光：适合未来城市和科技感。
- 低照度高反差：适合悬疑和紧张氛围。

弱写法：

```text
一个人在房间里，很电影感。
```

更好写法：

```text
夜晚工作室，显示器冷光照亮人物面部，右后方有暖色台灯形成轮廓光，背景微暗，浅景深，真实纪录片质感。
```

### 5.3 缺少空间连续性

AI 很容易生成每个镜头都好看的素材，但剪在一起不像同一部片。原因是镜头之间缺少连续性：

- 地点不一致。
- 光线方向不一致。
- 角色服装不一致。
- 背景道具不一致。
- 色彩风格不一致。

解决方法不是每条提示词都重写，而是建立 `style_guide.md` 和 `character_bible.md`。后面的第 04 卷会专门讲。

### 5.4 缺少剪辑节奏

一条短片通常需要景别变化：

```text
远景建立环境
-> 中景交代人物和动作
-> 近景强化信息
-> 特写制造记忆点
```

如果所有镜头都是正脸中景，观众很快疲劳。如果所有镜头都是大远景，观众又无法建立情绪连接。

电影感不仅来自生成模型，也来自剪辑阶段的选择。

## 6. 人物一致性、空间连续性和剪辑节奏

AI 视频最难的不是生成单个漂亮镜头，而是让多个镜头属于同一个作品。

### 6.1 人物一致性

人物一致性包括：

- 脸型。
- 年龄。
- 发型。
- 服装。
- 体型。
- 气质。
- 说话方式。
- 姿态和动作习惯。

如果每个镜头都重新描述人物，模型很容易把人物当成新角色。正确做法是建立角色圣经。

角色圣经示例：

```markdown
# Character Bible

角色：年轻 AI 视频创作者
年龄：28 岁左右
发型：黑色短发，略微凌乱
服装：灰绿色连帽衫
气质：专注、冷静、略带兴奋
固定元素：正面对镜头口播，双手靠近下巴或放在桌面
禁止变化：不要换发型，不要换年龄，不要换成西装，不要变成卡通风格
```

### 6.2 空间连续性

空间连续性指观众相信镜头发生在同一个世界里。

比如 Ross 案例中，虚拟口播人物前景相对稳定，背景可以从城市夜景切到火箭发射场，但整体仍保持真实电影摄影、冷暖对比、大字幕口播的统一感。

如果你让一个镜头是赛博朋克城市，下一镜头是日系动漫教室，再下一镜头是油画森林，观众会觉得素材来自不同项目。

### 6.3 剪辑节奏

剪辑节奏不是后期才考虑的事，它应该在分镜阶段就出现。

一个 15 秒短片可以这样安排：

| 时间 | 镜头 | 目的 |
| --- | --- | --- |
| 0-3s | 强画面开场 | 抓住注意力 |
| 3-6s | 主体出现 | 让观众知道看谁 |
| 6-9s | 方法或动作 | 展示过程 |
| 9-12s | 结果画面 | 证明价值 |
| 12-15s | 标题或结论 | 留下记忆点 |

这个结构比“随机生成 5 个好看的镜头”稳定得多。

## 7. 从手搓提示词到项目化工作流

早期 AI 创作者常常收藏很多提示词：

```text
cinematic, ultra realistic, 8k, masterpiece, dramatic lighting...
```

这有用，但不够。因为真实项目需要的不只是一条提示词，而是一套上下文。

项目化工作流会把信息拆成文件：

```text
project/
├── brief.md
├── script.md
├── storyboard.md
├── style_guide.md
├── character_bible.md
├── prompts/
│   ├── shot_01.md
│   ├── shot_02.md
│   └── shot_03.md
├── outputs/
│   ├── raw/
│   ├── selected/
│   └── final/
└── iteration_log.md
```

每个文件解决一个问题：

| 文件 | 作用 |
| --- | --- |
| `brief.md` | 项目目标和边界 |
| `script.md` | 口播、字幕、信息节奏 |
| `storyboard.md` | 镜头表 |
| `style_guide.md` | 统一画面风格 |
| `character_bible.md` | 保持人物一致 |
| `prompts/` | 每个镜头的生成提示词 |
| `outputs/` | 生成素材 |
| `iteration_log.md` | 失败样本和修改记录 |

Codex 的价值就在这里。它可以读这些文件，帮你保持上下文一致，而不是每次在聊天框里从零开始。

## 8. Ross 案例在本卷中的位置

Ross 的抖音案例标题是：

```text
全程用codex和libtv完成 手搓提示词的时代过去了
```

这个标题的重点不是“某条提示词很厉害”，而是“工作流变了”。

从已核验画面看，这条视频包含：

- AI 生成的电影感人物画面。
- 男性虚拟口播形象。
- 城市夜景和火箭发射场等背景切换。
- 黄色大字幕。
- 关于成本和多模型选择的口播。
- 对 Codex + LibTV 工作流的传播表达。

它可以被拆成四层：

| 层级 | Ross 案例中的体现 |
| --- | --- |
| 创意层 | 手搓提示词时代过去，Agent 工作流成为新方式 |
| 叙事层 | 开场吸引、展示虚拟形象、讲成本、讲模型选择、收束观点 |
| 镜头层 | 人物近景、口播中景、背景切换、黄色字幕 |
| 生产层 | Codex 组织内容，LibTV/多模型生成素材，后期剪辑发布 |

这就是本书第一阶段反复使用它的原因：它不是单个工具演示，而是一个完整工作流样本。

## 9. 一个 15 秒试验片的最小闭环

完全新手不要一开始做 1 分钟视频。第一条练习片建议只有 15 秒，目标是跑通流程。

### 9.1 试验片主题

```text
主题：我用 AI 做一条 15 秒视频的过程。
目标观众：和我一样刚开始学 AI 视频的人。
核心观点：先把流程跑通，比追求大片更重要。
```

### 9.2 5 个镜头

| 镜头 | 时长 | 画面 | 目的 |
| --- | --- | --- | --- |
| S01 | 3s | 夜晚桌面，电脑打开项目文件夹 | 建立创作场景 |
| S02 | 3s | 屏幕上出现 brief 和分镜表 | 展示项目化 |
| S03 | 3s | AI 生成的城市镜头 | 展示画面结果 |
| S04 | 3s | 剪辑时间线快速组接 | 展示后期 |
| S05 | 3s | 手机播放成片，出现标题 | 完成闭环 |

### 9.3 单镜头提示词示例

弱提示词：

```text
一个人在电脑前做 AI 视频，电影感。
```

改进提示词：

```text
夜晚的创作者工作室，桌面整洁，显示器上打开 AI 视频项目文件夹。年轻创作者坐在电脑前，屏幕冷光照亮脸部，右侧有暖色台灯形成轮廓光。中近景，固定机位，轻微手持微动，真实纪录片质感，低饱和，冷暖对比，浅景深。不要卡通感，不要乱码文字，不要多余手指。
```

### 9.4 最小闭环不是最小质量

最小闭环的意思不是随便做，而是只验证核心流程：

```text
brief -> storyboard -> style guide -> prompt -> generated asset -> selected asset -> edit -> review
```

如果这个闭环跑通，再增加时长、角色、复杂场景和商业要求。

### 9.5 第一条作品的决策清单

在你真正生成素材之前，先回答下面的问题。答不出来，就先不要打开生成工具。

| 问题 | 为什么重要 | 新手推荐答案 |
| --- | --- | --- |
| 视频给谁看 | 决定语言和节奏 | 给和我一样的新手看 |
| 视频多长 | 决定镜头数量和成本 | 15 秒 |
| 有几个镜头 | 决定提示词数量 | 5 个 |
| 是否有人物 | 决定一致性难度 | 第一条可以没有固定人物 |
| 是否需要字幕 | 决定后期工作 | 需要，但后期添加 |
| 是否需要配音 | 决定声音流程 | 可以先不用 |
| 是否需要复杂动作 | 决定生成风险 | 不需要 |
| 是否有固定风格 | 决定画面统一 | 真实纪录片或电影感工作室 |

这个清单的作用是降低难度。新手最常见的失败，不是审美不行，而是同时打开太多难题。比如你第一条片就想要固定人物、复杂动作、多场景、中文屏幕文字、配音、字幕、音乐、长镜头和商业质感，任何一个环节失败都会影响整体。

更稳的方式是把难题排队：

```text
第一条：只验证流程
第二条：增加人物
第三条：增加声音
第四条：增加复杂场景
第五条：尝试商业交付
```

这就是为什么本书会先讲基础，再讲环境，再讲 Codex，再讲 Context Engineering，最后才进入 LibTV。顺序本身就是降低失败率的方法。

### 9.6 失败原因的五类判断

当一个镜头失败时，不要立刻说“模型不行”。先把失败原因分成五类：

| 类型 | 表现 | 应该先改什么 |
| --- | --- | --- |
| 需求问题 | 不知道镜头要表达什么 | 改 brief 或分镜 |
| 提示词问题 | 主体、场景、动作、光线描述不清 | 改单镜头 prompt |
| 素材问题 | 参考图质量差或角色设定不稳定 | 换参考图或补角色圣经 |
| 模型问题 | 当前模型不擅长该任务 | 换模型或换生成方式 |
| 后期问题 | 素材能用但节奏、字幕、声音不好 | 进剪辑软件处理 |

举例：

```text
问题：屏幕上的中文乱码。
判断：这通常不是分镜问题，而是生成模型不擅长可读文字。
处理：提示词里弱化屏幕文字，后期用剪辑软件添加真实字幕或界面贴图。
```

```text
问题：人物每个镜头都变脸。
判断：可能是角色圣经不足，也可能是镜头运动和场景变化太大。
处理：固定角色描述，减少复杂运镜，先用更短镜头测试。
```

```text
问题：每个镜头都好看，但剪起来很乱。
判断：这通常是叙事层和剪辑节奏问题。
处理：回到 storyboard，重新安排镜头顺序和每个镜头目的。
```

只要你能把失败放进这五类，下一步就不会乱。

## 10. 常见错误

### 错误 1：一开始就追求大片

表现：

- 想做 3 分钟短片。
- 想要复杂剧情。
- 想要多角色对话。
- 想要跨很多场景。

后果：

- 人物变脸。
- 风格断裂。
- 成本失控。
- 失败后不知道从哪里改。

修正：

先做 15 秒，5 个镜头，只验证一个主题。

### 错误 2：把提示词当成全部

表现：

- 只问“提示词怎么写”。
- 不写 brief。
- 不写分镜。
- 不记录失败样本。

后果：

- 每次生成都像抽奖。
- 成功无法复用。
- 失败无法定位。

修正：

把提示词放回项目上下文里，让它服务分镜和风格手册。

### 错误 3：每个镜头都重新设定世界

表现：

- 每条提示词都重新描述角色、场景和风格。
- 没有统一风格段落。
- 不复用角色描述。

后果：

- 人物、服装、场景、光线全都漂移。

修正：

建立 `style_guide.md` 和 `character_bible.md`。

### 错误 4：让模型生成可读文字

表现：

- 要求屏幕上出现清晰中文。
- 要求海报、招牌、字幕都由视频模型生成。

后果：

- 文字乱码。
- 字体不稳定。
- 品牌信息错误。

修正：

模型负责画面，字幕和关键文字交给后期软件。

### 错误 5：删除失败样本

表现：

- 只保留最好看的结果。
- 失败视频直接删。

后果：

- 不知道下一轮该怎么改。
- 无法总结模型弱点。

修正：

保留代表性失败样本，在 `iteration_log.md` 中写清楚问题。

## 11. 故障排查

### 画面好看但不像同一部片

检查：

- 是否有统一风格手册。
- 色彩、光线、画幅是否一致。
- 角色是否保持同一服装和发型。

处理：

- 先固定风格段落。
- 每条镜头提示词都引用同一套风格。
- 减少场景跳跃。

### 人物频繁变脸

检查：

- 是否有角色圣经。
- 是否要求动作太复杂。
- 是否频繁切换机位和背景。

处理：

- 固定人物描述。
- 先用中景和近景，不急着环绕。
- 缩短单镜头时长。

### 镜头运动失控

检查：

- 是否同时写了推、拉、摇、移、环绕。
- 是否使用了过多 `dynamic`、`fast`、`dramatic` 之类词。

处理：

- 每个镜头只保留一种主要运动。
- 使用 `slow`、`stable`、`subtle` 等约束。

### 成本很快失控

检查：

- 是否每个镜头都生成太多版本。
- 是否没有先做低成本测试。
- 是否把问题归因给模型而不是提示词。

处理：

- 每个镜头先生成 2 个版本。
- 只对关键镜头加版本。
- 先复盘失败原因，再决定是否重生成。

## 12. 参数与概念速查

| 概念 | 简单解释 | 新手要记住 |
| --- | --- | --- |
| brief | 项目目标说明 | 决定视频为什么存在 |
| script | 脚本或口播稿 | 决定说什么 |
| storyboard | 分镜表 | 决定每个镜头拍什么 |
| style guide | 风格手册 | 决定画面是否统一 |
| character bible | 角色圣经 | 决定人物是否稳定 |
| prompt | 生成提示词 | 只负责把单个任务讲清楚 |
| negative prompt | 负面约束 | 告诉模型避免什么 |
| iteration log | 迭代记录 | 把失败变成资产 |
| raw output | 原始生成素材 | 不一定能直接用 |
| selected output | 已筛选素材 | 可以进入剪辑 |

如果这张表里有一半以上空着，就不要急着进入工具操作。回到本卷前面的章节，把每个概念用自己的话说一遍。

真正的理解不是能背定义，而是能判断：

```text
这个问题属于哪一层？
我应该改哪个文件？
我应该让 Codex 做什么？
我应该让生成平台做什么？
我应该留到剪辑阶段解决什么？
```

这五个问题，是后续所有学习的基础。

进入第 02 卷前，还有一个简单标准：你应该能在不打开任何生成平台的情况下，向别人讲清楚一条 15 秒 AI 视频会如何从想法变成文件。如果你能说出 `brief -> storyboard -> style guide -> prompt -> output -> iteration log` 这条链路，并知道每一步失败时该回到哪里修改，就说明本卷的地基已经打稳。

## 14. 本卷总结

本卷只讲一件事：AI 视频制作是一套流程，不是一句提示词。

你需要先建立四层地图：

```text
创意层：为什么做
叙事层：怎么讲
镜头层：看见什么
生产层：怎么落地
```

如果这四层清楚，工具只是执行方式。今天用 LibTV，明天用 Seedance，后天换成其他平台，底层逻辑仍然成立。

Ross 案例之所以值得拆，是因为它展示了一个趋势：AI 视频创作者正在从手搓提示词，转向项目化、上下文化、Agent 协作的工作流。

## 15. 课后练习

### 练习 1：写一个 brief

用 5 行写出：

```text
主题：

核心观点：
情绪：
时长：
```

### 练习 2：拆 5 个镜头

把你的 brief 拆成 5 个镜头。每个镜头只写：

```text
镜头编号：
时长：
画面：
目的：
```

### 练习 3：改写一个弱提示词

把这条弱提示词改成结构化提示词：

```text
一个人在电脑前做 AI 视频，电影感。
```

必须补充：

- 主体。
- 场景。
- 动作。
- 镜头类型。
- 运镜。
- 光线。
- 色彩。
- 风格。
- 负面约束。

创建 `iteration_log.md`，写入：

```markdown
# Iteration Log

| 日期 | 镜头 | 版本 | 问题 | 是否可用 | 下一步 |
| --- | --- | --- | --- | --- | --- |
```

先不生成视频，也要把表建好。
