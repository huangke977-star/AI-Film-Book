# 第 08 卷：导演思维

## 学习目标

学完本卷后，你应该能做到：

- 理解 AI 时代导演仍然重要，而且更像意图定义者和素材选择者。
- 写出一份可指导 Codex、视频模型和剪辑软件的导演阐述。
- 判断每个镜头是否服务叙事，而不是只看画面是否漂亮。
- 用“意图优先”原则控制镜头、节奏、风格和信息密度。
- 指挥 Codex 做创意助理、分镜审稿、素材筛选和复盘。
- 把普通画面描述改成带有导演判断的画面方案。

如果第 07 卷回答“镜头怎么拍”，第 08 卷回答“为什么这样拍”。

## 本卷导读：AI 时代导演仍然重要

很多人以为 AI 视频出现后，导演不重要了。其实刚好相反。

传统拍摄里，导演要和摄影、灯光、美术、演员、剪辑一起沟通。AI 视频里，导演要和 Codex、视频模型、图像模型、配音工具、剪辑软件沟通。工具变了，但核心问题没变：

```text
这个视频要表达什么？
观众应该先看到什么？
每个镜头为什么存在？
哪些素材可以用，哪些素材必须舍弃？
节奏应该快还是慢？
风格能不能保持一致？
```

AI 可以生成很多选择，但选择本身仍然是人的工作。导演思维不是摆姿态，而是让你不被素材牵着走。

## 1. 导演到底决定什么

导演决定的不是“按哪个按钮”，而是作品的判断标准。

### 1.1 导演的 7 个决定

| 决定 | 说明 | AI 视频里的表现 |
| --- | --- | --- |
| 主题 | 这条视频到底说什么 | brief 的核心观点 |
| 观众 | 给谁看 | 平台、语气、信息密度 |
| 情绪 | 观众应该感到什么 | 光线、节奏、音乐、字幕 |
| 风格 | 画面属于什么世界 | style_guide |
| 镜头目的 | 每个镜头为什么存在 | storyboard |
| 素材取舍 | 哪些素材进成片 | selected / rejected |
| 节奏 | 信息如何流动 | 剪辑、口播、停顿 |

### 1.2 AI 不替你决定

AI 可以给你十个开场，但它不知道哪个开场最适合你的观众。它可以生成很漂亮的火箭发射场，但如果你的视频主题是“本地小店如何用 AI 做宣传”，火箭画面可能就是噪音。

导演思维的第一条规则：

```text
不要因为素材好看就使用它。
```

素材必须服务目的。

## 2. 意图优先原则

意图优先的意思是：先决定这个镜头要完成什么，再决定画面怎么生成。

错误顺序：

```text
看到一个很酷的画面
-> 想办法把它塞进视频
-> 结果视频变散
```

正确顺序：

```text
确定镜头目的
-> 选择景别、机位、光线、动作
-> 生成素材
-> 按目的筛选
```

### 2.1 镜头目的示例

| 模糊写法 | 意图优先写法 |
| --- | --- |
| 做一个很酷的开场 | 用 3 秒建立“AI 视频已经进入项目化时代”的科技感 |
| 展示电脑屏幕 | 让观众看到创作过程有 brief、分镜和提示词文件 |
| 做一个人物口播 | 让观众相信有人在清楚解释方法 |
| 展示生成结果 | 证明前面的工作流可以产出成片 |

当意图写清楚，后面每个选择都会更稳。

### 2.2 意图检查句

写每个镜头前，先补完这句话：

```text
这个镜头存在，是为了让观众__________。
```

例如：

```text
这个镜头存在，是为了让观众意识到 AI 视频不是一条提示词，而是一个可管理的项目。
```

这句话比“电影感工作室画面”更重要。

## 3. 导演阐述怎么写

导演阐述是给整个项目定方向的文件。它不需要长，但必须清楚。

建议保存为：

```text
director_statement.md
```

### 3.1 导演阐述模板

```markdown
# Director Statement

## 核心意图

这条视频要让观众明白：

## 目标观众

他们现在的问题是：

## 观众看完后的变化

看完后，观众应该：

## 情绪曲线

开头：
中段：
结尾：

## 视觉风格

画面应该像：
画面不应该像：

## 镜头策略

主要使用：
避免使用：

## 节奏策略

前 3 秒：
中段：
结尾：

## 固定元素

## 可变化元素

## 禁止偏离项
```

### 3.2 示例

```markdown
# Director Statement

## 核心意图

让零基础观众明白：AI 视频制作正在从“手搓提示词”变成“项目化工作流”。

## 目标观众

看到 AI 视频案例很震撼，但不知道从哪里开始的人。

## 观众看完后的变化

他们不再只追问“提示词是什么”，而是开始关注 brief、分镜、上下文、生成复盘。

## 情绪曲线

开头：被高质量画面吸引。
中段：意识到背后有清晰工作流。
结尾：相信自己也可以按步骤学习。

## 视觉风格

真实纪录片 + 轻微科技感。低饱和，屏幕冷光，暖色台灯，不要廉价赛博朋克。

## 镜头策略

主要使用中近景、屏幕特写、俯拍桌面和慢推。避免复杂环绕和快速变焦。

## 节奏策略

前 3 秒给强画面，中段用字幕和工作流画面解释，结尾回到手机播放结果。

## 禁止偏离项

不要让视频变成纯工具广告。不要出现不可读中文。不要把失败样本删掉。
```

导演阐述写完后，后面的素材判断就有了依据。

## 4. 镜头为什么要有目的

没有目的的镜头，就算好看，也会拖慢视频。

### 4.1 镜头目的表

| 镜头目的 | 常见画面 | 判断标准 |
| --- | --- | --- |
| 吸引注意 | 强画面、反差、动作 | 前 3 秒能否让人停留 |
| 建立人物 | 口播、中近景、眼神 | 观众是否知道谁在讲 |
| 解释方法 | 文件、界面、分镜 | 信息是否清楚 |
| 证明结果 | 成片、对比、前后变化 | 观众是否相信方法有效 |
| 转场过渡 | 环境、手部、屏幕变化 | 是否让节奏不断 |
| 强化记忆 | 标题、结论、标志画面 | 观众是否记住观点 |

### 4.2 删镜头的方法

问三个问题：

```text
这个镜头是否提供新信息？
这个镜头是否推进情绪？
删掉它，观众是否更糊涂？
```

如果三个答案都是否定的，这个镜头可以删。

## 5. 节奏和信息控制

节奏不是单纯快剪。节奏是信息的呼吸。

### 5.1 短视频节奏

对 30 到 60 秒的短视频，可以这样理解：

| 时间段 | 任务 |
| --- | --- |
| 0-3 秒 | 让观众停下来 |
| 3-10 秒 | 说明这条视频和他有什么关系 |
| 10-40 秒 | 给出方法、过程或证据 |
| 40-55 秒 | 总结观点或展示结果 |
| 最后几秒 | 留下记忆点 |

### 5.2 信息密度

新手常犯的错误是每秒都想塞信息。结果是观众看不清、听不懂、记不住。

导演要控制：

- 画面信息。
- 字幕信息。
- 口播信息。
- 音效信息。
- 镜头运动信息。

如果一个镜头已经有复杂画面，就不要再放大量字幕。如果口播在讲关键概念，画面要辅助，而不是抢戏。

### 5.3 节奏示例

```text
S01：强画面开场，少字。
S02：口播提出问题，字幕一句话。
S03：文件夹和分镜表，解释工作流。
S04：生成素材，多版本对比。
S05：成片结果，给结论。
```

这比十几个随机酷炫镜头更像作品。

## 6. 如何判断素材是否服务叙事

素材筛选不是选最漂亮的，而是选最合适的。

### 6.1 素材选择表

```markdown
| 文件 | 镜头目的 | 是否服务目的 | 风格一致 | 技术问题 | 情绪是否正确 | 结论 |
| --- | --- | --- | --- | --- | --- | --- |
| S01_V01.mp4 | 强画面开场 | 是 | 是 | 轻微手部问题 | 是 | 可用 |
| S01_V02.mp4 | 强画面开场 | 否 | 是 | 无 | 太平 | 不用 |
| S02_V01.mp4 | 展示项目文件 | 部分 | 是 | 屏幕乱码 | 是 | 可剪裁 |
```

### 6.2 选择优先级

1. 是否服务镜头目的。
2. 是否符合导演阐述。
3. 是否和前后镜头连续。
4. 是否有严重技术问题。
5. 是否足够好看。

把“好看”放在最后，不是说它不重要，而是因为好看不能替代目的。

### 6.3 selected 和 rejected 都要保留

成熟的项目应该有：

```text
outputs/selected/
outputs/rejected/
iteration_log.md
```

被拒绝的素材要写原因。以后你会知道自己为什么拒绝它，而不是忘记判断过程。

## 7. 如何指导 Codex 做创意助理

Codex 很适合做导演助理，但你要给它清楚的工作边界。

### 7.1 让 Codex 审导演阐述

```text
请读取 brief.md、script.md、storyboard.md 和 director_statement.md。
请检查导演阐述是否回答了核心意图、目标观众、情绪曲线、视觉风格、镜头策略和禁止偏离项。
不要重写全文，只列出缺失项和修改建议。
```

### 7.2 让 Codex 审分镜目的

```text
请读取 storyboard.md。
逐镜头检查“镜头目的”是否明确。
如果某个镜头只是好看但没有信息作用，请标记为“目的不清”，并给出一个更明确的镜头目的。
```

### 7.3 让 Codex 做素材筛选表

```text
请根据 iteration_log.md 和 storyboard.md，生成素材选择表。
评价维度包括：是否服务镜头目的、风格一致、技术问题、情绪是否正确、结论。
不要只按画面质量排序。
```

### 7.4 让 Codex 改写普通画面

```text
请把下面画面描述改写成导演化镜头方案。
要求输出：镜头目的、景别、机位、焦段感、运镜、光线、构图、情绪、负面约束。
原始描述：一个人在电脑前做 AI 视频。
```

### 7.5 让 Codex 控制节奏

```text
请读取 script.md 和 storyboard.md。
检查 60 秒视频的信息节奏是否过密。
请标出需要删减、合并或延长停顿的镜头。
输出格式为表格：镜头、当前问题、修改建议、原因。
```

### 7.6 让 Codex 做复盘

```text
请读取 iteration_log.md。
从导演角度判断失败原因：镜头目的不清、情绪不对、风格漂移、节奏不对、技术问题。
请不要直接重写提示词，先做归因。
```

这些指令的共同点是：先读文件，再做判断；先归因，再修改。

## 8. 从普通画面到导演化画面

### 8.1 普通描述

```text
一个人坐在电脑前，屏幕上有 AI 视频工具。
```

### 8.2 导演化版本 A：知识教程

```markdown
## 镜头目的

让观众知道主角正在演示一个可学习的工作流。

## 画面

中近景，平视机位，创作者坐在电脑前，屏幕光柔和照亮脸部。背景干净，桌面有笔记本和键盘。主体在画面中央，底部留字幕空间。

## 情绪

清楚、可信、亲近。

## 运镜

固定机位，轻微手持微动。
```

### 8.3 导演化版本 B：电影感开场

```markdown
## 镜头目的

用氛围建立“夜晚创作”的沉浸感。

## 画面

远景，侧后方机位，夜晚工作室窗外有城市灯光。创作者背对镜头坐在电脑前，屏幕冷光和台灯暖光形成冷暖对比。

## 情绪

专注、安静、略带未来感。

## 运镜

固定镜头，只有轻微环境运动。
```

### 8.4 导演化版本 C：商业广告

```markdown
## 镜头目的

展示 AI 视频工作流可以提升效率。

## 画面

屏幕特写，多个项目文件整齐排列，素材缩略图从 raw 到 selected 被筛选出来。画面干净明亮，构图简洁。

## 情绪

高效、可靠、专业。

## 运镜

极慢推近到 selected 文件夹。
```

同一个原始描述，可以导向三种不同作品。导演思维就是先选择方向，再生成画面。

## 9. Ross 案例中的导演判断

Ross 案例可以学习的不是某个神秘按钮，而是导演判断：

| 观察 | 可能的导演判断 |
| --- | --- |
| 开头用高质量画面 | 先证明 AI 视频画面已经足够吸引 |
| 虚拟口播人物出现 | 让观点有讲述者 |
| 背景不断切换 | 防止知识讲解变单调 |
| 黄色大字幕 | 适配短视频快速阅读 |
| 提到成本和模型选择 | 降低观众进入门槛 |
| 回到 Codex + LibTV 工作流 | 强化核心观点 |

你不需要复制它的画面，但可以复制它的判断结构：

```text
先吸引
-> 再解释
-> 给证据
-> 展示方法
-> 留下观点
```

## 10. 导演检查清单

生成前检查：

- 核心意图是否一句话说清。
- 目标观众是否明确。
- 每个镜头是否有目的。
- 是否有导演阐述。
- 风格是否和主题匹配。
- 前 3 秒是否有钩子。
- 是否给字幕留空间。
- 是否避免复杂运镜堆叠。

生成后检查：

- 素材是否服务镜头目的。
- 情绪是否正确。
- 风格是否连续。
- 技术问题是否可剪辑处理。
- 是否需要重生成。
- 是否应该删掉某个镜头。
- 是否记录 selected / rejected 原因。

剪辑前检查：

- 信息节奏是否过密。
- 是否有重复镜头。
- 字幕是否压住主体。
- 音乐是否抢口播。
- 结尾是否回到核心观点。

## 11. 完整案例：30 秒工作流短视频的导演方案

下面把导演思维落到一个完整案例里。

项目主题：

```text
手搓提示词的时代正在过去，AI 视频进入 Codex + 视频生成模型的项目化工作流。
```

### 11.1 核心意图

这条视频不是炫耀工具，而是改变观众的提问方式：

```text
从“提示词是什么？”
变成“项目文件、分镜、风格、生成复盘怎么搭？”
```

### 11.2 观众画像

```text
看过 AI 视频作品，觉得震撼，但不知道如何开始。
有一点工具兴趣，但没有导演、剪辑和项目管理基础。
容易被模型名字和提示词案例带着跑。
```

### 11.3 情绪曲线

| 时间 | 情绪 | 画面策略 |
| --- | --- | --- |
| 0-3 秒 | 惊讶 | 高质量电影感开场 |
| 3-8 秒 | 好奇 | 口播提出“不是一句提示词” |
| 8-18 秒 | 理解 | 项目文件夹、分镜、提示词、生成记录 |
| 18-25 秒 | 相信 | 多版本素材筛选和成片片段 |
| 25-30 秒 | 记住 | 回到核心观点和标题 |

### 11.4 镜头方案

| 镜头 | 时长 | 镜头目的 | 画面 | 导演判断 |
| --- | --- | --- | --- | --- |
| S01 | 3s | 先让观众停下 | 夜晚城市工作室远景 | 开场要强，但不要赛博过度 |
| S02 | 4s | 提出观点 | 虚拟口播人物中近景 | 人物稳定比背景华丽更重要 |
| S03 | 5s | 展示项目化 | 俯拍桌面和项目文件 | 文件结构要比文字可读更重要 |
| S04 | 5s | 展示生成过程 | 抽象生成界面和素材缩略图 | 不出现假品牌，不要求可读中文 |
| S05 | 6s | 展示筛选判断 | 多版本素材 selected/rejected | 让观众知道不是一次出片 |
| S06 | 5s | 展示结果 | 手机播放短视频 | 回到发布平台语境 |
| S07 | 2s | 留下观点 | 标题字幕和人物回看镜头 | 强化记忆点 |

### 11.5 为什么这个方案成立

它不是按“哪个画面好看”排序，而是按观众理解过程排序：

```text
先被吸引
-> 知道观点
-> 看见方法
-> 看见过程
-> 看见结果
-> 记住结论
```

如果生成时出现一个特别漂亮的火箭发射镜头，但它不能服务这个顺序，就不应该强塞进去。最多把它作为情绪转场，而不能让它抢走主题。

## 12. 素材评审会议怎么做

你一个人做项目，也可以模拟“素材评审会议”。这会让你从素材收集者变成导演。

### 12.1 评审前准备

准备三个文件夹：

```text
outputs/raw/
outputs/selected/
outputs/rejected/
```

准备三份文件：

```text
storyboard.md
director_statement.md
iteration_log.md
```

没有导演阐述就评审素材，等于没有尺子量长度。

### 12.2 评审顺序

```text
先读镜头目的
-> 看素材
-> 判断是否服务目的
-> 判断技术问题是否可接受
-> 判断是否需要重生成
-> 写入 iteration_log.md
```

不要先看哪条最漂亮。先看镜头目的。

### 12.3 素材评审表

```markdown
| 镜头 | 文件 | 镜头目的 | 是否服务目的 | 情绪是否正确 | 风格是否连续 | 技术问题 | 决定 | 理由 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| S01 | S01_V01.mp4 | 强开场 | 是 | 是 | 是 | 轻微手部问题 | selected | 手部不在视觉中心 |
| S01 | S01_V02.mp4 | 强开场 | 否 | 否 | 是 | 无 | rejected | 画面漂亮但太平 |
| S03 | S03_V01.mp4 | 展示项目化 | 部分 | 是 | 是 | 屏幕乱码 | selected_partial | 可用裁切和字幕处理 |
```

### 12.4 selected_partial 的意义

不是所有素材都只有“可用”和“不可用”。有些素材可以局部使用：

- 前 1 秒可用，后面崩坏。
- 背景可用，人物不可用。
- 画面可用，但需要遮住文字。
- 适合做转场，不适合做主镜头。

建议增加一个状态：

```text
selected_partial
```

这能减少浪费，也能让剪辑阶段更灵活。

## 13. 导演给不同工具的指令包

导演思维要落地，必须能翻译成工具指令。

### 13.1 给 Codex 的指令

```text
请读取 brief.md、director_statement.md、storyboard.md 和 style_guide.md。
请从导演角度检查这条视频是否存在以下问题：
1. 镜头目的不清。
2. 情绪曲线断裂。
3. 风格和主题不匹配。
4. 前 3 秒没有钩子。
5. 某些镜头只是好看但不服务叙事。

输出表格：镜头、问题、修改建议、原因。
不要生成最终提示词，先做导演审稿。
```

### 13.2 给视频模型的任务卡

```markdown
# S01 导演任务卡

## 镜头目的

用 3 秒建立“AI 视频已经具备电影级视觉吸引力”的第一印象。

## 导演意图

观众应该先被画面吸引，然后愿意听后面的工作流解释。

## 景别和机位

远景，侧后方，夜晚工作室。

## 情绪

安静、专注、略带未来感。

## 禁止偏离

不要变成夸张赛博朋克，不要出现可读中文，不要让人物做复杂动作。
```

### 13.3 给剪辑阶段的指令

```text
请按照导演阐述检查剪辑时间线：
1. 前 3 秒是否足够吸引。
2. 每 5 到 8 秒是否有信息推进。
3. 字幕是否覆盖主体。
4. 音乐是否抢口播。
5. 结尾是否回到核心观点。
```

### 13.4 给配音阶段的指令

```text
这条视频的声音风格是冷静、可信、略带兴奋。
不要夸张销售腔。
前 3 秒语速略快，中段解释放慢，结尾强调“工作流”这个关键词。
```

这些指令看起来像不同工具的输入，本质上都来自同一份导演阐述。

## 14. 导演如何处理失败样本

失败样本不只是技术问题，也可能是导演问题。

### 14.1 技术失败和导演失败

| 失败类型 | 表现 | 处理 |
| --- | --- | --- |
| 技术失败 | 变脸、手指错、乱码 | 改提示词、换参考图、换模型 |
| 导演失败 | 好看但不服务目的 | 改分镜或删镜头 |
| 节奏失败 | 画面都可用但剪起来拖 | 调整镜头顺序和时长 |
| 情绪失败 | 画面情绪和主题不符 | 改光线、音乐、色彩 |
| 信息失败 | 观众看不懂 | 改字幕、口播和镜头目的 |

### 14.2 失败样本复盘句式

不要写：

```text
不好，重做。
```

写成：

```text
S03 技术上可用，但导演目的不成立。它没有展示项目化工作流，只是一个普通电脑画面。下一轮应改成俯拍桌面和文件结构，重点展示 brief、storyboard、prompts 三类文件。
```

这类复盘能直接指导下一轮生成。

## 15. 导演思维和后续卷的关系

第 08 卷会影响后面的所有卷：

| 后续卷 | 关系 |
| --- | --- |
| 第 09 卷人物一致性 | 角色是否稳定，不只是技术问题，也关系到观众是否相信人物 |
| 第 10 卷 AI 绘图 | 参考图要服务导演意图，不能只是漂亮图 |
| 第 11 卷 AI 配音 | 声音表演要符合情绪曲线 |
| 第 12 卷 AI 音乐 | 音乐要托住节奏，不要抢信息 |
| 第 13 卷剪辑 | 剪辑是导演判断的最终落地 |
| 第 14-18 卷商业与自动化 | 商业项目更需要明确意图和判断标准 |

所以不要把导演思维当成抽象审美课。它是后续所有制作动作的判断中心。

## 16. 导演日常工作流

真正做项目时，导演思维要变成每天能执行的顺序。

### 16.1 开始前 10 分钟

先打开：

```text
brief.md
director_statement.md
storyboard.md
style_guide.md
```

只问四个问题：

```text
今天要解决哪个镜头？
这个镜头的目的是什么？
它必须保持哪些风格和角色规则？
今天最多允许生成几次？
```

如果这四个问题答不出来，不要急着打开生成工具。

### 16.2 生成前 10 分钟

让 Codex 做一次轻量检查：

```text
请读取 S03 的任务卡、director_statement.md 和 style_guide.md。
检查这个镜头的目的、景别、机位、光线、动作是否互相冲突。
只输出冲突项和修改建议。
```

这一步能避免很多无效生成。

### 16.3 生成后 15 分钟

不要马上重试。先把素材放进对应目录：

```text
outputs/raw/
outputs/selected/
outputs/rejected/
```

然后写三句话：

```text
这个版本哪里符合导演意图？
哪里偏离了导演意图？
下一轮只改哪一个变量？
```

导演不是靠灵感工作，而是靠一次只改一个变量。今天只改景别，就不要同时改人物、背景、光线和风格。变量越少，你越知道下一轮为什么变好或变坏。

### 16.4 剪辑前 10 分钟

剪辑前再读一次导演阐述。检查 selected 素材是否真的能串成情绪曲线：

```text
吸引
-> 解释
-> 证明
-> 结果
-> 记忆点
```

如果 selected 里有漂亮素材但无法放进这条线，就先不要用。它可以留作素材库，不必进入当前成片。

## 17. 常见错误

### 错误 1：用素材反推主题

看到好看的素材，就想把主题改成它。

修正：回到导演阐述。素材服务主题，而不是主题服务素材。

### 错误 2：每个镜头都想震撼

如果每个镜头都震撼，观众会疲劳。

修正：让镜头有强弱。强画面负责吸引，平稳画面负责解释。

### 错误 3：把 Codex 当自动导演

Codex 可以帮你分析和改写，但最终判断要由你做。

修正：让 Codex 输出候选和理由，你决定采用哪个。

### 错误 4：不舍得删素材

花了成本生成的素材，不代表必须使用。

修正：不服务目的就进 rejected，并记录原因。

### 错误 5：只管画面不管节奏

画面都不错，但剪起来拖沓。

修正：检查每个镜头是否提供新信息或推进情绪。

## 18. 故障排查

### 视频看起来像素材堆砌

原因：

- 没有导演阐述。
- 镜头目的不清。
- 只按画面质量排序。

处理：

- 写 `director_statement.md`。
- 给每个镜头补目的。
- 重新按叙事顺序筛素材。

### 视频节奏很乱

原因：

- 信息密度过高。
- 镜头长短没有变化。
- 字幕、口播、画面同时抢信息。

处理：

- 给重点句留停顿。
- 合并重复镜头。
- 把复杂信息拆成两段。

### 画面好看但不记得内容

原因：

- 视觉压过观点。
- 缺少明确结论。
- 字幕没有帮助记忆。

处理：

- 每 10 到 15 秒回到一次核心观点。
- 用字幕强化关键词。
- 结尾重复一句主张。

### Codex 给的方案太散

原因：

- 没给导演阐述。
- 没给目标观众。
- 任务要求过宽。

处理：

- 让 Codex 先读取 brief 和 director_statement。
- 每次只让它解决一个问题。
- 要求输出表格和修改理由。

## 19. 本卷总结

导演思维的核心不是“我要亲自拍每个镜头”，而是“我知道每个镜头为什么存在”。

AI 视频时代，导演要做四件事：

```text
定义意图
-> 组织镜头
-> 筛选素材
-> 控制节奏
```

工具越强，选择越多；选择越多，导演判断越重要。不要让漂亮素材替你决定视频方向。先写导演阐述，再写分镜，再生成素材，再按镜头目的筛选。

导演判断也不是一次完成。第一轮判断方向，第二轮判断素材，第三轮判断剪辑，最后再判断观众是否能记住核心观点。每一轮都可以让 Codex 帮你整理表格和发现矛盾，但最终采用哪个方案，仍然要回到导演阐述和目标观众。

如果你暂时不会判断画面好坏，就先判断镜头目的是否清楚。目的清楚，审美可以慢慢提高；目的不清，画面再漂亮也很难成为作品。

这也是新手最可靠的入门顺序：先会问为什么，再去追求怎么拍得更漂亮。

先写下来，再执行；先有判断，再让工具放大判断。

这是导演的基本功。

继续练。
