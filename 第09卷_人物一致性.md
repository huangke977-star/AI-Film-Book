# 第 09 卷：人物一致性

## 学习目标

学完本卷后，你应该能做到：

- 理解人物一致性为什么是 AI 视频中最常见、也最影响可信度的问题。
- 知道 `same character` 不是魔法，不能单独解决变脸、变年龄、变服装。
- 写出一份可用于多镜头生成的 Character Bible。
- 设计参考图策略，知道什么时候用文生视频，什么时候用图生视频。
- 在分镜表中记录角色状态、服装发型、连续性备注和参考图路径。
- 用失败样本复盘人物漂移原因，并决定是改提示词、改参考图、改动作还是改镜头。
- 理解商业项目里肖像、授权、真人相似性和客户确认的风险。

人物一致性不是一个按钮，而是一套文件、素材和复盘流程。

## 本卷导读：不要把 same character 当魔法

很多新手会在提示词里写：

```text
same character
```

然后期待模型自动保持同一个人。结果常常是：

- 第一镜头 25 岁，第二镜头 35 岁。
- 发型从短发变长发。
- 绿色外套变成黑色西装。
- 脸型、气质、肤色都变了。
- 口播人物越生成越像另一个人。

原因很简单：模型需要具体约束。你不能只告诉它“同一个人”，还要告诉它“这个人是谁、长什么样、穿什么、可以怎么动、不能怎么变”。

本卷的核心流程是：

```text
Character Bible
-> 参考图
-> 分镜连续性字段
-> 单镜头任务卡
-> 生成后对比
-> 失败样本复盘
```

## 1. 人物一致性的本质

人物一致性不只是“脸一样”。它包含多个维度。

| 维度 | 说明 | 常见失败 |
| --- | --- | --- |
| 脸部 | 五官、脸型、年龄、肤色 | 变脸、变年龄 |
| 发型 | 长短、颜色、刘海、发量 | 发型换掉 |
| 服装 | 颜色、款式、材质 | 外套变色或换衣 |
| 身材 | 高矮、胖瘦、比例 | 体型漂移 |
| 气质 | 冷静、活泼、专业、神秘 | 表情和气质不统一 |
| 动作边界 | 能做什么动作 | 大动作导致变形 |
| 场景关系 | 人物和背景是否统一 | 人物像贴上去 |

如果你只控制脸，不控制服装和气质，观众仍然会觉得“不是同一个人”。

## 2. 为什么 AI 视频容易变脸

AI 视频生成本质上是在连续帧里预测画面。只要输入不够稳定，模型就可能在运动中重新解释人物。

常见原因：

- 提示词太短。
- 没有参考图。
- 每个镜头重新描述人物，描述不一致。
- 动作幅度太大。
- 镜头太长。
- 机位变化太剧烈。
- 光线和服装变化太大。
- 同时要求人物说话、转身、挥手、走路。

人物一致性的难点不是单帧好看，而是跨镜头稳定。

## 3. Character Bible 写法

Character Bible 可以翻译为“角色圣经”。它是角色的固定设定文件。

建议文件名：

```text
character_bible.md
```

### 3.2 示例

```markdown
# Character Bible

## 角色编号

CHAR_HOST_001

## 角色用途

AI 视频教程虚拟口播主持人。

## 基础身份

- 年龄范围：25 到 30 岁。
- 性别表达：年轻男性。
- 职业气质：冷静、专业、像创作者而不是销售员。
- 角色关键词：专注、可信、克制、技术感。

## 脸部特征

- 脸型：偏瘦的椭圆脸。
- 眼睛：自然黑色眼睛。
- 鼻子：自然，不夸张。
- 嘴：正常比例。
- 肤色：自然亚洲肤色。
- 年龄感：不要超过 30 岁。

## 发型

- 长度：短黑发。
- 颜色：黑色。
- 造型：干净，略微蓬松。
- 禁止变化：不要长发，不要染发。

## 服装

- 上衣：灰绿色连帽衫。
- 外套：无。
- 固定颜色：灰绿色。
- 禁止变化：不要西装，不要白大褂，不要夸张潮牌。

## 表情范围

- 常用表情：冷静、轻微微笑、专注。
- 禁止表情：夸张惊讶、过度兴奋、销售式笑容。

## 动作范围

- 允许动作：轻微点头、眼神变化、轻微呼吸。
- 高风险动作：大幅挥手、转身、快速走动。
- 禁止动作：复杂手势、多人互动。
```

## 4. 参考图策略

人物一致性通常需要参考图。

### 4.1 参考图类型

| 类型 | 用途 | 备注 |
| --- | --- | --- |
| 正面头像 | 固定脸部 | 口播人物必备 |
| 半身图 | 固定服装和上半身比例 | 短视频常用 |
| 全身图 | 固定体型和服装 | 剧情人物有用 |
| 侧面图 | 固定侧脸和转头 | 难度更高 |
| 表情图 | 固定情绪范围 | 适合虚拟角色 |

### 4.2 命名方式

```text
REF_character_host_v01_front.png
REF_character_host_v01_half_body.png
REF_character_host_v01_side.png
REF_character_host_v02_front.png
```

命名要能看出：

- 类型。
- 角色。
- 版本。
- 角度。

### 4.3 参考图不是越多越好

如果参考图之间差异很大，模型会更混乱。新手建议：

```text
先用 1 张稳定正面图
-> 再增加半身图
-> 再测试侧面和全身
```

不要一开始丢十张风格不同的图。

## 5. 服装、发型、道具约束

观众判断“是不是同一个人”时，不只看脸。服装和发型也很重要。

### 5.1 固定项

建议固定：

- 发色。
- 发型长度。
- 上衣颜色。
- 外套款式。
- 标志性道具。
- 口播气质。

### 5.2 可变化项

可以变化：

- 表情强弱。
- 轻微姿态。
- 背景。
- 景别。
- 光线强弱。

### 5.3 禁止项

禁止：

- 年龄变化。
- 服装大换。
- 发型大换。
- 夸张表情。
- 大幅转身。
- 手部复杂动作。

这些规则要写进 `character_bible.md`，也要写进单镜头任务卡。

## 6. 表情、动作和年龄漂移

人物漂移常常发生在动作变化时。

| 动作 | 风险 | 建议 |
| --- | --- | --- |
| 轻微呼吸 | 低 | 可以使用 |
| 眼神变化 | 低 | 可以使用 |
| 轻微点头 | 中 | 控制幅度 |
| 微笑 | 中 | 不要夸张 |
| 抬手 | 中到高 | 手不要占主体 |
| 大幅挥手 | 高 | 拆镜头或不用 |
| 转身 | 高 | 容易变脸 |
| 走路 | 高 | 对全身一致性要求高 |

年龄漂移常见于：

- 表情过度。
- 光线变化太大。
- 脸部角度变化太大。
- 参考图和提示词不一致。

解决方法：

```text
减少动作
-> 缩短时长
-> 固定机位
-> 强化年龄和脸部特征
-> 使用参考图
```

## 7. 多镜头人物连续性

多镜头时，人物一致性要写进分镜表。

### 7.1 连续性表

```markdown
| 镜头 | 景别 | 角色状态 | 服装/发型 | 动作 | 参考图 | 连续性备注 |
| --- | --- | --- | --- | --- | --- | --- |
| S01 | 中近景 | 面向镜头 | 灰绿色连帽衫、短黑发 | 轻微呼吸 | REF_character_host_v01_front.png | 建立人物 |
| S02 | 侧后方中景 | 看电脑屏幕 | 同 S01 | 手放键盘 | REF_character_host_v01_half_body.png | 不改变服装 |
| S03 | 近景 | 看向屏幕 | 同 S01 | 眼神变化 | REF_character_host_v01_front.png | 保持年龄 |
```

### 7.2 连续性顺序

建议先生成：

```text
人物正面口播镜头
-> 人物侧后方工作镜头
-> 人物细节镜头
-> 背景切换镜头
```

先确定人物，再扩展场景。

## 8. 口播人物和剧情人物的不同控制方法

### 8.1 口播人物

特点：

- 面向镜头。
- 动作小。
- 表情稳定。
- 主要任务是讲清信息。

控制策略：

- 中近景。
- 固定机位。
- 短镜头。
- 参考图稳定。
- 字幕后期添加。

### 8.2 剧情人物

特点：

- 有动作。
- 有场景变化。
- 可能有多人互动。
- 需要更强连续性。

控制策略：

- 先写角色圣经。
- 用更多参考图。
- 控制动作幅度。
- 用剪辑拆动作。
- 每个镜头记录服装和状态。

口播人物更适合新手练习。剧情人物要等你能稳定控制角色后再做。

## 9. 失败样本复盘

人物失败要记录具体类型。

```markdown
| 镜头 | 文件 | 问题 | 类型 | 可能原因 | 下一步 |
| --- | --- | --- | --- | --- | --- |
| S01 | S01_V01.mp4 | 脸变老 | 年龄漂移 | 光线和表情过度 | 强化年龄，减少表情 |
| S02 | S02_V01.mp4 | 衣服变黑 | 服装漂移 | 服装描述不够固定 | 写入固定服装 |
| S03 | S03_V01.mp4 | 手指变形 | 动作问题 | 手部动作复杂 | 避开手部或改动作 |
```

### 9.1 给 Codex 的复盘指令

```text
请读取 character_bible.md、storyboard.md 和 iteration_log.md。
只复盘人物一致性问题。
请按脸部、年龄、发型、服装、动作、气质六类归因。
不要重写全部提示词，只输出需要修改的镜头、原因和修改建议。
```

### 9.2 修改下一轮

```text
请只修改 S02 的任务卡。
问题：服装从灰绿色连帽衫变成黑色外套，年龄看起来变大。
要求：
1. 保持 CHAR_HOST_001。
2. 固定短黑发和灰绿色连帽衫。
3. 动作改为轻微看向屏幕，不要转身。
4. 保持中近景和固定机位。
```

## 10. 商业项目中的授权和风险

人物一致性还涉及法律和商业风险。

### 10.1 不要随便仿真人

如果你生成的角色明显像某个真实人物，可能涉及肖像权、名誉权、商业代言误导等问题。商业项目尤其要谨慎。

### 10.2 客户项目要确认

客户项目中建议明确：

- 人物是否是真人。
- 是否有授权。
- 是否允许 AI 生成相似形象。
- 是否可以用于广告投放。
- 是否需要客户确认每轮角色图。

### 10.3 文件记录

建议保留：

```text
authorization_notes.md
client_approval_log.md
reference_source.md
```

本书不是法律意见，但制作流程必须有风险意识。

## 11. Ross 风格虚拟口播人物复盘

Ross 案例中可以观察到虚拟口播人物、背景切换和字幕信息。我们不能断言具体工具和隐藏参数，但可以学习控制思路。

合理策略：

| 环节 | 控制重点 |
| --- | --- |
| 虚拟人物 | 正面中近景，动作克制 |
| 背景切换 | 背景变化，人物设定不变 |
| 字幕 | 后期添加，避免模型生成乱码 |
| 成本说明 | 用口播和字幕完成，不强依赖画面 |
| 多模型提及 | 作为内容信息，不一定每个模型都生成画面 |

如果你要做类似视频，先稳定口播人物，再做背景和字幕。

## 12. 完整流程：创建一个虚拟口播主持人

下面用一个完整小项目演示人物一致性的实际操作。

项目目标：

```text
做一条 45 秒知识短视频，用虚拟主持人解释“AI 视频不是一句提示词，而是一套工作流”。
```

### 12.1 第一步：先写角色用途

不要一上来生成脸。先写角色用途：

```markdown
## 角色用途

这个人物不是剧情主角，而是知识型虚拟口播主持人。
他的任务是解释复杂工作流，所以要可信、克制、清楚。
他不需要夸张表演，不需要走路，不需要复杂手势。
```

用途会决定后面的所有限制。既然是口播主持人，就不该设计太多动作。

### 12.2 第二步：写最小 Character Bible

新手可以先写最小版本：

```markdown
# Character Bible

## 角色编号

CHAR_HOST_001

## 固定外观

年轻亚洲男性，25 到 30 岁，短黑发，椭圆脸，自然肤色，灰绿色连帽衫。

## 固定气质

冷静、专注、可信，像认真做项目的创作者。

## 允许动作

轻微呼吸、轻微点头、眼神变化、嘴部自然口播。

## 禁止变化

不要变老，不要换衣服，不要染发，不要夸张笑容，不要大幅挥手，不要转身走动。

## 参考图

references/REF_character_host_v01_front.png
```

最小版本也比一句 `same character` 稳定得多。

### 12.3 第三步：生成或选择参考图

参考图要求：

| 要求 | 说明 |
| --- | --- |
| 脸清楚 | 不要侧脸太多 |
| 服装清楚 | 至少看清上衣颜色 |
| 背景不抢戏 | 背景简单 |
| 风格匹配 | 和最终视频风格一致 |
| 权利清楚 | 不用未经授权真人照片 |

保存为：

```text
references/REF_character_host_v01_front.png
```

### 12.4 第四步：写 3 个测试镜头

不要直接做完整 45 秒。先测试 3 个镜头：

| 镜头 | 目的 | 画面 | 风险 |
| --- | --- | --- | --- |
| T01 | 测脸 | 正面中近景，固定机位 | 脸是否稳定 |
| T02 | 测口播 | 同一人物轻微说话 | 嘴型和表情 |
| T03 | 测场景变化 | 人物不变，背景换工作室 | 背景是否影响人物 |

只有这三个测试基本通过，才进入正式分镜。

### 12.5 第五步：进入正式分镜

正式分镜里，每个有人物的镜头都要写：

```text
角色编号：
角色状态：
服装/发型：
参考图路径：
连续性备注：
```

示例：

```markdown
| 镜头 | 角色编号 | 角色状态 | 服装/发型 | 参考图路径 | 连续性备注 |
| --- | --- | --- | --- | --- | --- |
| S01 | CHAR_HOST_001 | 面向镜头 | 灰绿色连帽衫、短黑发 | references/REF_character_host_v01_front.png | 建立主持人 |
| S02 | CHAR_HOST_001 | 看向屏幕 | 同 S01 | references/REF_character_host_v01_front.png | 不要转身 |
| S05 | CHAR_HOST_001 | 回到镜头 | 同 S01 | references/REF_character_host_v01_front.png | 和 S01 年龄一致 |
```

### 12.6 第六步：生成后做对比

对比时不要只看“像不像”。要逐项检查：

```markdown
| 镜头 | 脸 | 年龄 | 发型 | 服装 | 气质 | 动作 | 结论 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| S01 | 5 | 5 | 5 | 5 | 4 | 5 | 可用 |
| S02 | 4 | 3 | 5 | 4 | 4 | 3 | 部分可用 |
| S05 | 3 | 3 | 4 | 5 | 4 | 4 | 重生成 |
```

评分不需要绝对客观，它的作用是让你知道哪里不稳定。

## 13. Character Bible 转任务卡

角色圣经不是写完就放着，要转进每个镜头任务卡。

### 13.1 转换前

```markdown
## Character Bible 摘要

CHAR_HOST_001，年轻亚洲男性，短黑发，灰绿色连帽衫，冷静可信，动作克制。
```

### 13.2 转换到镜头任务卡

```markdown
# S01 人物口播任务卡

## 角色

CHAR_HOST_001。

## 固定人物描述

young Asian male presenter, short black hair, natural skin tone, gray-green hoodie, calm focused expression, trustworthy creator look

## 镜头目的

建立虚拟主持人的可信度，让观众愿意听后面的工作流解释。

## 景别

中近景。

## 动作

轻微呼吸，自然口播，眼神稳定看向镜头。

## 负面约束

age change, different clothing, long hair, exaggerated smile, large hand gesture, face morphing, extra fingers
```

### 13.3 转换原则

- Character Bible 管长期固定。
- 任务卡管当前镜头。
- 任务卡不要推翻 Character Bible。
- 如果镜头需要改变服装或发型，必须先修改角色设定，并记录原因。

## 14. 人物一致性评分表

生成后可以用 1 到 5 分评分。

| 维度 | 1 分 | 3 分 | 5 分 |
| --- | --- | --- | --- |
| 脸部 | 完全像别人 | 有轻微差异 | 明显同一人 |
| 年龄 | 年龄变化明显 | 稍微偏差 | 年龄稳定 |
| 发型 | 发型换了 | 小幅变化 | 完全一致 |
| 服装 | 换衣服 | 颜色或细节变化 | 一致 |
| 气质 | 角色感觉变了 | 基本接近 | 很稳定 |
| 动作 | 变形严重 | 有小问题 | 自然 |

### 14.1 使用规则

```text
总分 26-30：可进入 selected。
总分 20-25：selected_partial 或重生成。
总分低于 20：进入 rejected。
```

但不要机械迷信总分。如果脸部只有 1 分，即使其他维度高，也不适合做主镜头。

### 14.2 示例

```markdown
| 镜头 | 脸部 | 年龄 | 发型 | 服装 | 气质 | 动作 | 总分 | 结论 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| S01_V01 | 5 | 5 | 5 | 5 | 4 | 5 | 29 | selected |
| S02_V01 | 4 | 3 | 5 | 4 | 4 | 3 | 23 | selected_partial |
| S03_V01 | 2 | 2 | 4 | 5 | 3 | 4 | 20 | 不做主镜头 |
```

## 15. 多角色项目的额外规则

当项目里有多个角色时，要更严格。

### 15.1 每个角色独立编号

```text
CHAR_HOST_001
CHAR_CLIENT_001
CHAR_ENGINEER_001
```

不要用：

```text
man1, woman1, another person
```

### 15.2 避免角色混淆

多角色同框风险很高。新手建议：

- 先单人镜头。
- 用剪辑制造对话关系。
- 不要一开始做复杂多人互动。
- 每个角色服装颜色区分明显。

### 15.3 多角色分镜表

```markdown
| 镜头 | 角色 | 服装 | 位置 | 动作 | 连续性 |
| --- | --- | --- | --- | --- | --- |
| S01 | CHAR_HOST_001 | 灰绿色连帽衫 | 画面左侧 | 看镜头 | 主讲 |
| S02 | CHAR_CLIENT_001 | 深蓝外套 | 画面右侧 | 看屏幕 | 客户 |
```

如果模型总是混淆角色，就拆镜头。

## 16. 商业交付确认流程

商业项目中，人物一致性必须让客户确认。

### 16.1 确认节点

| 节点 | 客户确认内容 |
| --- | --- |
| 角色设定 | 年龄、职业气质、服装方向 |
| 参考图 | 是否接受这个人物外观 |
| 测试镜头 | 是否接受动态效果 |
| 第一批素材 | 是否进入正式生产 |
| 成片前 | 是否允许发布或投放 |

### 16.2 确认记录

```markdown
# Client Approval Log

| 日期 | 项目 | 确认内容 | 客户意见 | 结论 |
| --- | --- | --- | --- | --- |
| 2026-07-10 | AI 工作流短视频 | CHAR_HOST_001 正面参考图 | 接受，服装保持灰绿色 | 通过 |
```

不要只在聊天里口头确认。商业项目要留记录。

### 16.3 风险提示

如果客户要求“做得像某个明星”“像某个博主”“像某个真实员工但没有授权”，必须谨慎。你可以改成：

```text
参考某类气质，不复制具体真人外貌。
```

这样更安全。

## 18. 人物一致性提示词模板库

下面这些模板可以放进 `Prompt库/`，也可以直接复制到 Codex 对话里使用。

### 18.1 生成 Character Bible

```text
请根据以下项目 brief，生成一份 Character Bible。
```

```text
请读取 character_bible.md。
检查它是否足以支撑多镜头 AI 视频生成。
```

### 18.3 从角色圣经生成镜头任务卡

```text
请读取 character_bible.md、storyboard.md 和 style_guide.md。

```

### 18.4 复盘人物漂移

```text
请读取 iteration_log.md 和 character_bible.md。
只分析人物一致性问题。
按脸部、年龄、发型、服装、气质、动作六类归因。
请输出：镜头、问题、归因、下一轮只改一个变量的建议。
```

### 18.5 生成客户确认摘要

```text
请根据 character_bible.md 和参考图文件名，生成一份客户确认摘要。
内容包括：角色用途、固定外观、固定服装、允许变化、禁止变化、需要客户确认的问题。
```

如果你以后学习本卷，可以按 7 天练。

| Day 04 | 写 3 个测试镜头 | `storyboard_test.md` |

### 19.1 练习验收

完成后，你应该能回答：

```text
这个角色为什么存在？
他有哪些固定特征？
哪些动作不能做？
如果脸变了，我怎么归因？
如果服装变了，我改哪里？
如果客户要商用，我需要确认什么？
```

能回答这些问题，说明你已经从“提示词试运气”进入“角色管理”。

## 20. 常见错误

### 错误 1：只写 same character

修正：写具体脸部、发型、服装、年龄、气质和参考图。

### 错误 2：每个镜头重新描述人物

修正：统一引用 `character_bible.md`。

### 错误 3：动作太复杂

修正：先固定机位、小动作、短时长。

### 错误 4：参考图风格不一致

修正：保留少量稳定参考图，不混用不同风格。

### 错误 5：失败后只改模型

修正：先归因，再决定改提示词、参考图、动作或镜头。

## 21. 故障排查

### 脸不一样

处理：

- 补脸部特征。
- 使用正面参考图。
- 缩小动作幅度。
- 避免大角度转头。

### 年龄漂移

处理：

- 明确年龄范围。
- 避免过度皱纹、夸张表情。
- 保持光线一致。

### 服装变化

处理：

- 写固定服装。
- 服装颜色和款式重复出现在任务卡。
- 避免“fashionable outfit”这类宽泛词。

### 气质不统一

处理：

- 写角色关键词。
- 控制表情范围。
- 统一口播语气和画面风格。

### 手部变形

处理：

- 不让手成为主体。
- 避免复杂手势。
- 用剪辑避开问题帧。

## 22. 人物一致性检查清单

生成前：

- 是否有 Character Bible。
- 是否有稳定参考图。
- 是否写清发型和服装。
- 是否写清允许动作。
- 是否写清禁止漂移项。
- 是否在分镜表中记录角色状态。

生成后：

- 脸是否一致。
- 年龄是否一致。
- 发型是否一致。
- 服装是否一致。
- 气质是否一致。
- 动作是否自然。
- 是否需要重生成或局部使用。

## 23. 本卷总结

人物一致性是 AI 视频的系统问题，不是单条提示词问题。

稳定流程是：

```text
写 Character Bible
-> 准备参考图
-> 在分镜表记录连续性
-> 单镜头生成
-> 对比脸、发型、服装、气质
-> 记录失败样本
-> 再修改下一轮
```

如果你是新手，先练口播人物。不要一开始就做多人剧情、大幅动作和复杂场景切换。

最低可行控制法是：一个角色、一张正面参考图、一个固定服装、三个短镜头、每轮只改一个变量。你不需要第一天就解决所有一致性问题，只要能知道“这一轮为什么变脸，下一轮只改什么”，就已经在正确方向上。人物一致性是长期训练，不是一次提示词比赛。

不要越级挑战多人短剧。先让一个虚拟主持人在三个镜头里保持稳定，再扩展到五个镜头、两个场景、更多表情。每一次扩展都只增加一个难度，否则你很难判断问题来自角色、动作、场景还是模型。

真正的稳定来自记录。每次生成后都把人物变化写下来，下一轮才知道该收紧哪条规则，而不是凭感觉反复试。

当你能解释每一次漂移的原因，人物一致性就开始可控了。

先做到可控，再追求更复杂的表演。

这会让后面的短剧和商业项目更稳。

慢慢加难度。

稳住。
