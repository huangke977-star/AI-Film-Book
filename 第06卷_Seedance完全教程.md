# 第 06 卷：Seedance 完全教程

## 学习目标

学完本卷后，你应该能做到：

- 理解 Seedance 或同类视频生成模型在 AI 视频工作流中的位置。
- 区分文生视频、图生视频和多镜头生成的输入输出。
- 把第 04 卷的上下文文件和第 05 卷的生成循环，转成视频模型可以执行的单镜头任务。
- 判断一个视频生成失败，是因为镜头目的不清、动作太复杂、参考图不足、人物漂移，还是模型本身不适合。
- 写出一张包含镜头目的、主体、动作、景别、运镜、光线、风格、负面约束和复盘字段的任务卡。
- 用 Seedance、LibTV、Veo、Kling 或同类工具时，不被“画面好看”带偏，而是按项目目标筛选结果。

本卷不会写死具体按钮、价格、套餐、可用地区和隐藏参数。这些信息变化很快，实际操作时必须以当前平台页面和账号权限为准。本卷讲的是更稳定的东西：视频模型如何进入你的项目工作流。

## 本卷导读：从 LibTV 工作流走向视频模型判断

第 05 卷讲 LibTV 或同类平台在工作流里的位置。你已经知道一个稳定循环应该是：

```text
Codex 整理上下文
-> 生成单镜头提示词
-> 平台生成素材
-> 人类筛选
-> 记录失败
-> Codex 修改下一轮
```

第 06 卷继续往里走一步：当你面对 Seedance、Veo、Kling、Hailuo、Pika、PixVerse 等视频生成模型时，怎么判断它适合做什么，怎么写输入，怎么复盘输出。

官方 ByteDance Seed 页面和技术报告把 Seedance 放在视频生成模型的语境里讨论，强调视频生成质量、提示词跟随、多镜头叙事、运动表现和电影感等方向。对新手来说，不需要一开始追逐所有技术指标，只要先记住：

```text
视频生成模型不是成片机器，而是镜头素材生成器。
```

它能给你画面和运动，但不能自动替你完成导演判断、剪辑节奏、品牌策略和成本控制。

## 1. Seedance 在 AI 视频工作流中的位置

在本书里，Seedance 是“视频生成模型能力”的代表之一。你可以把它理解为第 05 卷生成环节里的一个具体引擎：

```text
Brief / Script / Storyboard / Style Guide
-> Codex 整理单镜头任务卡
-> Seedance 或同类视频模型生成素材
-> 下载到 outputs/raw/
-> selected / rejected 筛选
-> iteration_log.md 复盘
```

它和 LibTV 的关系可以这样理解：

| 层级 | 作用 | 例子 |
| --- | --- | --- |
| 项目管理层 | 管文件、上下文、提示词、复盘 | Codex |
| 工作流入口层 | 提交任务、管理结果、连接 Agent | LibTV 或同类平台 |
| 视频模型层 | 根据文本、图片或视频输入生成画面 | Seedance、Veo、Kling 等 |
| 后期层 | 剪辑、字幕、声音、调色、导出 | 剪映、Premiere、DaVinci |

有的平台会把工作流入口和模型能力包装在一起。你不必纠结它到底属于哪一层。真正重要的是：输入是什么，输出是什么，失败如何复盘。

## 2. 视频生成模型解决什么问题

视频生成模型主要解决“从描述或参考素材生成动态画面”的问题。

它擅长：

- 生成一个短镜头的视觉素材。
- 把一张参考图变成有轻微运动的视频。
- 生成环境镜头、氛围镜头、产品镜头、人物口播辅助画面。
- 根据提示词表现景别、光线、风格、情绪和部分运镜。
- 快速试出多个画面方向，帮助创作者选择。

比如你要做一条“AI 视频不是一句提示词，而是一套工作流”的短视频。视频模型可以帮你生成：

- 夜晚工作室里的创作者。
- 屏幕前的项目文件夹。
- 抽象生成界面。
- 城市夜景和未来感背景。
- 多版本素材对比画面。

但这些素材是否要进入成片，仍然要由人判断。

## 3. 视频生成模型不解决什么问题

新手最容易误解的一点是：画面越好，不代表视频越好。

视频生成模型通常不解决：

- 选题是否值得做。
- 观众为什么要看。
- 镜头之间是否有逻辑。
- 口播信息是否清楚。
- 人物是否跨镜头稳定。
- 字幕是否清晰可读。
- 成片节奏是否舒服。
- 商业项目是否符合授权要求。

它也不能保证每次生成都稳定。你可能会遇到：

- 手指变形。
- 人物年龄漂移。
- 背景突然换风格。
- 镜头运动过猛。
- 画面出现乱码文字。
- 第一秒很好，后三秒崩坏。
- 参考图主体被模型误解。

所以，本卷的核心不是“怎样写一条神奇提示词”，而是“怎样建立可复盘的生成流程”。

## 4. 文生视频工作流

文生视频是用文字直接生成视频。

适合：

- 环境镜头。
- 抽象概念镜头。
- 不要求固定人物的镜头。
- 风格测试。
- 开场氛围镜头。

不适合：

- 要求严格人物一致的连续剧情。
- 要求精确产品结构的商业展示。
- 要求清晰中文文字的界面或招牌。
- 复杂动作和多人交互。

### 4.1 文生视频基础流程

```text
选择 storyboard 里的一个镜头
-> 确认镜头目的
-> 写主体、场景、动作、景别、运镜、光线、风格
-> 加负面约束
-> 生成 2 个版本
-> 下载到 outputs/raw/
-> 写入 iteration_log.md
```

### 4.2 弱提示词示例

```text
做一个很酷的 AI 视频工作室画面，电影感，科技感，适合抖音。
```

问题：

- 没有主体。
- 没有动作。
- 没有景别。
- 没有运镜。
- 没有时长。
- 没有失败风险。
- “很酷”“电影感”“科技感”都太空。

### 4.3 改进后的任务卡

```markdown
# S02 文生视频任务卡

## 镜头目的

让观众看到 AI 视频创作是一个项目文件夹，而不是一条临时聊天记录。

## 输入类型

文生视频。

## 主体

夜晚工作室里的电脑桌面和显示器。

## 场景

桌面整洁，有键盘、鼠标、笔记本、台灯。显示器上是抽象项目文件夹界面，不要求可读文字。

## 动作

镜头从桌面缓慢推近到显示器，文件夹界面轻微变化。

## 景别

中近景到屏幕特写。

## 运镜

very slow push in，只使用一种主要运镜。

## 光线

屏幕冷光和台灯暖光形成冷暖对比。

## 风格

真实纪录片质感，低饱和，浅景深，轻微胶片颗粒。

## Negative Prompt

readable Chinese text, distorted keyboard, extra hands, cartoon, logo, watermark, cluttered desktop

## 参数建议

9:16，3 到 5 秒，动作克制。

## 可能失败点

屏幕文字乱码，键盘变形，桌面过乱，镜头运动过快。
```

这个版本不是因为词更多就更好，而是因为它把生成任务拆清楚了。

## 5. 图生视频工作流

图生视频是用一张或多张参考图作为输入，再让模型生成运动。

适合：

- 固定人物外观。
- 固定产品外观。
- 固定场景风格。
- 把概念图变成动态镜头。
- 做角色开场、封面延展、产品展示。

风险：

- 动作太大时参考图容易崩。
- 人物转头或走动会增加变形概率。
- 参考图本身不清楚，会把问题带进视频。
- 如果参考图风格和提示词冲突，输出会摇摆。

### 5.1 图生视频前要检查参考图

| 项目 | 检查问题 |
| --- | --- |
| 主体清晰 | 人物或产品是否清楚 |
| 角度明确 | 正面、侧面、半身还是全身 |
| 背景干净 | 背景是否干扰主体 |
| 风格一致 | 是否符合 style_guide |
| 权利清楚 | 是否有使用权 |
| 文件命名 | 是否能看出用途和版本 |

### 5.2 图生视频任务卡

```markdown
# S01 图生视频任务卡

## 参考图

references/REF_character_host_v01_front.png

## 镜头目的

建立虚拟口播人物的第一印象。

## 运动要求

人物保持正面，轻微呼吸，眼神自然变化，不做复杂手势。

## 镜头

中近景，固定机位。

## 光线

柔和主光，轻微轮廓光，背景低对比。

## 风格

真实口播，克制，适合知识类短视频。

## 负面约束

face morphing, age change, clothing change, exaggerated gesture, extra fingers, unreadable text
```

图生视频的关键不是“让图动起来”，而是让图在可控范围里动起来。

## 6. 单镜头生成任务卡

从第 06 卷开始，你应该养成一个习惯：每个镜头都有任务卡。

任务卡模板：

```markdown
# SXX 生成任务卡

## 镜头目的

## 输入类型

文生视频 / 图生视频 / 视频生视频

## 上下文来源

- brief.md：
- storyboard.md：
- style_guide.md：
- character_bible.md：

## 主体

## 场景

## 动作

## 景别

## 机位

## 运镜

## 光线

## 风格

## 中文提示词

## English Prompt

## Negative Prompt

## 参数建议

## 可能失败点

## 生成后检查
```

任务卡让你知道每次生成是在测试什么。如果没有任务卡，失败后你只能说“不好看”。有了任务卡，你可以说：

```text
这个镜头的目的成立，但动作幅度过大，人物脸部漂移。
下一轮不改风格，只缩小动作，并把运镜改成固定机位。
```

这就是可复盘。

## 7. 多镜头生成和连续性

很多 AI 视频失败不是因为单镜头差，而是因为镜头之间没有连续性。

### 7.1 多镜头生成顺序

不要按时间线一次性全生成。建议顺序：

```text
关键人物镜头
-> 关键场景镜头
-> 解释性工作流镜头
-> 过渡镜头
-> 结尾镜头
```

原因：

- 人物镜头最容易失败，要先验证。
- 场景风格决定全片质感，要早确定。
- 过渡镜头可以最后补。
- 不要把预算花在不重要镜头上。

### 7.2 多镜头连续性表

```markdown
| 镜头 | 主体 | 场景 | 风格 | 连续性要求 | 可变化项 |
| --- | --- | --- | --- | --- | --- |
| S01 | 虚拟口播人物 | 城市夜景 | 真实电影感 | 服装、年龄、发型固定 | 背景灯光 |
| S02 | 项目文件夹 | 工作室 | 同一色调 | 冷暖对比保持 | 屏幕内容抽象 |
| S03 | 生成过程 | 抽象界面 | 科技但克制 | 不出现品牌 Logo | 缩略图变化 |
```

### 7.3 先锁定固定元素

生成多镜头前，先写：

```text
固定元素：
- 人物年龄、发型、服装。
- 画幅 9:16。
- 低饱和真实质感。
- 字幕后期添加。

可变化元素：
- 背景。
- 镜头景别。
- 屏幕里的抽象界面。

禁止漂移项：
- 人物变年龄。
- 从写实变卡通。
- 出现可读乱码文字。
- 背景变成无关场景。
```

## 8. 运镜控制

视频模型能理解一些运镜描述，但新手不要贪多。

### 8.1 常用运镜

| 运镜 | 适合场景 | AI 风险 |
| --- | --- | --- |
| 固定机位 | 口播、产品、界面 | 画面可能显得静 |
| 慢推 | 强调重点、进入状态 | 推太快会变形 |
| 慢拉 | 展示环境 | 主体可能变小 |
| 横移 | 展示空间关系 | 背景容易拉扯 |
| 跟拍 | 人物移动 | 动作复杂时容易崩 |
| 环绕 | 展示主体立体感 | 高风险，容易变形 |

### 8.2 新手原则

```text
一个镜头只写一种主要运镜。
```

不要这样写：

```text
镜头推近、横移、环绕人物、快速变焦，同时背景切换。
```

改成：

```text
固定机位，中近景，人物轻微呼吸，背景保持稳定。
```

AI 视频里，克制经常比炫技更稳定。

## 9. 动作幅度和失败风险

动作越大，失败概率越高。

| 动作 | 风险 | 新手建议 |
| --- | --- | --- |
| 轻微呼吸 | 低 | 可以使用 |
| 眼神变化 | 低 | 可以使用 |
| 轻微转头 | 中 | 控制角度 |
| 手部小动作 | 中 | 不要强调手指 |
| 快速跑动 | 高 | 慎用 |
| 多人互动 | 高 | 拆镜头 |
| 转身走远 | 高 | 容易变脸和换衣 |
| 拿复杂道具 | 高 | 用后期或实拍替代 |

写提示词时，不要只问“我想不想要这个动作”，还要问：

```text
这个动作是否值得承担失败风险？
```

如果不值得，就用剪辑解决。

## 10. 人物一致性初步

人物一致性会在第 09 卷展开。本卷先给你一个最低规则：

```text
不要只写 same character。
```

你需要写具体特征：

```text
same young Asian male presenter, short black hair, gray-green hoodie, calm focused expression, facing camera, realistic talking-head video style
```

但这仍然不够。更稳的做法是：

- 写 `character_bible.md`。
- 准备参考图。
- 每个镜头引用同一份人物描述。
- 避免大动作和大角度转身。
- 每轮生成后对比脸、年龄、发型、服装和气质。

人物一致性不是一句提示词解决的，而是项目文件和生成复盘共同控制的。

## 11. 与 LibTV、Veo、Kling 等工具的关系

新手容易把工具当成互相替代。更好的理解是：不同工具对应不同任务能力。

| 工具或能力 | 可以怎么理解 | 使用提醒 |
| --- | --- | --- |
| LibTV | 工作流入口或生成平台 | 关注任务管理和 Agent 接入 |
| Seedance | 视频生成模型能力之一 | 关注镜头质量、运动和提示词跟随 |
| Veo | 高质量视频生成方向 | 注意可用地区和账号权限 |
| Kling | 常见图生视频/生视频工具方向 | 关注中文生态和短视频工作流 |
| Hailuo / PixVerse / Pika | 快速试错或特定风格生成 | 关注成本和稳定性 |

不要问：

```text
哪个工具永远最好？
```

应该问：

```text
这个镜头需要什么输入？
这个工具是否擅长这种镜头？
失败后能否复盘？
成本是否可控？
```

工具会变，判断方式更稳定。

## 12. Ross 风格短视频中的生成策略

Ross 案例标题强调“全程用 Codex 和 LibTV 完成”，视频中可以观察到电影感画面、虚拟口播人物、背景切换、成本说明、多模型选择和黄色大字幕。我们不能知道作者的隐藏参数，但可以复原一种合理工作流。

### 12.1 合理拆分

| 镜头类别 | 生成策略 | 复盘重点 |
| --- | --- | --- |
| 电影感开场 | 文生视频或图生视频 | 画面是否抓人 |
| 虚拟口播人物 | 参考图 + 克制动作 | 人物是否稳定 |
| 工作流展示 | 抽象界面 + 后期字幕 | 不要求模型生成可读字 |
| 成本说明 | 口播 + 字幕 + 图形 | 信息是否清楚 |
| 背景切换 | 多个短镜头 | 风格是否统一 |
| 结语 | 回到人物或手机画面 | 是否强化观点 |

### 12.2 生成顺序

```text
先测人物口播稳定性
-> 再测开场电影感画面
-> 再生成工作流抽象镜头
-> 最后补背景切换和过渡镜头
```

如果人物口播一直不稳定，不要继续生成 20 个背景镜头。先解决主镜头。

## 13. 生成复盘表

生成后不要只保存文件，要写复盘。

```markdown
| 镜头 | 文件 | 输入类型 | 是否符合目的 | 主要问题 | 问题归因 | 下一轮动作 |
| --- | --- | --- | --- | --- | --- | --- |
| S01 | S01_V01_raw.mp4 | 文生视频 | 部分符合 | 背景太科幻 | 风格漂移 | 降低赛博感，强调真实城市 |
| S01 | S01_V02_raw.mp4 | 文生视频 | 不符合 | 人物脸变形 | 动作过大 | 改固定机位，减少表情变化 |
| S02 | S02_V01_raw.mp4 | 图生视频 | 符合 | 轻微手部问题 | 模型细节 | 可剪辑，避开手部 |
```

### 13.1 归因分类

| 归因 | 说明 | 处理 |
| --- | --- | --- |
| 镜头目的问题 | 不知道镜头要表达什么 | 改 storyboard |
| 提示词问题 | 描述不清或约束冲突 | 改任务卡 |
| 动作问题 | 动作太大或太复杂 | 缩小动作 |
| 参考图问题 | 图像不稳定或权利不清 | 换参考图 |
| 模型问题 | 当前模型不擅长 | 换工具或生成方式 |
| 后期问题 | 素材可用但需剪辑处理 | 留给剪辑 |

### 13.2 让 Codex 修改下一轮

```text
请读取 storyboard.md、style_guide.md、prompts/shot_01.md 和 iteration_log.md。
只修改 S01 的生成任务卡。
上一轮失败原因是：背景太科幻、人物脸部变形、动作幅度过大。
请保持镜头目的不变，把运镜改成固定机位，把人物动作改成轻微呼吸和眼神变化。
不要改 S02-S05。
```

### 13.3 30 秒生成案例：从工作流观点到素材清单

下面用一个 30 秒短视频做完整示例。主题仍然延续第一阶段：

```text
AI 视频不是一句提示词，而是一套 Codex + 生成模型 + 剪辑复盘的工作流。
```

目标不是马上做出大片，而是让你学会怎样安排生成顺序。

#### 项目镜头表

| 镜头 | 时长 | 画面 | 输入类型 | 生成重点 | 风险 |
| --- | --- | --- | --- | --- | --- |
| S01 | 4s | 夜晚城市里的创作者中近景 | 文生视频或图生视频 | 电影感开场 | 人物漂移 |
| S02 | 4s | 工作室电脑打开项目文件夹 | 文生视频 | 项目化工作流 | 屏幕文字乱码 |
| S03 | 5s | 抽象分镜表和提示词文件切换 | 文生视频或后期录屏 | Codex 产物 | 界面混乱 |
| S04 | 5s | 视频生成过程和素材缩略图 | 文生视频 | 生成模型环节 | 出现假品牌 |
| S05 | 5s | 多版本素材对比，选中一个版本 | 后期合成或文生视频 | 筛选意识 | 信息不清 |
| S06 | 5s | 剪辑时间线和字幕叠加 | 文生视频或录屏 | 成片闭环 | 字幕乱码 |
| S07 | 2s | 手机播放最终短视频 | 文生视频 | 结果回收 | 画面太满 |

#### 批次安排

第一批只生成 S01 和 S02。原因是这两个镜头决定观众第一眼是否相信这个视频有质感。如果 S01 的人物和 S02 的工作室风格都不稳定，后面镜头先不要做。

第二批生成 S03 和 S04。它们是工作流解释镜头，重点不是出现真实可读文字，而是让观众感到“这里有文件、有任务、有生成过程”。如果模型生成了乱码文字，不要强修，改成抽象界面，并把真实说明放到剪辑字幕里。

第三批生成 S05、S06、S07。它们更接近后期表达，很多内容可以通过剪辑软件完成，不一定全部交给视频模型。

#### 批次记录模板

```markdown
## 第 1 批：风格和主视觉

| 镜头 | 版本 | 结果 | 判断 |
| --- | --- | --- | --- |
| S01 | V01 | 画面好，但人物年龄偏大 | 不直接用，改人物描述 |
| S01 | V02 | 人物稳定，背景略普通 | 可作为 selected |
| S02 | V01 | 桌面好看，屏幕乱码 | 可剪辑，后期遮盖 |
| S02 | V02 | 场景变成网吧 | rejected |
```

#### 第二轮修改示例

```text
请只修改 S01 和 S02 的任务卡。
S01：保留夜晚城市和中近景，但人物年龄要更稳定，动作只保留轻微呼吸，不要转身。
S02：保留工作室桌面，但屏幕内容改成抽象文件块，不要求可读文字。
不要改变全片低饱和真实纪录片风格。
```

这个案例的重点是：生成不是一次性完成，而是一批一批锁定变量。第一批锁风格，第二批锁信息表达，第三批补过渡和收尾。

### 13.4 什么时候该换工具或换方案

很多人会在一个失败镜头上连续生成十几次。更专业的做法是设定停止条件。

建议规则：

| 情况 | 动作 |
| --- | --- |
| 同一镜头连续 2 轮都是同类失败 | 先改任务卡，不继续烧版本 |
| 人物每次都漂移 | 改用参考图或缩短镜头 |
| 屏幕文字一直乱码 | 放弃模型文字，后期加字 |
| 复杂动作一直变形 | 拆成两个镜头 |
| 当前模型风格不匹配 | 换工具或换生成方式 |
| 镜头目的本身不清 | 回到 storyboard 重写 |

换工具不是失败，继续盲目重试才是失败。视频生成模型只是工作流中的一环，你可以用剪辑、参考图、改分镜、换模型、补字幕来解决问题。

### 13.5 把复盘结果沉淀成资产

每次生成结束后，至少留下三类资产：

```text
prompts/shot_xx.md
outputs/selected/
iteration_log.md
```

如果一个镜头失败，也不要删除。把它放进 `outputs/rejected/`，并记录原因。以后你会发现，失败样本比成功样本更能训练你的判断力。

一个成熟项目的资产结构可能是：

```text
project/
├── prompts/
│   ├── shot_01.md
│   └── shot_02.md
├── outputs/
│   ├── raw/
│   ├── selected/
│   └── rejected/
├── references/
│   ├── REF_character_host_v01_front.png
│   └── REF_scene_workspace_v01_wide.png
├── iteration_log.md
└── cost_log.md
```

当你能从这些文件回看每一次决策，说明你已经不再是“碰运气生成”，而是在做项目化创作。

## 14. 常见错误

### 错误 1：把整条视频写进一条提示词

后果：模型不知道哪个画面更重要，结果像混合梦境。

修正：拆成单镜头任务卡。

### 错误 2：同一镜头动作太多

后果：前半秒还好，后面开始变形。

修正：一个镜头只保留一个核心动作。

### 错误 3：要求模型生成清晰中文

后果：屏幕、招牌、字幕乱码。

修正：画面里写抽象界面，真实文字后期加。

### 错误 4：只看画面好看

后果：素材进了剪辑后不服务叙事。

修正：按镜头目的筛选。

### 错误 5：无限重试同一个失败镜头

后果：成本上升，问题没有变清楚。

修正：先归因，再决定重试、换输入、换模型或改分镜。

## 15. 故障排查

### 画面太散

可能原因：

- 主体不明确。
- 场景元素太多。
- 镜头目的不清。

处理：

- 明确一个主体。
- 删除无关道具。
- 在任务卡第一行写清镜头目的。

### 运镜失控

可能原因：

- 同时写了多个运镜。
- 运镜速度太快。
- 主体动作和镜头运动都很复杂。

处理：

- 固定机位或慢推。
- 保留一个主要运动。
- 用剪辑完成复杂节奏。

### 人物漂移

可能原因：

- 没有角色圣经。
- 没有参考图。
- 动作太大。
- 镜头太长。

处理：

- 使用 `character_bible.md`。
- 用图生视频或稳定参考图。
- 缩短镜头。
- 减少转身和复杂手势。

### 背景漂移

可能原因：

- 风格约束太弱。
- 背景描述和主体描述冲突。
- 每个镜头都重新写了不同风格。

处理：

- 引用同一份 `style_guide.md`。
- 写固定元素和可变化元素。
- 把背景变化写成叙事需要，而不是随机变化。

### 成本失控

可能原因：

- 一次生成太多版本。
- 没有筛选标准。
- 对不重要镜头反复重试。

处理：

- 第一轮每镜头 2 个版本。
- 先生成关键镜头。
- 写 `cost_log.md`。

## 16. 参数速查

视频生成前检查：

| 项目 | 建议 |
| --- | --- |
| 输入类型 | 先明确文生视频还是图生视频 |
| 镜头时长 | 新手先 3 到 5 秒 |
| 画幅 | 抖音优先 9:16 |
| 主体 | 一个镜头一个中心主体 |
| 动作 | 先小动作，后复杂动作 |
| 运镜 | 一个镜头一种主要运镜 |
| 字幕 | 后期添加 |
| 参考图 | 人物或产品需要准备 |
| 版本数 | 第一轮 2 个 |
| 输出目录 | `outputs/raw/` |
| 复盘文件 | `iteration_log.md` |

## 17. 本卷总结

Seedance 或同类视频模型让 AI 视频创作者可以快速得到动态画面，但真正的能力不在“一次生成”，而在“能不能让每次生成变成下一轮改进的依据”。

记住本卷的主线：

```text
不要追逐神奇模型
-> 先拆单镜头
-> 写任务卡
-> 控制动作和运镜
-> 生成少量版本
-> 复盘失败原因
-> 再进入下一轮
```

如果你能把一个镜头从任务卡、生成、下载、筛选、复盘、修改跑通，就已经掌握了视频生成模型的核心学习方法。

## 18. 课后练习

### 练习 1：改写弱提示词

把下面提示词改成任务卡：

```text
做一个很有电影感的 AI 创作者视频，科技感，适合抖音。
```

要求包含：

- 镜头目的。
- 主体。
- 场景。
- 动作。
- 景别。
- 运镜。
- 光线。
- 负面约束。

### 练习 2：设计图生视频任务

假设你有一张虚拟口播人物参考图，写一张 5 秒图生视频任务卡。

限制：

- 人物不能大幅转身。
- 不要求生成字幕。
- 只允许一种主要运镜。

### 练习 3：写生成复盘

创建一个复盘表：

```markdown
| 镜头 | 文件 | 输入类型 | 是否符合目的 | 主要问题 | 问题归因 | 下一轮动作 |
| --- | --- | --- | --- | --- | --- | --- |
```

填入两个假想失败样本。

### 练习 4：做 3 镜头生成计划

为一条 15 秒视频设计 3 个镜头：

```text
S01：开场吸引
S02：展示工作流
S03：结尾观点
```

每个镜头只写一个目的和一个主要动作。
