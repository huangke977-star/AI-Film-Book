# 学习路线

## 先说结论

如果你现在对 AI 视频完全不了解，不要从 18 卷目录开始硬读。正确顺序是先完成第一阶段：

```text
第01卷_AI视频基础
-> 第02卷_环境搭建
-> 第03卷_ChatGPT与Codex
-> 第04卷_ContextEngineering
-> 第05卷_LibTV完全教程
-> 14天入门训练营
-> Ross 案例复盘
```

第 06-18 卷是后续进阶内容，适合在你已经理解基础工作流后再读。你现在的目标不是马上学会所有工具，而是先明白一条 AI 视频从想法到成片到底经历哪些环节。

## 第一阶段：先学第 01-05 卷

第一阶段解决三个问题：

1. AI 视频到底是什么，不是什么。
2. 如何把创作过程放进一个可维护的项目文件夹。
3. 如何用 Codex 和 LibTV 或同类工具形成可复用工作流。

这五卷不是工具说明书，而是一条入门路径：

| 顺序 | 文件 | 作用 |
| --- | --- | --- |
| 1 | `第01卷_AI视频基础.md` | 建立全局地图，理解 AI 视频不是一句提示词出片 |
| 2 | `第02卷_环境搭建.md` | 建立项目目录、Git、素材命名和安全规则 |
| 3 | `第03卷_ChatGPT与Codex.md` | 理解聊天助手和项目 Agent 的分工 |
| 4 | `第04卷_ContextEngineering.md` | 把手写提示词升级成项目上下文工程 |
| 5 | `第05卷_LibTV完全教程.md` | 把 Codex 输出变成可生成、可复盘的视频素材 |

第一阶段完成后，你应该能看懂 Ross 那条“全程用 Codex 和 LibTV 完成”的视频为什么不是炫技，而是一个新的生产方式。

## 你不需要先懂什么

开始学习前，不需要先懂：

- 编程。
- 电影专业术语。
- 剪辑软件高级操作。
- 所有 AI 视频平台。
- API、MCP、自动化脚本。

你只需要先理解几个基本概念：

- `brief`：这个视频要做什么。
- `script`：视频要说什么。
- `storyboard`：每个镜头拍什么。
- `style guide`：画面风格如何统一。
- `prompt`：给模型的具体生成指令。
- `iteration log`：失败样本和修改记录。

这些概念会在第 01-05 卷里逐步展开。

## 第 1 周：建立基本认知

第一周不要急着生成视频。先把脑子里的地图搭起来。

建议顺序：

1. 阅读第 01 卷，理解 AI 视频制作的四层结构。
2. 阅读第 02 卷，建立一个空的视频项目文件夹。
3. 复制 `分镜模板/短视频分镜模板.md`，写一个 15 秒试验片的 5 个镜头。
4. 不生成视频，只检查每个镜头是否有目的、主体、场景和动作。

第一周的产出不是成片，而是：

```text
brief.md
storyboard.md
style_guide.md
prompts/
outputs/
iteration_log.md
```

只要你能把这些文件的作用说清楚，第一周就已经成功。

## 第 2 周：理解 Codex + LibTV 工作流

第二周再进入工具协作。

建议顺序：

1. 阅读第 03 卷，学会让 Codex 读取项目文件。
2. 阅读第 04 卷，理解为什么单条提示词不够。
3. 阅读第 05 卷，理解 LibTV 或同类平台在生成环节中的位置。
4. 按 `工作流/Codex_LibTV_端到端视频制作流程.md` 做一个 15 秒试验片方案。
5. 对照 `案例拆解/Ross_Codex_LibTV_抖音案例.md`，分析自己的方案缺了什么。

第二周的产出是：

```text
5 到 8 个镜头的分镜表
每个镜头一条结构化提示词
一份失败样本记录模板
一份准备投喂给 LibTV 或同类工具的生成清单
```

你可以选择在第二周真正生成素材，也可以先只完成方案。对完全新手来说，先完成方案更稳。

## 不建议一开始做的事

刚开始不要做这些事：

- 不要同时研究十几个视频模型。
- 不要一上来就追求 60 秒以上成片。
- 不要把所有镜头写进同一条长提示词。
- 不要让模型直接生成字幕里的中文文字。
- 不要把 API Key、Cookie、Token 写进 Markdown。
- 不要删除失败样本。
- 不要把 Ross 案例理解成一键生成成片。

最小目标是做出一个可复盘的 15 秒方案，而不是第一天就做出商业广告。

## 学完第一阶段后应该能做到什么

学完第 01-05 卷后，你应该能做到：

- 说清楚 AI 视频制作的基本流程。
- 建立一个标准项目目录。
- 用 Codex 生成 brief、脚本、分镜和提示词。
- 解释 prompt 和 context 的区别。
- 判断一个镜头失败是因为提示词、素材、模型、参数还是剪辑。
- 用 LibTV 或同类平台生成单镜头素材。
- 记录失败样本，并让 Codex 帮你修改下一轮提示词。
- 看懂 Ross 案例的制作逻辑。

如果这些能做到，再进入第 06 卷以后学习 Seedance、镜头语言、导演思维、人物一致性、配音、音乐、剪辑和商业变现。

## 后续阶段如何继续

第一阶段之后，后续学习分成三条线：

### 工具线

继续学习 Seedance、Kling、Veo、PixVerse、Hailuo、Pika、AI 绘图、AI 配音、AI 音乐和剪辑软件。

### 创作线

继续学习镜头语言、导演思维、人物一致性、视觉风格、节奏和叙事。

### 商业线

继续学习广告案例、短剧案例、n8n 自动化、MCP 工作流、报价、成本核算和商业交付。

不要急着三条线同时走。最推荐的顺序是：

```text
第一阶段基础工作流
-> 镜头语言和导演思维
-> 人物一致性
-> 剪辑和声音
-> 商业项目
```
