# 故障排查

## LibTV 生成循环问题

### prompt too broad

问题：提示词描述太宽，画面元素很多但没有重点。

处理：回到 `storyboard.md`，明确主体、动作、景别和镜头目的。

### model mismatch

问题：当前模型或生成方式不适合这个镜头。

处理：换生成方式，例如先文生图再图生视频，或换更适合该任务的模型。

### character drift

问题：人物脸型、年龄、服装或气质变化。

处理：补 `character_bible.md`，减少复杂动作，使用稳定参考图。

### background drift

问题：背景突然变成另一个场景，或风格断裂。

处理：强化 `style_guide.md` 中的固定元素和禁止漂移项。

### unreadable text

问题：屏幕文字、招牌、字幕乱码。

处理：生成时弱化文字，后期用剪辑软件添加真实文字。

### cost overrun

问题：一个镜头反复生成，成本持续上升。

处理：停止重试，先写入 `iteration_log.md` 归因，再决定改分镜、改提示词或换模型。

## 人物变脸

可能原因：

- 没有角色圣经。
- 每个镜头的人物描述不同。
- 运镜过于复杂。
- 镜头时长过长。

解决方法：

- 固定人物描述。
- 使用参考图。
- 减少环绕和大幅度运动。
- 把长镜头拆成多个短镜头。

## 手部错误

可能原因：

- 手部动作过细。
- 要求敲键盘、拿杯子、指向屏幕等复杂动作。

解决方法：

- 改成“手停在键盘附近轻微移动”。
- 避免手部特写。
- 用剪辑和音效暗示动作。

## 屏幕文字乱码

可能原因：

- 模型不擅长直接生成可读文字。

解决方法：

- 提示词中写“抽象界面”。
- 不要求生成具体中文。
- 后期用剪辑软件添加真实文字。

## 镜头运动失控

可能原因：

- 同时要求多种运镜。
- 使用“dynamic”“fast”“dramatic”等过强词。

解决方法：

- 每个镜头只写一种运动。
- 加入“slow, stable, subtle”。
- 必要时改成固定镜头。

## 风格不统一

可能原因：

- 没有统一 style guide。
- 每个镜头临时写提示词。
- 光线、色彩、场景不断变化。

解决方法：

- 建立 `style_guide.md`。
- 每个提示词都引用相同风格段落。
- 把色彩、光线、质感固定下来。

## 配音听起来像机器人

可能原因：

- 口播稿太书面。
- 句子太长，没有自然停顿。
- 选用的声音角色和视频气质不匹配。

解决方法：

- 先把书面稿改成短句口播稿。
- 在关键句前后增加停顿提示。
- 生成 2 到 3 个语速和情绪版本，在时间线上试听。

## 音乐盖住人声

可能原因：

- 选择了有歌词或旋律太强的音乐。
- 音乐音量过大。
- 转场音效放在了关键句上。

解决方法：

- 优先选择无歌词背景音乐。
- 人声出现时降低音乐音量。
- 删除不必要音效，只保留有画面动作依据的声音。
- 用手机外放完整听一遍。

## 字幕导出后消失

可能原因：

- 字幕是独立轨道但导出时没有包含。
- 软件导出设置没有选择烧录字幕或字幕轨。
- 视频版本和字幕文件版本不一致。

解决方法：

- 导出前确认字幕是烧录进视频还是单独 SRT。
- 导出后完整播放成片检查字幕。
- 如果交付 SRT，文件名要和视频版本一致。

## 导出后画质变差

可能原因：

- 分辨率或码率过低。
- 同一个文件反复压缩。
- 竖屏视频误用横屏导出设置。

解决方法：

- 按发布平台确认画幅和分辨率。
- 保留高质量母版，再分别导出平台版本。
- 文件名写清平台、分辨率和版本。

## 客户反复改方向

可能原因：

- Brief 没有确认。
- 修改范围没有写清。
- 客户把小修改和大方向修改混在一起。

解决方法：

- 回到 Brief 和报价范围。
- 把意见写进 `revision_log.md`。
- 区分字幕、节奏、个别镜头这类小修改，以及整体风格、卖点、脚本重写这类大修改。

## 短剧人物和场景不连续

可能原因：

- 没有人设表。
- 场景太多。
- 每集临时改设定。

解决方法：

- 建立 `series_bible.md`。
- 固定角色参考图和场景参考图。
- 试播集先控制在 2 到 3 个角色、1 到 2 个场景。

## n8n 工作流反复失败

可能原因：

- 字段命名不统一。
- 凭证过期。
- 节点输入为空。
- 没有失败记录和重试上限。

解决方法：

- 查看 execution 记录。
- 用最小样本测试失败节点。
- 统一字段名。
- 给高成本节点加人工确认。

## MCP 工具调用不可控

可能原因：

- 开放能力过多。
- 没有工具调用审计表。
- 输出目录不受限制。

解决方法：

- 先只开放必要 Resource 和低风险 Tool。
- 建立 `tool_audit_log.md`。
- 限制写入目录。
- 对高风险动作加人工确认。

## 商业服务不知道怎么报价

可能原因：

- 没有服务菜单。
- 交付物不清楚。
- 成本没有记录。

解决方法：

- 先写服务菜单，不急着写金额。
- 拆分时长、难度、修改、授权和交付物。
- 每个项目记录时间、工具、重试和沟通成本。
