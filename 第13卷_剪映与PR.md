# 第 13 卷：剪映与 PR

## 学习目标

学完本卷后，你应该能做到：

- 理解 AI 生成素材不等于成片，剪辑才是最后的导演环节。
- 知道剪映、Premiere Pro、DaVinci Resolve、Final Cut Pro 分别适合什么场景。
- 把 AI 生成的视频、图片、配音、音乐、音效、字幕整理进剪辑工程。
- 建立时间线轨道命名、版本管理和导出检查流程。
- 完成粗剪、精剪、字幕、混音、调色统一和封面准备。
- 处理 AI 素材常见瑕疵，而不是把每个问题都丢回生成模型。
- 为抖音、视频号、小红书、B 站、YouTube 等平台做通用导出判断。
- 为商业项目准备终审表和交付文件包。

本卷不把你变成专业剪辑师，但会让你知道：一条 AI 视频要怎样真正变成可以发布的作品。

## 本卷导读：剪辑是最后的导演环节

很多新手以为 AI 视频流程是：

```text
写提示词
-> 生成视频
-> 发布
```

真实流程更接近：

```text
写 Brief
-> 写脚本
-> 做分镜
-> 生成画面
-> 生成配音
-> 生成音乐
-> 剪辑
-> 字幕
-> 混音
-> 调色统一
-> 导出
-> 终审
-> 发布
```

剪辑不是最后的杂活。剪辑会重新决定：

- 哪个镜头保留。
- 哪句话删掉。
- 哪个节奏更适合平台。
- 哪个画面问题可以用字幕、裁切、转场或补镜头解决。
- 哪些素材不值得继续抢救。

AI 生成给你原材料，剪辑把它变成作品。

## 1. AI 生成不是成片

AI 生成素材经常有这些问题：

- 镜头时长不统一。
- 画面节奏不适合口播。
- 人物动作开头或结尾突兀。
- 某些帧有轻微变形。
- 字幕空间不够。
- 不同镜头色调不一致。
- 画面很好，但不服务脚本。

剪辑要做的不是把所有素材拼起来，而是筛选、重组和删减。

### 1.1 成片标准

一条可发布视频至少要满足：

| 标准 | 问题 |
| --- | --- |
| 信息清楚 | 观众是否知道你在说什么 |
| 节奏合理 | 是否拖沓或过快 |
| 音画同步 | 配音、字幕、画面是否对齐 |
| 风格统一 | 色彩、字体、音乐是否一致 |
| 技术合格 | 分辨率、画幅、音量、格式是否正确 |
| 授权清楚 | 配音、音乐、图片、素材来源是否记录 |

只要其中一项严重失败，画面再好也不稳。

## 2. 剪映、Premiere Pro、DaVinci Resolve、Final Cut Pro 的定位

你不需要一开始学所有软件。先知道它们适合什么。

| 工具 | 适合场景 | 优点 | 注意点 |
| --- | --- | --- | --- |
| 剪映 / CapCut | 中文短视频、快速字幕、社媒发布 | 上手快、模板多、字幕方便 | 专业工程管理和复杂交付不如专业 NLE |
| Premiere Pro | 商业剪辑、团队项目、Adobe 工作流 | 通用、插件多、字幕和导出体系成熟 | 学习成本更高 |
| DaVinci Resolve | 调色、专业后期、音频 Fairlight | 调色和后期能力强 | 新手界面压力大 |
| Final Cut Pro | Mac 用户、高效率剪辑 | 性能好、Mac 生态顺 | Windows 用户不适用 |

### 2.1 新手推荐

如果你完全零基础：

```text
短视频优先剪映
商业项目逐步学习 Premiere Pro
调色和专业后期以后再看 DaVinci Resolve
Mac 深度用户可以考虑 Final Cut Pro
```

不要因为别人说某个软件“更专业”，就第一天打开复杂工程界面把自己劝退。

### 2.2 本书如何处理软件差异

本卷讲的是通用剪辑流程，不写死按钮路径。因为不同软件版本、地区客户端和账号权限会变化。

你要学会的是：

```text
导入素材
-> 建立时间线
-> 放配音
-> 放画面
-> 做字幕
-> 加音乐音效
-> 调整色彩和音量
-> 导出
-> 终审
```

具体按钮在哪里，以当前软件为准。

## 3. 剪辑前文件整理

进入剪辑软件前，先整理文件。不要把所有素材堆在桌面。

推荐项目目录：

```text
project/
├── brief.md
├── script.md
├── storyboard.md
├── character_bible.md
├── style_guide.md
├── references/
├── outputs/
│   ├── raw/
│   ├── selected/
│   └── rejected/
├── audio/
│   ├── voice/
│   ├── music/
│   └── sfx/
├── subtitles/
├── edit/
│   ├── project_files/
│   ├── exports/
│   └── review/
└── delivery/
```

### 3.1 剪辑前素材清单

```markdown
| 类型 | 文件夹 | 检查项 |
| --- | --- | --- |
| 视频素材 | `outputs/selected/` | 是否都是可用镜头 |
| 参考图 | `references/` | 是否只作为参考，不误放进成片 |
| 配音 | `audio/voice/selected/` | 是否是最终或候选版本 |
| 音乐 | `audio/music/selected/` | 是否有授权记录 |
| 音效 | `audio/sfx/` | 是否有来源 |
| 字幕稿 | `subtitles/` | 是否和配音一致 |
| 封面图 | `delivery/cover/` | 是否留标题空间 |
```

整理文件的目的不是好看，而是防止后期找不到素材。

## 4. 工程目录和版本

每次重要修改都要保存版本。

### 4.1 工程命名

```text
EDIT_ai_workflow_v01_draft.prproj
EDIT_ai_workflow_v02_voice_locked.prproj
EDIT_ai_workflow_v03_subtitle.prproj
EDIT_ai_workflow_v04_color_mix.prproj
EDIT_ai_workflow_v05_final.prproj
```

剪映项目也可以用同样逻辑命名：

```text
AI_workflow_v01_粗剪
AI_workflow_v02_配音锁定
AI_workflow_v03_字幕完成
AI_workflow_v04_终审
```

### 4.2 版本日志

```markdown
| 版本 | 日期 | 修改 | 问题 | 是否可交付 |
| --- | --- | --- | --- | --- |
| v01 | 2026-07-10 | 完成粗剪 | 第 3 个镜头太长 | 否 |
| v02 | 2026-07-10 | 锁定配音 | 字幕未校对 | 否 |
| v03 | 2026-07-10 | 字幕完成 | 音乐略大 | 否 |
| v04 | 2026-07-10 | 混音调色 | 待客户确认 | 可评审 |
```

不要用 `最终版`、`最终版2`、`真的最终版`。这会让项目混乱。

## 5. 时间线轨道命名

时间线要有秩序。

通用轨道建议：

```text
V5  字幕强调 / 贴纸 / 标注
V4  主字幕
V3  B-roll / 补充画面
V2  主画面
V1  背景 / 基础视频

A5  音效
A4  音乐
A3  备用配音
A2  主配音
A1  原始现场声或参考音
```

如果软件不支持重命名轨道，也可以用颜色或文件命名区分。

### 5.1 轨道原则

- 主画面和字幕不要混乱堆叠。
- 主配音永远优先。
- 音乐和音效分开。
- 试验素材放在单独轨道，确认不用就删掉。
- 导出前隐藏或删除参考素材。

## 6. 粗剪

粗剪目标不是好看，而是把结构拼出来。

### 6.1 粗剪顺序

知识短视频建议：

```text
先放主配音
-> 按配音断句切时间线
-> 放对应画面
-> 删除明显无用镜头
-> 检查总时长
```

广告片可以：

```text
先放产品关键镜头
-> 放品牌信息
-> 放配音或字幕
-> 再加音乐节奏
```

短剧则可能：

```text
先按剧情顺序排镜头
-> 放角色对白
-> 检查情绪连续性
-> 再加音乐音效
```

### 6.2 粗剪检查

```markdown
| 检查项 | 是否通过 |
| --- | --- |
| 开头 3 秒是否说明看点 |
| 每个镜头是否有用途 |
| 配音是否和画面基本对应 |
| 总时长是否接近目标 |
| 明显失败素材是否删掉 |
| 是否缺少过渡镜头 |
```

粗剪阶段不要沉迷调色和字体。结构没定，细节越做越浪费。

## 7. 精剪

精剪开始处理节奏和细节。

### 7.1 精剪要做什么

- 缩短拖沓镜头。
- 调整镜头切点。
- 让字幕和配音同步。
- 加必要 B-roll。
- 处理开头和结尾。
- 调整音乐切点。
- 删除重复信息。

### 7.2 AI 素材的剪辑策略

AI 生成素材经常在前 0.5 秒或最后 0.5 秒不稳定。可以：

- 裁掉开头怪动作。
- 裁掉结尾变形帧。
- 用快速切换避开问题。
- 用字幕或画面遮挡轻微瑕疵。
- 用 B-roll 覆盖不适合口播的画面段。

但如果人物脸严重变形、产品错误、品牌元素错，就不要硬救。回到生成阶段重做。

## 8. 字幕工作流

字幕是短视频的第二条信息线。

Adobe Premiere Pro 官方帮助中有自动转录、字幕和字幕轨道导出的相关功能说明；CapCut/剪映也提供自动字幕和字幕导出相关能力。但具体入口会随版本变化，本书只讲原则。

### 8.1 字幕来源

字幕可以来自：

- 配音稿。
- 自动识别。
- 手动输入。
- 外部 SRT 文件。

最稳方式：

```text
配音稿
-> 自动或手动对齐
-> 人工校对
-> 关键词强调
-> 导出前复查
```

不要完全相信自动字幕。尤其是：

- 人名。
- 工具名。
- 英文缩写。
- 数字。
- 专业术语。

### 8.2 字幕断句

好的字幕不是越全越好。

原则：

- 一行不要太长。
- 每屏只放一个语义重点。
- 关键词可以加粗、变色或单独出现。
- 不要遮挡人物脸、产品和关键画面。
- 不要和平台 UI 冲突。

### 8.3 字幕检查表

```markdown
| 检查项 | 是否通过 | 备注 |
| --- | --- | --- |
| 是否无错别字 |  |  |
| 工具名是否正确 |  | Codex、LibTV、Seedance 等 |
| 数字和日期是否正确 |  |  |
| 字幕是否同步 |  |  |
| 字幕是否挡住主体 |  |  |
| 字幕风格是否统一 |  |  |
| 手机小屏是否看得清 |  |  |
```

## 9. 配音、音乐和音效混音

混音的核心原则：

```text
人声第一，音乐第二，音效第三。
```

### 9.1 音量关系

新手不用先记复杂响度标准，可以先做听感检查：

- 主配音听得清楚。
- 音乐不盖人声。
- 音效不刺耳。
- 开头不突然炸。
- 结尾不突然断。
- 手机外放能听清。

### 9.2 常见处理

- 配音整体音量保持稳定。
- 音乐在人声出现时降低。
- 音效只在关键动作上出现。
- 结尾音乐淡出。
- 不同素材音量差异过大时统一调整。

### 9.3 混音终审表

```markdown
| 项目 | 是否通过 |
| --- | --- |
| 开头第一句是否清楚 |
| 配音全程是否稳定 |
| 音乐是否压人声 |
| 音效是否过多 |
| 手机外放是否可听 |
| 戴耳机是否刺耳 |
| 结尾是否自然 |
```

## 10. 调色统一

AI 生成素材常常来自不同模型、不同批次、不同提示词。剪在一起时，会出现色彩不统一。

常见问题：

- 有的镜头偏蓝。
- 有的镜头偏黄。
- 有的镜头对比度太高。
- 有的镜头像电影，有的像广告。
- 人物肤色不一致。

### 10.1 新手调色目标

不要一上来追求电影级调色。先做到：

```text
亮度接近
色温接近
对比度接近
肤色不跳
整体风格一致
```

### 10.2 参考图

可以从第 10 卷的概念图或风格图中选一个作为参考：

```text
REF_style_ai_workflow_v01.png
```

剪辑时问：

- 所有镜头是否接近这个参考？
- 有没有某个镜头明显跳出来？
- 字幕颜色是否和画面冲突？

## 11. AI 瑕疵处理

不是所有瑕疵都要重生成。

### 11.1 可以剪辑处理的问题

| 问题 | 处理 |
| --- | --- |
| 开头半秒不稳 | 裁掉 |
| 结尾动作怪 | 提前切走 |
| 背景小瑕疵 | 用字幕或裁切弱化 |
| 某镜头太长 | 加 B-roll 或切到细节 |
| 画面节奏慢 | 加音乐切点或压缩时长 |
| 色彩略不同 | 做基础调色统一 |

### 11.2 必须重做的问题

| 问题 | 原因 |
| --- | --- |
| 主角脸严重变形 | 影响信任 |
| 产品外观错误 | 商业风险 |
| 品牌 Logo 错误 | 商业风险 |
| 手指或动作明显异常 | 观众出戏 |
| 字幕或画面生成乱码 | 不专业 |
| 人物身份混乱 | 连续性失败 |

剪辑不是万能修复器。要学会判断“救”还是“重做”。

## 12. 封面和标题

发布型视频必须考虑封面。

封面可以来自：

- 成片截图。
- 第 10 卷生成的封面图。
- 单独设计的封面背景。

### 12.1 封面检查

```markdown
| 检查项 | 是否通过 |
| --- | --- |
| 小图能否看清主体 |
| 标题是否清楚 |
| 是否和视频内容一致 |
| 是否避免夸大承诺 |
| 是否不遮挡人物脸 |
| 是否适合平台画幅 |
```

### 12.2 标题

标题要和内容一致。不要为了点击写视频里没有兑现的承诺。

Ross 风格案例可以用类似方向：

```text
AI 视频已经不是手搓提示词了
Codex + LibTV 如何做一条 AI 视频
从提示词到工作流：AI 视频新做法
```

标题不是越夸张越好，而是让目标观众知道为什么要点开。

## 13. 导出参数

导出前先确定平台和画幅。

### 13.1 常见平台

| 平台 | 常见画幅 | 常见用途 |
| --- | --- | --- |
| 抖音 | 9:16 | 竖屏短视频 |
| 视频号 | 9:16 | 竖屏内容 |
| 小红书 | 9:16 或 3:4 | 种草、教程 |
| B 站 | 16:9 | 横屏教程、长视频 |
| YouTube | 16:9 或 Shorts 9:16 | 长视频或短视频 |

### 13.2 通用导出检查

```markdown
| 检查项 | 是否通过 |
| --- | --- |
| 画幅是否正确 |
| 分辨率是否符合平台 |
| 帧率是否和素材基本一致 |
| 音频是否正常 |
| 字幕是否被导出 |
| 封面是否单独保存 |
| 文件名是否包含版本 |
| 导出后是否完整播放一遍 |
```

### 13.3 文件命名

```text
FINAL_ai_workflow_douyin_1080x1920_v01.mp4
FINAL_ai_workflow_bilibili_1920x1080_v01.mp4
COVER_ai_workflow_douyin_v01.png
```

不要导出成 `新建项目.mp4`。三天后你会忘记它是什么。

## 14. 平台发布前检查

导出后不要马上发布。先做终审。

### 14.1 技术终审

```markdown
| 项目 | 是否通过 |
| --- | --- |
| 视频能完整播放 |
| 开头无黑屏或空白 |
| 结尾无多余画面 |
| 音频左右声道正常 |
| 字幕无错别字 |
| 画面无明显变形 |
| Logo 和品牌信息正确 |
| 文件大小和格式可上传 |
```

### 14.2 内容终审

```markdown
| 项目 | 是否通过 |
| --- | --- |
| 开头是否有看点 |
| 内容是否兑现标题 |
| 信息是否准确 |
| 是否夸大效果 |
| 是否侵犯他人肖像、声音、音乐或商标 |
| 是否有平台敏感风险 |
```

终审最好隔 10 分钟再看一遍。刚剪完时，你会自动忽略很多问题。

## 15. Ross 风格短视频剪辑结构

根据项目中已核验的 Ross 抖音案例信息，这类视频大致是：

- 标题或钩子先行。
- 快速展示 AI 生成画面。
- 解释 Codex 和 LibTV 的工作流价值。
- 展示虚拟形象或关键画面。
- 给出趋势判断。

我们不能断言原视频剪辑时间线，但可以设计复刻结构：

| 时间 | 内容 | 剪辑重点 |
| --- | --- | --- |
| 00:00-00:03 | 钩子句 | 大字幕 + 强画面 |
| 00:03-00:10 | 展示成片质感 | 快速切换高质量画面 |
| 00:10-00:25 | 解释旧方式问题 | 手搓提示词、失败样本、项目文件 |
| 00:25-00:40 | 展示 Codex 作用 | 文件、分镜、任务卡、复盘 |
| 00:40-00:55 | 展示 LibTV 或同类生成 | 多镜头素材、候选版本 |
| 00:55-01:06 | 模型选择和组合使用 | 强调不是单一工具 |
| 01:06-01:12 | 总结 | 趋势判断 + 结尾字幕 |

### 15.1 时间线设计

```text
A2 主配音：贯穿全片
A4 背景音乐：科技感，低于人声
A5 关键音效：开头、转场、结尾
V2 主 AI 画面：成片素材
V3 工具和流程 B-roll：文件、分镜、任务卡
V4 字幕：黄色或高对比关键词
V5 标注：箭头、框选、关键词
```

复刻时不要只复刻视觉风格，更要复刻“信息结构”。

## 16. 商业交付文件包

商业项目不能只交一个 MP4。

推荐交付：


## 17. 常见错误

### 错误 1：把所有生成素材都放进成片

修正：只保留服务脚本的镜头。

### 错误 2：没有先锁定配音

修正：知识短视频先定声音主线，再精剪画面。

### 错误 3：字幕太满

修正：一屏一个重点，关键词突出。

### 错误 4：音乐太大

修正：人声第一，音乐第二。

### 错误 5：不做导出后复看

修正：导出后完整播放一遍，检查字幕、音频和结尾。

### 错误 6：工程文件没有版本

修正：使用 `v01_draft`、`v02_voice_locked`、`v03_final_review` 这类命名。

### 错误 7：试图剪辑修复所有 AI 问题

修正：轻微问题剪辑处理，严重问题回到生成阶段重做。

## 18. 故障排查

### 视频开头留不住人

处理：

- 把结论提前。
- 开头 3 秒加最强画面。
- 删除铺垫句。
- 加清楚标题字幕。
- 音乐开头做轻微钩子。

### 画面和配音对不上

处理：

- 先按配音断句重新切时间线。
- 缩短或重配过长句子。
- 增加 B-roll。
- 删除不服务信息的镜头。

### 字幕太乱

处理：

- 减少每屏字数。
- 统一字体、字号、颜色。
- 只突出关键词。
- 校对工具名和数字。

### 画面风格不统一

处理：

- 按参考图做基础调色。
- 降低某些镜头饱和度或对比度。
- 必要时换掉风格跳跃的镜头。
- 下一轮生成时统一风格提示词。

### 导出后字幕没了

处理：

- 检查字幕是烧录进视频还是单独字幕轨。
- 检查导出设置是否包含字幕。
- 导出后完整播放确认。
- 如果给平台上传 SRT，确认文件和视频版本一致。

### 上传平台后画质差

处理：

- 检查导出分辨率和码率是否过低。
- 避免反复压缩同一个文件。
- 上传前保留高质量母版。
- 需要不同平台时分别导出版本。

## 21. 本卷总结

AI 视频的后期流程可以记成：

```text
整理素材
-> 建立工程
-> 放配音
-> 粗剪画面
-> 精剪节奏
-> 做字幕
-> 加音乐和音效
-> 调色统一
-> 导出
-> 终审
-> 发布或交付
```

新手最该记住：

- AI 生成不是成片。
- 剪辑要服务信息，不是堆素材。
- 配音是主线，字幕是辅助信息线，音乐是情绪线。
- 导出后必须完整复看。
- 商业项目要留下授权和交付记录。

学完第 11-13 卷，你已经补齐了从画面素材到可发布成片的后期闭环。下一阶段才适合进入广告案例、短剧案例、n8n 自动化、MCP 工作流和商业变现。

### 21.1 新手最低成片标准

如果你现在还不知道自己剪得好不好，就先用这条最低标准：

```text
15 到 30 秒
一个明确主题
一个清楚开头
5 个以内主要镜头
一条主配音
一首不抢人声的背景音乐
字幕无错字
导出后完整看完一遍
```

不要第一条作品就追求复杂叙事。真正重要的是做出一个可以复盘的闭环：素材从哪里来，为什么这样剪，哪里失败了，下一版怎么改。只要这四个问题能回答，你的剪辑学习就已经进入正轨。

必须包含：

- 镜头文件。
- 配音内容。
- 字幕重点。
- 音乐或音效。

### 练习 3：字幕检查

找一段自动字幕，人工校对。

重点检查：

- 工具名。
- 数字。
- 英文缩写。
- 断句。
- 是否挡住主体。

### 练习 4：导出终审

导出一个 15 秒视频后，写终审表。

必须包含：

- 画幅。
- 分辨率。
- 音频。
- 字幕。
- 画面瑕疵。
- 文件名。
- 是否完整播放过。
