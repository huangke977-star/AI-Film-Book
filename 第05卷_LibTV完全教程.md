# 第 05 卷：LibTV 完全教程

## 学习目标

本卷先建立 LibTV 在 AI 视频工作流中的位置。具体按钮、模型名称、价格和账户权限可能随平台变化，应以实际界面和官方说明为准。

读完后你应该理解：

- LibTV 在项目中负责哪一环。
- 如何把 Codex 输出转成可生成的视频任务。
- 如何记录每次生成结果。
- 如何把失败样本反馈给 Codex 继续迭代。

## 1. LibTV 的工作流定位

在本项目中，LibTV 不承担全部创作，它主要负责生成画面素材。

```text
Codex：整理项目、写脚本、做分镜、写提示词、复盘
LibTV：根据提示词生成图片或视频素材
剪辑软件：组接、字幕、配乐、音效、调色、导出
```

这套分工的好处是清晰：

- Codex 管上下文和文件。
- LibTV 管生成。
- 剪辑软件管最终成片。

## 2. 输入前要准备什么

不要直接把一句话扔进平台。先准备：

- `brief.md`：这个视频要表达什么。
- `storyboard.md`：每个镜头的时长、画面、目的。
- `style_guide.md`：统一风格。
- `prompts/shot_xx.md`：每个镜头的生成提示词。

如果是人物视频，还要准备：

- `character_bible.md`：角色一致性描述。
- 参考图或角色图。
- 服装、发型、道具约束。

## 3. 单镜头生成流程

建议一次只生成一个镜头，不要把整条视频直接丢给模型。

步骤：

1. 打开 `storyboard.md`。
2. 选择一个镜头。
3. 打开对应的提示词文件。
4. 在 LibTV 中生成 2 到 4 个版本。
5. 下载到 `outputs/raw`。
6. 挑选可用版本复制到 `outputs/selected`。
7. 把失败原因写进 `iteration_log.md`。

## 4. 镜头提示词结构

推荐结构：

```text
主体：
场景：
动作：
镜头类型：
镜头运动：
光线：
色彩：
风格：
时长：
画幅：
负面约束：
```

示例：

```text
主体：一位年轻创作者坐在电脑前，屏幕显示 AI 视频项目文件夹
场景：夜晚工作室，桌面整洁，有键盘、显示器和一杯咖啡
动作：创作者敲击键盘，屏幕中快速出现分镜表
镜头类型：中近景
镜头运动：缓慢推近
光线：屏幕冷光照亮脸部，背景有柔和暖色台灯
色彩：冷暖对比，低饱和
风格：真实电影摄影，纪录片质感
时长：4 秒
画幅：9:16
负面约束：不要卡通感，不要过度磨皮，不要额外手指，不要文字乱码
```

## 5. 迭代记录模板

```markdown
## S01 第 1 轮

- 输入文件：prompts/shot_01.md
- 输出文件：outputs/raw/S01_V01_raw.mp4
- 问题：
  - 屏幕文字乱码。
  - 手部动作不自然。
  - 镜头运动过快。
- 修改方向：
  - 弱化屏幕文字，不要求可读。
  - 把“敲键盘”改成“手停在键盘上轻微移动”。
  - 运镜改为“very slow push in”。
```

## 6. 常见问题

### 人物不一致

可能原因：

- 没有角色圣经。
- 每个镜头都重新描述人物。
- 参考图或身份约束不足。

处理方法：

- 固定角色描述。
- 每个镜头引用同一份 `character_bible.md`。
- 减少服装、发型、年龄的变化。

### 镜头太乱

可能原因：

- 一个镜头里动作太多。
- 同时要求推镜、摇镜、环绕、变焦。
- 场景元素过多。

处理方法：

- 每个镜头只保留一个核心动作。
- 只使用一种主要运镜。
- 用剪辑完成复杂叙事，不要让单镜头承担太多。

### 画面有文字乱码

AI 视频模型通常不擅长生成可读文字。处理方法：

- 生成时减少文字要求。
- 后期在剪辑软件中加字幕或屏幕贴图。
- 让提示词写“abstract interface”而不是具体中文文本。
