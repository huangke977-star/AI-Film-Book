# 素材命名与归档规范

## 与第 02 卷的关系

第 02 卷讲环境搭建，本文件是可复制的素材命名规范。实际项目中可以直接复制本文件作为团队规则。

## 1. 为什么要规范命名

AI 视频生成会产生大量相似文件。如果没有命名规范，很快就会不知道哪个版本能用、哪个版本失败、哪个版本已经进入剪辑。

## 2. 推荐目录

```text
outputs/
├── raw/
├── selected/
├── rejected/
└── final/
```

## 3. 命名格式

```text
镜头号_版本号_状态_说明.ext
```

示例：

```text
S01_V01_raw_workspace_opening.mp4
S01_V02_selected_workspace_opening.mp4
S02_V03_rejected_text_garbled.mp4
S06_V01_final_phone_playback.mp4
```

## 4. 状态定义

| 状态 | 含义 |
| --- | --- |
| raw | 原始输出 |
| selected | 可进入剪辑 |
| rejected | 失败样本 |
| final | 已进入终版 |

## 5. 归档规则

- 不删除有代表性的失败样本。
- 每个失败样本至少记录一个失败原因。
- 成片导出后保留最终工程文件和封面。
- 发布文案、标题、话题标签也要归档。

参考图建议放入 `references/` 或 `assets/`，并在分镜表或任务卡中写清用途。不要把参考图和生成结果混在 `outputs/raw/` 里。
