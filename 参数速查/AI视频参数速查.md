# AI 视频参数速查

不同平台参数名称会变化，本表记录通用概念。

## 画幅

| 平台 | 推荐画幅 | 分辨率 |
| --- | --- | --- |
| 抖音 | 9:16 | 1080x1920 |
| 视频号 | 9:16 | 1080x1920 |
| 小红书 | 9:16 或 3:4 | 1080x1920 / 1080x1440 |
| B 站 | 16:9 | 1920x1080 |
| YouTube | 16:9 | 1920x1080 或 3840x2160 |

## 时长

| 类型 | 建议时长 |
| --- | --- |
| 单镜头测试 | 3 到 5 秒 |
| 抖音技巧类 | 20 到 45 秒 |
| 产品展示 | 15 到 60 秒 |
| 概念短片 | 30 到 90 秒 |

## 提示词字段

| 字段 | 必填程度 | 说明 |
| --- | --- | --- |
| 主体 | 必填 | 谁或什么在画面中 |
| 场景 | 必填 | 发生在哪里 |
| 动作 | 必填 | 主体做什么 |
| 镜头类型 | 强烈建议 | 远景、中景、近景、特写 |
| 运镜 | 强烈建议 | 固定、慢推、横移等 |
| 光线 | 强烈建议 | 决定质感 |
| 风格 | 强烈建议 | 真实、商业、纪录片等 |
| 负面约束 | 强烈建议 | 避免常见问题 |

## 常见负面词

```text
cartoon, anime, plastic skin, distorted hands, extra fingers, deformed face, unreadable text, watermark, logo, low quality, over sharpened
```

## LibTV 或同类工具生成前检查

| 项目 | 检查问题 |
| --- | --- |
| ratio | 是否明确 9:16、16:9 或其他画幅 |
| duration | 是否明确单镜头时长 |
| input type | 是文生视频、图生视频，还是口播素材 |
| reference image | 是否需要参考图 |
| negative prompt | 是否写清避免项 |
| output folder | 下载后是否放入 `outputs/raw/` |
| cost note | 是否准备记录次数、单价和结果 |

生成前先检查这张表，可以减少无效重试。

## 视频生成模型任务前检查

| 项目 | 检查问题 |
| --- | --- |
| 输入类型 | 文生视频、图生视频还是视频生视频 |
| 镜头目的 | 这个镜头服务哪一句信息 |
| 主体 | 谁或什么是画面中心 |
| 动作幅度 | 动作是否过于复杂 |
| 运镜 | 是否只使用一种主要运镜 |
| 时长 | 是否先控制在 3 到 5 秒 |
| 画幅 | 是否匹配发布平台 |
| 参考图 | 是否需要固定人物或场景 |
| 负面约束 | 是否排除乱码、变形、多余主体 |

这张表适合第 06 卷的 Seedance 或同类视频模型生成任务。先把镜头目的和动作幅度写清楚，再去选择模型和平台。

## 人物一致性生成前检查

| 项目 | 检查问题 |
| --- | --- |
| Character Bible | 是否有固定人物描述 |
| 参考图 | 是否命名清楚并放在项目目录 |
| 服装发型 | 是否跨镜头一致 |
| 动作幅度 | 是否避免复杂手势和大幅转身 |
| 镜头长度 | 是否先用短镜头降低漂移 |
| 失败记录 | 是否记录脸、年龄、服装、气质变化 |

人物一致性检查要在生成前做，不要等全片生成完再发现角色已经变成了几个人。

## AI 绘图到视频生成检查

| 项目 | 检查问题 |
| --- | --- |
| 用途 | 概念图、角色图、场景图、封面还是图生视频 |
| 画幅 | 是否匹配最终视频或封面 |
| 角色一致性 | 是否和 Character Bible 一致 |
| 文字 | 是否避免让图像模型生成复杂中文 |
| 文件名 | 是否能看出用途和版本 |
| 入库位置 | 是否放入 references 或 assets 目录 |

AI 绘图输出要先确定用途。没有用途的漂亮图，很难稳定进入视频生产。

## 音频生成前检查

| 项目 | 检查问题 |
| --- | --- |
| 声音用途 | 旁白、口播、角色配音还是广告 VO |
| 声音角色 | 年龄感、气质、语速和情绪是否写清 |
| 口播稿 | 是否已经把书面稿改成适合朗读的短句 |
| 停顿 | 关键句前后是否留停顿 |
| 输出版本 | 是否至少准备 2 到 3 个候选 |
| 授权 | 是否涉及真人声音、声音克隆或商业发布 |
| 入库 | 是否放入 `audio/voice/raw/` 或 `audio/voice/selected/` |

## 音乐生成前检查

| 项目 | 检查问题 |
| --- | --- |
| 音乐用途 | 背景音乐、主题曲、片头音效还是转场音效 |
| 情绪 | 核心情绪和不要的情绪是否写清 |
| 人声 | 是否需要无歌词，避免抢旁白 |
| 结构 | 开头、中段、结尾是否有不同功能 |
| 节奏 | 是否适合剪辑点和字幕阅读 |
| 上传素材 | 如果上传参考音频，是否拥有使用权 |
| 授权 | 商用、分发和客户复用范围是否记录 |

## 剪辑导出前检查

| 项目 | 检查问题 |
| --- | --- |
| 画幅 | 是否匹配平台，竖屏或横屏 |
| 分辨率 | 是否使用目标平台需要的清晰度 |
| 字幕 | 是否无错别字，是否被正确导出 |
| 音频 | 人声是否清楚，音乐是否压过配音 |
| 色彩 | 多个 AI 镜头是否风格统一 |
| 文件名 | 是否包含项目、平台、分辨率和版本 |
| 复看 | 导出后是否完整播放一遍 |

第 11-13 卷的核心检查顺序是：先让声音听得清楚，再让音乐托住情绪，最后让剪辑和导出稳定交付。
