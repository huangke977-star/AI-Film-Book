# 第 02 卷：环境搭建

## 学习目标

本卷解决一个问题：如何把 AI 视频创作从“聊天记录”变成“可复用项目”。

完成后，你应该拥有：

- 一个本地项目目录。
- 一个 Git 仓库。
- 一套素材命名规则。
- 一个 Codex 协作方式。
- 一个 LibTV 或同类视频生成工具的操作入口。

## 1. 推荐目录结构

每个作品单独建一个目录，不要把所有视频混在一起。

```text
projects/
└── 2026-07-ross-style-demo/
    ├── brief.md
    ├── script.md
    ├── storyboard.md
    ├── style_guide.md
    ├── character_bible.md
    ├── prompts/
    │   ├── shot_01.md
    │   └── shot_02.md
    ├── outputs/
    │   ├── raw/
    │   ├── selected/
    │   └── final/
    ├── edit/
    └── iteration_log.md
```

## 2. 安装和准备 Git

Git 的作用不是只为上传 GitHub。它更重要的价值是记录创作过程。

建议每完成一个重要阶段就提交一次：

```bash
git init
git add .
git commit -m "Create AI video project structure"
```

提交节奏：

- 完成 brief 后提交一次。
- 完成分镜后提交一次。
- 完成第一批提示词后提交一次。
- 完成复盘后提交一次。

## 3. 准备 Codex 工作方式

Codex 适合处理“有文件、有步骤、有迭代”的任务。使用时不要只说“帮我写提示词”，而是给它明确角色和项目路径。

推荐开场：

```text
你是我的 AI 视频项目制片助理。
请先阅读当前目录下的 brief.md、style_guide.md、storyboard.md。
接下来所有提示词都必须遵守这些文件。
如果发现信息缺失，先列出缺失项，再给出可执行的补全建议。
```

## 4. 准备 LibTV 或同类生成工具

LibTV 在这个项目中的定位是视频素材生成平台。你可以把 Codex 产出的镜头提示词投喂给 LibTV，再把生成结果下载到项目目录中复盘。

基本流程：

```text
Codex 写分镜和提示词
-> LibTV 生成单镜头素材
-> 下载素材到 outputs/raw
-> 人工挑选可用素材到 outputs/selected
-> Codex 根据失败样本改提示词
-> 剪辑软件合成
```

如果使用 Agent / Skill / MCP 方式连接 LibTV，需要把账号、模型、输出路径、会话记录等配置单独保存，并避免把密钥提交到公开仓库。

## 5. 素材命名规则

建议命名格式：

```text
镜头号_版本号_状态_简短说明.ext
```

示例：

```text
S01_V01_raw_city_opening.mp4
S01_V02_selected_city_opening.mp4
S03_V04_reject_face_drift.mp4
```

状态建议：

- `raw`：未筛选原始输出。
- `selected`：可进入剪辑。
- `reject`：失败样本，但保留用于复盘。
- `final`：终版输出。

## 6. 密钥和配置安全

不要把 API Key、登录 token、cookie、付费账号信息写进公开 Markdown。

建议：

- 使用 `.env` 保存本地密钥。
- 在 `.gitignore` 中忽略 `.env`。
- 文档中只写配置项名称，不写真实值。

## 7. 最小环境检查清单

- 本地有项目目录。
- 项目目录里有 `brief.md`。
- 项目目录里有 `storyboard.md`。
- 项目目录里有 `outputs/raw` 和 `outputs/selected`。
- 已安装 Git。
- 能正常使用 Codex。
- 能正常登录 LibTV 或其他视频生成工具。
- 知道最终用哪个剪辑软件合成。
