画面主体切换为男性虚拟口播形象，穿灰绿色连帽衫，正面对镜头说话，背景是城市夜景和湿润地面反光。画面还出现过类似手机拍摄小窗，像是在展示原始讲述来源或“真人/手机口播输入”。

## 4. 这条视频真正展示的趋势

这条视频的重点不是一条神奇提示词，而是：

```text
把视频制作变成一个由 Codex 管理的项目
再用 LibTV 或类似平台组织多模型生成
最后通过剪辑、字幕和复盘完成发布
```

所谓“手搓提示词的时代过去了”，更准确的理解是：

- 不再每次临时写一条长提示词。
- 不再把所有角色、场景、风格、镜头信息塞进单个输入框。
- 不再依赖聊天记录找版本。
- 改为用项目文件保存脚本、分镜、风格、角色、提示词、失败样本和导出记录。
- 用 Codex 这类 Agent 帮你维护上下文和批量生成材料。

## 5. 工具链判断

### Codex 的可能作用

从标题和成片结构看，Codex 更可能承担“制片助理”和“上下文工程师”的角色，而不是直接生成画面。

它适合做：

- 根据选题生成脚本和口播稿。
- 把 72 秒视频拆成章节和镜头。
- 生成每个镜头的提示词。
- 维护角色描述、背景风格、字幕文案。
- 记录每轮失败样本和修改意见。
- 整理项目目录，形成可复用模板。

### LibTV 的可能作用

LibTV 更可能承担生成和多模型调度角色：

- 根据提示词生成单镜头视频素材。
- 生成或承接虚拟口播人物。
- 切换不同背景场景。
- 连接 Seedance、Kling、GPT、Midjourney 等工具或模型能力。
- 降低在多个平台之间手动复制提示词的成本。

### 剪辑工具的作用

视频成片仍然依赖后期：

- 加黄色大字幕。
- 控制节奏。
- 处理画面切换。
- 同步口播和字幕。
- 发布到抖音并适配平台样式。

因此不能把这个案例理解为“Codex 一键生成成片”。更合理的理解是“Codex 组织项目，LibTV/模型生成素材，剪辑软件完成表达”。

## 6. 可复刻的制作流程

### 第 1 步：写项目 Brief

```markdown
# brief

主题：展示一个短视频作品可以全程由 Codex + LibTV 协作完成。
观众：AI 创作者、短视频创作者、想学习自动化工作流的人。
情绪：惊喜、效率感、未来感。
时长：72 秒以内。
平台：抖音。
核心观点：手搓提示词正在被项目化 Agent 工作流取代。
```

### 第 2 步：写口播稿

口播稿不要像文章，要像短视频：

```text
开场：制造信息差。
展示：让观众看到虚拟形象和画面质量。
成本：告诉观众门槛没有想象中高。
方法：说明多模型组合，而不是死磕单一模型。
结尾：收束到工作流和思维方式。
```

### 第 3 步：生成分镜表

| 镜头 | 时长 | 画面 | 目的 |
| --- | --- | --- | --- |
| S01 | 0-8s | 电影感人物近景，黄色字幕开场 | 证明画面质量，制造钩子 |
| S02 | 8-25s | 多个高质量 AI 场景快速切换 | 展示 AI 视频质感 |
| S03 | 25-38s | 男性虚拟口播形象，城市夜景背景 | 展示虚拟形象自然度 |
| S04 | 38-52s | 虚拟人物 + 火箭发射场背景 | 讲成本和模型选择 |
| S05 | 52-66s | 继续口播，强调多模型组合 | 讲方法论 |
| S06 | 66-72s | 结语字幕，回到核心观点 | 强化“手搓提示词时代过去了” |

### 第 4 步：建立风格手册

```text
真实电影摄影，冷暖对比，低饱和，浅景深，湿润地面反光，城市夜景，真实人物皮肤质感，黄色大字幕，抖音横版播放器内展示，AI 生成痕迹尽量弱化。
```

负面约束：

```text
不要卡通感，不要塑料皮肤，不要过度磨皮，不要乱码文字，不要多余手指，不要嘴型严重错位，不要背景闪烁，不要人物脸部漂移。
```

### 第 5 步：用 Codex 生成镜头提示词

给 Codex 的指令：

```text
你是我的 AI 视频制片助理。
请读取 brief.md、script.md、storyboard.md、style_guide.md。
为每个镜头生成可用于 LibTV 或同类视频平台的提示词。

要求：
1. 每个镜头单独保存为 prompts/shot_XX.md。
2. 每条提示词包含主体、场景、动作、镜头类型、运镜、光线、色彩、风格、负面约束。
3. 口播人物必须保持同一发型、服装、脸型和姿态。
4. 背景可以切换，但人物身份不能漂移。
5. 字幕不要交给视频模型生成，字幕由后期添加。
```

### 第 6 步：用 LibTV/多模型生成素材

生成策略：

- 人物口播镜头优先保证脸和嘴型稳定。
- 场景镜头优先保证电影感和真实光影。
- 背景切换不要太频繁，否则容易造成风格断裂。
- 每个镜头至少生成 2 到 4 个版本。

素材保存：

```text
outputs/raw/S01_V01_raw.mp4
outputs/raw/S01_V02_raw.mp4
outputs/selected/S01_V02_selected.mp4
```

### 第 7 步：剪辑和发布

后期重点：

- 黄色大字幕必须清楚，和口播节奏同步。
- 画面不要一直停留在口播人物，可以穿插高质量示例画面。
- 保留“成本低”“多模型组合”“不用手搓提示词”三个传播点。
- 结尾要收束到观点，而不是停在工具名。

## 7. 这条视频为什么容易传播

它同时满足四个传播条件：

1. 有结果：开头直接展示电影感画面。
2. 有人设：虚拟口播形象增强可信度和陪伴感。
3. 有反差：画面质量高，但成本被描述为很低。
4. 有观点：从“手搓提示词”转向“Agent 工作流”。

这个标题本身就有冲突：

```text
全程用codex和libtv完成 手搓提示词的时代过去了
```

它不是单纯说“这个工具很好用”，而是在说“旧方法要被新工作流替代”。这类表达更容易激发评论、收藏和转发。

## 8. 可以沉淀到本书的知识点

- AI 视频教学不能只讲工具按钮，要讲工作流。
- 角色一致性、背景切换、字幕后期是 AI 口播视频的三大重点。
- 多模型组合比单一模型崇拜更重要。
- Codex 的价值在于维护项目上下文，而不是替代所有创作判断。
- LibTV 这类平台的价值在于把生成任务、模型选择和结果管理集中起来。
- 成本叙事本身也是短视频传播的一部分。
