当前环境未能直接抓取视频画面、字幕和完整评论区信息。因此本文不是逐帧复述，而是基于标题线索、Codex 工作方式、LibTV Agent/技能公开信息和 AI 视频通用生产流程，复原一套可执行的制作方法。后续如果拿到视频截图或字幕，可以把本文件升级为逐帧版拆解。

## 1. 这条视频真正展示的趋势

这类作品的核心不是“某条神奇提示词”，而是：

```text
把视频制作变成一个由 Codex 管理的项目
再让 LibTV 负责具体画面生成
最后通过人工筛选和剪辑完成成片
```

所谓“手搓提示词的时代过去了”，更准确的理解是：

- 不再每次临时写一条长提示词。
- 不再把所有信息塞进单个输入框。
- 不再依赖记忆和聊天记录找版本。
- 改为用项目文件保存脚本、分镜、风格、角色、提示词、失败样本和导出记录。

## 2. 可能使用的工具

### Codex

角色：

- 项目管理员。
- 脚本和分镜生成器。
- 提示词批量生产器。
- 迭代记录整理器。
- Git/文件夹维护助手。

Codex 负责把“我要做一个视频”拆成多个可以执行的文件。

### LibTV

角色：

- AI 图像/视频生成平台。
- 接收 Codex 生成的提示词。
- 生成单镜头素材。
- 通过任务/会话方式返回结果。

如果使用 LibTV Skill 或类似 MCP/Agent 能力，Codex 可以进一步参与创建会话、提交任务、查询状态、下载结果等动作。

### 剪辑软件

可能是剪映、Premiere Pro、DaVinci Resolve 或其他工具。

角色：

- 选择素材。
- 调整节奏。
- 添加字幕。
- 添加配乐和音效。
- 调色和导出。

## 3. 推测的完整制作流程

### 第 1 步：确定作品目标

先不是写提示词，而是写 brief：

```markdown
# brief

主题：展示一个短视频作品可以全程由 Codex + LibTV 协作完成。
观众：AI 创作者、短视频创作者、想学习自动化工作流的人。
情绪：惊喜、效率感、未来感。
时长：30 到 60 秒。
平台：抖音。
核心观点：手搓提示词正在被项目化 Agent 工作流取代。
```

### 第 2 步：让 Codex 生成分镜

Codex 不直接生成成片，而是先输出镜头表。

示例：

| 镜头 | 时长 | 画面 | 运镜 | 目的 |
| --- | --- | --- | --- | --- |
| S01 | 3s | 桌面上打开 Codex 项目目录 | 俯拍慢推 | 建立“项目化制作”概念 |
| S02 | 4s | Codex 自动生成分镜文档 | 屏幕录制/特写 | 展示 Agent 写作能力 |
| S03 | 4s | LibTV 生成视频素材界面 | 横向滑动 | 展示生成环节 |
| S04 | 5s | 多个 AI 镜头素材快速闪回 | 快切 | 展示产出速度 |
| S05 | 4s | 剪辑时间线完成合成 | 推近 | 展示后期整合 |
| S06 | 3s | 成片在手机上播放 | 手持微动 | 回到发布场景 |

### 第 3 步：Codex 为每个镜头生成提示词

Codex 按统一模板输出：

```text
主体：
场景：
动作：
镜头：
光线：
色彩：
风格：
负面约束：
```

关键在于每个镜头共享同一份风格手册：

```text
真实电影摄影，现代创作者工作室，冷暖对比，低饱和，轻微胶片颗粒，浅景深，9:16 竖屏，适合抖音发布。
```

### 第 4 步：LibTV 生成镜头素材

建议每个镜头生成多个版本：

```text
S01_V01_raw.mp4
S01_V02_raw.mp4
S01_V03_raw.mp4
```

不要急着删失败样本。失败样本要写入迭代记录，因为它能帮助 Codex 判断下一轮提示词应该怎么改。

### 第 5 步：Codex 帮助复盘失败样本

把观察结果写给 Codex：

```text
S01 的桌面很真实，但屏幕文字乱码。
S02 的镜头运动太快。
S03 的界面看起来不像创作工具。
请只修改这三个镜头的提示词，不要改变整体风格。
```

Codex 的输出应该是局部修改，不是重写全片。

### 第 6 步：人工筛选和剪辑

AI 生成阶段不负责最终叙事，剪辑阶段才负责节奏。

操作重点：

- 删除最不稳定的镜头。
- 用字幕解释工具流程。
- 用快切压缩等待时间。
- 用音效强化动作。
- 用调色统一素材差异。

### 第 7 步：发布和复盘

发布时保留：

- 标题。
- 封面图。
- 话题标签。
- 使用工具。
- 成片路径。
- 生成成本。
- 失败原因。

## 4. 这类视频为什么容易火

它同时满足三个传播点：

1. 结果可见：观众能看到“真的做出来了”。
2. 方法新鲜：Codex + LibTV 的组合有工具稀缺感。
3. 观念转折：从“手搓提示词”到“Agent 工作流”，形成清晰冲突。

短视频标题可以围绕这个冲突写：

```text
我用 Codex + LibTV 做了一条 AI 短片，终于不用手搓提示词了
```

## 5. 如何复刻一个自己的版本

### 准备文件

```text
brief.md
storyboard.md
style_guide.md
prompts/
outputs/
iteration_log.md
```

### 给 Codex 的第一条指令

```text
你是我的 AI 视频制片助理。
我要做一条抖音短视频，主题是“用 Codex 和 LibTV 完成一条 AI 视频”。
请先生成 brief.md、storyboard.md 和 style_guide.md。
视频时长 30 秒，竖屏 9:16，风格是真实电影摄影、创作者工作室、冷暖对比。
```

### 给 Codex 的第二条指令

```text
请读取 brief.md、storyboard.md、style_guide.md。
为每个镜头生成单独的 LibTV 视频提示词，保存到 prompts/shot_01.md 到 prompts/shot_06.md。
每条提示词必须包含主体、场景、动作、镜头类型、运镜、光线、色彩、风格、负面约束。
```

### 生成后复盘指令

```text
我已经把 LibTV 第一轮输出放在 outputs/raw。
下面是每个镜头的问题：
S01：屏幕文字乱码。
S02：镜头运动太快。
S03：画面像软件广告，不像真实工作流。
请更新对应 prompts 文件，并在 iteration_log.md 中记录修改原因。
```
