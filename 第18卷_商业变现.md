# 第 18 卷：商业变现

## 学习目标

学完本卷后，你应该能做到：

- 理解 AI 视频能力如何变成可交付服务，而不是停留在工具演示。
- 会设计 AI 视频服务菜单、报价结构、成本核算和修改边界。
- 知道商业变现的核心是稳定交付、客户信任和长期维护，不是短期噱头。
- 能区分项目服务、模板产品、课程产品、顾问内训、个人 IP 内容等不同路径。
- 会记录版权、肖像、声音、音乐、字体和素材授权。
- 能设计 90 天成长路线，从作品集到首个小项目，再到可复购服务。
- 知道哪些做法不适合长期发展。

本卷不承诺收入，也不教投机打法。它讲的是如何把能力变成清楚、稳定、可复盘的交付。

## 本卷导读：变现来自可交付能力

很多人问：

```text
学完 AI 视频能不能赚钱？
```

更好的问题是：

```text
我能不能为某个客户解决一个明确问题？
我能不能按约定交付？
我能不能让客户愿意下次继续找我？
```

AI 视频商业化的底层不是“会用最新工具”，而是：

- 能理解需求。
- 能控制范围。
- 能做出作品。
- 能解释成本。
- 能管理修改。
- 能处理授权。
- 能持续维护。

工具会变，商业能力会留下。

## 1. AI 视频服务类型

你可以把 AI 视频能力包装成不同服务。

| 服务类型 | 适合客户 | 交付 |
| --- | --- | --- |
| 短视频广告 | 商家、品牌、课程、产品 | 15 到 60 秒成片、封面、文案 |
| 产品展示 | 电商、门店、品牌新品 | 产品镜头、卖点视频 |
| 个人 IP 内容 | 博主、讲师、顾问 | 选题、脚本、剪辑、字幕 |
| AI 短剧试播 | 内容团队、品牌栏目 | 试播集、人物表、分集大纲 |
| 企业培训素材 | 公司内部培训 | 课程短视频、流程演示 |
| 模板和资料包 | 创作者、新手 | Prompt、分镜、流程模板 |
| 顾问和内训 | 团队和机构 | 工作流设计、培训、复盘 |

新手不要一开始什么都接。建议先选择一到两个最清楚的服务。

## 2. 从样片到服务菜单

客户不会因为你说“我会 AI 视频”就下单。客户需要知道你能交付什么。

### 2.1 服务菜单模板

```markdown
# AI 视频服务菜单

## A. 15 秒产品展示短视频

适合：新品、门店、简单卖点。
交付：1 条 15 秒竖屏视频 + 1 张封面 + 1 份发布文案。
包含：Brief、脚本、5 镜头分镜、AI 素材生成、剪辑字幕、1 轮小修改。
不包含：线下拍摄、复杂人物剧情、多平台版本、长期素材授权。

## B. 30 秒商业广告短视频

适合：品牌新品、活动宣传、知识产品。
交付：1 条 30 秒竖屏视频 + 封面 + 文案 + 字幕文件。
包含：需求访谈、脚本、分镜、AI 生成、配音、音乐、剪辑、2 轮小修改。

## C. AI 短剧试播集

适合：想测试短剧形式的团队或个人。
交付：1 条 30 到 60 秒试播集 + 人设表 + 场景表 + 3 集大纲。
```

菜单不是越多越好。越清楚越好。

### 2.2 样片

每个服务最好有一个样片。

作品集结构：

```text
服务类型
-> 目标客户
-> 样片链接
-> 制作流程
-> 交付范围
-> 注意事项
```

不要只展示最终视频。展示流程会让客户更信任你。

## 3. 报价方式

本书不提供固定价格。你需要学报价结构。

### 3.1 常见报价维度

| 维度 | 说明 |
| --- | --- |
| 时长 | 15 秒、30 秒、60 秒 |
| 难度 | 产品准确性、人物一致性、场景复杂度 |
| 交付 | 成片、封面、文案、字幕、源文件 |
| 修改 | 包含几轮，什么算小修改 |
| 授权 | 配音、音乐、字体、素材、复用范围 |
| 周期 | 普通交付还是加急 |
| 平台 | 一个平台还是多平台版本 |

### 3.2 报价说明

```markdown
# Quote Scope

## 项目

30 秒 AI 产品广告。

## 包含

- 1 次需求访谈。
- 1 版脚本。
- 1 版分镜。
- AI 素材生成和筛选。
- 配音、音乐和字幕。
- 1 条 9:16 成片。
- 1 张封面。
- 2 轮小修改。

## 不包含

- 线下拍摄。
- 长期账号运营。
- 广告投放管理。
- 未约定的多语言版本。
- 未约定的源文件交付。

## 额外费用可能来自

- 大方向修改。
- 多平台版本。
- 加急。
- 客户新增产品或卖点。
- 需要额外授权素材。
```

报价不是为了把客户吓走，而是为了让双方知道项目边界。

## 4. 成本核算

AI 视频成本不只是工具费用。

### 4.1 成本项目

| 成本 | 说明 |
| --- | --- |
| 时间 | 访谈、脚本、分镜、生成、剪辑、修改 |
| 工具 | 视频生成、图像、配音、音乐、剪辑 |
| 重试 | 失败样本和多版本生成 |
| 授权 | 字体、音乐、素材、声音 |
| 沟通 | 客户会议、反馈整理 |
| 管理 | 文件归档、复盘、交付 |

### 4.2 成本表

```markdown
| 项目 | 预估 | 实际 |
| --- | --- | --- |
| 需求沟通 | 1 小时 |  |
| 脚本分镜 | 2 小时 |  |
| AI 生成 | 3 小时 |  |
| 剪辑后期 | 3 小时 |  |
| 修改 | 2 小时 |  |
| 工具消耗 | 待填 |  |
| 授权素材 | 待填 |  |
```

如果你不知道成本，就很难报价，也很难判断项目值不值得接。

## 5. 项目周期和修改边界

商业项目要写时间线。

### 5.1 项目周期示例

```markdown
Day 01：需求访谈和 Brief
Day 02：脚本和分镜
Day 03：AI 素材生成
Day 04：配音、音乐、粗剪
Day 05：精剪、字幕、初审
Day 06：修改
Day 07：终审和交付
```

### 5.2 修改边界

```markdown
小修改：
- 字幕文案微调。
- 音乐音量调整。
- 个别镜头替换。
- 节奏微调。

大修改：
- 改核心卖点。
- 改整体风格。
- 改目标观众。
- 重写脚本。
- 重做大部分镜头。
```

修改边界越早说清，合作越稳。

## 6. 客户沟通

客户沟通要有节奏。

### 6.1 沟通节点

```text
需求确认
-> Brief 确认
-> 脚本确认
-> 分镜确认
-> 初版成片确认
-> 修改确认
-> 终版交付确认
```

不要等视频都做完才让客户第一次看方向。

### 6.2 沟通记录

```markdown
| 日期 | 沟通内容 | 客户确认 | 下一步 |
| --- | --- | --- | --- |
| 07-10 | 确认产品卖点 | 清爽低酸 | 写脚本 |
| 07-11 | 确认脚本 | 去掉夸张表达 | 做分镜 |
| 07-12 | 初版成片 | 修改字幕 | 出 v02 |
```

商业项目不要只靠聊天印象。

## 7. 版权、肖像、声音和音乐授权

变现项目必须有授权意识。

### 7.1 需要确认的权利

| 类型 | 例子 |
| --- | --- |
| 肖像 | 真人照片、客户员工、模特 |
| 声音 | TTS、声音克隆、主播声音 |
| 音乐 | AI 音乐、素材库音乐、客户提供音乐 |
| 字体 | 商用字体、平台字体 |
| 图片 | 产品图、参考图、图库 |
| 商标 | 客户 Logo、竞品 Logo、场景中品牌 |
| IP | 角色、世界观、短剧设定 |

### 7.2 授权确认表

```markdown
| 素材 | 来源 | 用途 | 是否可商用 | 是否可跨平台 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 产品图 | 客户提供 | 广告画面 | 客户确认 | 客户确认 | 只用于本项目 |
| 配音 | AI 生成 | 主旁白 | 按账号确认 | 待确认 | 不用于其他项目 |
| 音乐 | AI 生成 | 背景音乐 | 按平台条款确认 | 待确认 | 记录生成日期 |
```

不确定时，不要把素材放进商业交付。

## 8. 个人 IP 内容

如果你不想一开始接客户，可以先做个人 IP 内容。

适合方向：

- AI 视频制作过程分享。
- 案例拆解。
- 工具对比。
- 提示词和上下文工程教程。
- 项目复盘。

### 8.1 内容栏目

```markdown
| 栏目 | 内容 |
| --- | --- |
```

个人 IP 的价值是积累信任和作品集。

## 11. 维护和复购

一次项目结束后，可以设计后续服务。

### 11.1 可复购服务

| 服务 | 说明 |
| --- | --- |
| 月度短视频包 | 每月固定条数 |
| 活动视频包 | 节日、新品、促销 |
| 模板维护 | 更新 Prompt、分镜和交付模板 |
| 账号内容策划 | 选题和脚本 |
| 团队陪跑 | 每周复盘和流程优化 |

### 11.2 维护记录

```markdown
| 月份 | 交付 | 问题 | 下月改进 |
| --- | --- | --- | --- |
| 7 月 | 4 条短视频 | 字幕风格不统一 | 建立字幕模板 |

```

复购来自稳定和改进，不是一次性惊艳。

## 12. 不建议做的变现方式

不建议：

- 使用未经授权的人脸或声音。
- 使用来源不清的音乐和素材。
- 对客户承诺不可控的结果。
- 用夸张宣传卖课程。
- 只卖工具名单，不交付方法。
- 用客户案例时不征得允许。
- 把学习样片包装成真实商业案例。

长期做下去，信用比短期转化更重要。

## 13. 90 天成长路线

### 第 1-30 天：做作品集

目标：

- 完成 3 条 15 到 30 秒样片。
- 每条都有 Brief、脚本、分镜、生成记录和复盘。
- 至少一条商业广告模拟案例。
- 至少一条 AI 短剧试播。

输出：

```text
portfolio/
├── ad_demo/
├── short_drama_pilot/
└── workflow_demo/
```

### 第 31-60 天：做小服务

目标：

- 找 1 到 3 个小范围项目。
- 控制交付范围。
- 写清报价和修改规则。
- 交付后复盘。

不要追求复杂项目。先证明你能交付。

### 第 61-90 天：产品化

目标：

- 总结服务菜单。
- 整理模板包。
- 形成报价结构。
- 建立客户复盘表。
- 发布 10 条个人 IP 内容。

这 90 天的目标不是成为行业大师，而是建立一个能持续进步的商业基础。

## 14. 作品集怎么做

作品集不是把视频链接堆在一起。客户需要看到你能解决什么问题。

### 14.1 单个作品集页面

```markdown
# Portfolio Case

## 项目类型

30 秒 AI 产品广告。

## 客户类型

本地饮品品牌。

## 目标

让年轻上班族知道新品冷萃咖啡。

## 交付

- 30 秒竖屏视频。
- 封面图。
- 发布文案。
- 字幕文件。

## 流程

需求访谈 -> Brief -> 脚本 -> 分镜 -> AI 生成 -> 配音音乐 -> 剪辑 -> 交付。

## 难点

产品外观要准确，音乐不能压过旁白。

## 解决方式

使用客户产品图作为参考，产品镜头单独评审，音乐在配音下方做避让。

## 结果

本案例为教学样片 / 客户确认后可展示。
```

### 14.2 作品集分类

```text
商业广告样片
AI 短剧试播
工作流教程
工具自动化演示
模板产品展示
```

刚开始没有客户案例时，可以明确写“教学样片”。不要把练习作品包装成真实客户项目。

## 15. 获客入口

新手可以从低成本入口开始。

| 入口 | 做法 |
| --- | --- |
| 个人内容 | 持续发布制作过程、案例拆解、失败复盘 |
| 朋友圈/社群 | 展示样片和服务菜单 |
| 本地商家 | 选择产品清楚的小店，做小范围试单 |
| 行业朋友 | 给课程、咨询、设计、摄影团队做补充服务 |
| 模板产品 | 把流程表、Prompt、分镜表做成资料包 |

### 15.1 不要只发结果

只发成片，别人可能觉得“这是某个工具自动做的”。你要展示：

- 需求怎么拆。
- 分镜怎么写。
- 哪些镜头失败。
- 为什么选择最终版。
- 如何交付。

过程就是你的专业证明。

## 16. 服务套餐设计

套餐不是为了套路客户，而是为了让范围清楚。

### 16.1 三档服务示例

```markdown
## 基础版：15 秒产品展示

适合：单一产品或活动。
交付：1 条 15 秒竖屏成片 + 1 张封面。
包含：脚本、5 镜头分镜、AI 生成、剪辑字幕、1 轮小修改。

## 标准版：30 秒商业短视频

适合：新品、门店、课程、个人 IP。
交付：1 条 30 秒成片 + 封面 + 发布文案 + 字幕文件。
包含：需求访谈、脚本、分镜、配音、音乐、剪辑、2 轮小修改。

## 增强版：系列内容试运营

适合：准备连续发布的账号或品牌。
交付：3 到 5 条短视频方案和成片，附复盘表。
包含：选题、脚本、分镜、成片、封面、复盘。
```

### 16.2 套餐边界

每个套餐都要写：

- 包含什么。
- 不包含什么。
- 修改几轮。
- 交付什么文件。
- 是否包含源文件。
- 是否包含授权素材。
- 是否包含平台运营。

没有边界的套餐，会变成无限服务。

## 17. 合同或确认单里的关键项

真实商业项目建议使用正式合同或至少有书面确认。下面是需要和客户确认的项目，不构成法律文本。

```markdown
| 项目 | 需要确认 |
| --- | --- |
| 项目名称 | 做什么 |
| 交付内容 | 成片、封面、文案、字幕、源文件 |
| 交付时间 | 哪天初稿，哪天终稿 |
| 修改次数 | 包含几轮，什么算大修改 |
| 素材来源 | 客户提供还是制作方生成 |
| 授权范围 | 可用于哪些平台、多久、是否可二次剪辑 |
| 付款节点 | 启动、初稿、终稿或其他方式 |
| 取消规则 | 项目中止如何处理 |
| 展示权限 | 是否允许放入作品集 |
```

最容易被忽略的是“展示权限”。如果客户不允许公开，你就不能把案例放到作品集里。

## 18. 客户入门流程

当有人问你“能不能做 AI 视频”，你可以按这套流程走：

```text
发送服务菜单
-> 让客户填写需求表
-> 约一次 20 分钟沟通
-> 输出 Brief 和范围
-> 确认报价和周期
-> 收集素材
-> 开始制作
```

### 18.1 客户素材清单

```markdown
| 素材 | 是否需要 | 客户提供 |
| --- | --- | --- |
| Logo | 是 |  |
| 产品图 | 是 |  |
| 品牌色 | 是 |  |
| 参考视频 | 可选 |  |
| 禁止表达 | 是 |  |
| 字体要求 | 可选 |  |
| 促销信息 | 可选 |  |
| 发布平台 | 是 |  |
```

客户素材越早收齐，后面返工越少。

## 19. 复购设计

复购不是靠不断推销，而是靠项目结束后的复盘和下一步建议。

### 19.1 交付后复盘报告

```markdown
# Project Review

## 本次交付

交付了哪些文件。

## 制作发现

哪些素材最有效。
哪些镜头最容易失败。
客户品牌最适合什么风格。

## 下次建议

是否可以做系列内容。
是否需要补充产品素材。
是否需要建立字幕模板。
是否需要做月度内容计划。
```

### 19.2 复购服务

```text
月度 4 条短视频
新品发布视频包
节日活动视频包
账号视觉模板维护
短剧试播后续集
团队流程培训
```

复购来自“我更懂你的项目了”，不是来自反复催客户。

## 24. 服务选择决策树

当客户来问你时，可以这样判断接什么服务：

```text
客户是否有明确产品或主题？
-> 没有：先做选题和 Brief 服务
-> 有：继续问

客户是否有素材？
-> 没有：先做素材准备或样片规划
-> 有：继续问

客户是否只需要一条视频？
-> 是：做单条广告或展示视频
-> 否：继续问

客户是否需要长期发布？
-> 是：做月度内容包或流程搭建
-> 否：做一次性项目
```

### 24.1 接单判断表

```markdown
| 问题 | 如果答案是否定 |
| --- | --- |
| 目标是否清楚 | 先做需求访谈 |
| 产品素材是否足够 | 先让客户补素材 |
| 授权是否明确 | 暂缓使用相关素材 |
| 时间是否合理 | 调整周期或范围 |
| 修改边界是否能确认 | 先不进入制作 |
| 是否符合你的能力 | 缩小范围或推荐其他人 |
```

不是所有机会都适合接。能拒绝不合适的项目，也是一种商业能力。

## 25. 交付红线

有些事情不要为了接单去做。

```markdown
| 情况 | 建议 |
| --- | --- |
| 客户无法提供产品真实信息 | 暂缓制作关键卖点 |
| 客户要求使用他人肖像或声音 | 要求明确授权 |
| 客户要求使用来源不清素材 | 换素材或让客户提供授权 |
| 客户要求做无法证明的宣传 | 改成可验证表达 |
| 客户不接受修改边界 | 不进入制作 |
| 客户要求不留任何书面确认 | 谨慎合作 |
```

短期看，拒绝会让你少接一个项目；长期看，它会保护你的信誉和精力。

## 26. 从免费样片到付费服务

新手可以做练习样片，但要避免长期免费劳动。

更稳的路径：

```text
教学样片：自己设定客户和产品
-> 试合作：范围很小，交付清楚
-> 正式小单：写清范围和修改
-> 标准服务：有菜单、模板和交付包
```

如果做低价或友情项目，也要写清：

- 这是一次什么合作。
- 交付到什么程度。
- 修改几次。
- 是否允许公开展示。
- 是否可作为后续案例。

没有边界的免费项目，最容易消耗你的学习热情。

## 27. 首次沟通消息模板

当你主动联系潜在客户时，不要上来就说自己会很多 AI 工具。可以这样写：

```text
你好，我最近在做 AI 短视频制作服务，方向是帮本地品牌或个人 IP 把产品卖点做成 15 到 30 秒竖屏视频。

我不是只提供工具生成，而是会先确认产品卖点、脚本、分镜、配音、字幕和交付范围。

如果你近期有新品、活动或账号内容需要，我可以先给你看一个样片和服务范围。你觉得合适的话，我们再确认是否适合合作。
```

这段话的重点是：你解决什么问题，你怎么交付，客户下一步可以做什么。不要把沟通变成炫耀工具名称。

先讲价值，再讲工具。

稳稳住。

## 28. 常见错误

### 错误 1：没有作品集就报价

修正：先做 3 个可复盘样片。

### 错误 2：报价不写范围

修正：明确交付物、修改次数和不包含内容。

### 错误 3：不记录成本

修正：每个项目写时间、工具、重试和授权成本。

### 错误 4：素材授权不清

修正：建立授权确认表。

### 错误 5：什么客户都接

修正：先选择你能稳定交付的类型。

### 错误 6：只讲工具不讲结果

修正：客户关心的是能否解决问题。

## 29. 故障排查

### 客户觉得贵

处理：

- 展示交付范围。
- 拆分基础版和增强版。
- 减少交付物，而不是无限降价。
- 说明修改次数和授权边界。

### 项目做亏了

处理：

- 复盘时间花在哪里。
- 记录生成失败次数。
- 下次缩小范围或提高报价。
- 把重复工作模板化。

### 客户一直改

处理：

- 回到修改规则。
- 区分小修改和大方向修改。
- 每轮修改都记录。
- 超出范围时重新确认费用和周期。

### 不知道做什么服务

处理：

- 从你最稳定的样片反推服务。
- 先做 15 到 30 秒短视频。
- 选择一个行业或场景。
- 不要一开始做全品类。

### 个人 IP 没反馈

处理：

- 检查内容是否太泛。
- 展示真实流程和失败复盘。
- 把教程拆小。
- 持续发布，不要只发成果图。

## 30. 本卷总结

AI 视频变现的顺序是：

```text
先学会流程
-> 做出样片
-> 记录过程
-> 形成服务菜单
-> 控制报价范围
-> 稳定交付
-> 复盘改进
-> 产品化和复购
```

不要把变现理解成卖一个神奇工具。真正能长期发展的，是你能把复杂流程变简单，把不确定项目变可交付，把一次合作变成信任。

学完本书后，你可以从一个 15 秒试验片开始。每次只做小一点、稳一点、记录清楚一点。作品会积累，流程会成熟，客户也会更容易理解你的价值。
